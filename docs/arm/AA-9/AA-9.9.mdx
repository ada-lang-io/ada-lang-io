#### Dynamic Semantics

<MarginText>1</MarginText>
<p>For a <code><a href="/docs/arm/AA-4/AA-4.1#S0093">prefix</a></code> T that is of a task type [(after any implicit dereference)], the following attributes are defined: <br /></p>

<MarginText>2</MarginText>
<dt><br/>T'Callable</dt>
<dl>
<dd>Yields the value True when the task denoted by T is <i>callable</i>, and False otherwise; a task is callable unless it is completed or abnormal. The value of this attribute is of the predefined type Boolean.<br /></dd>
<MarginText>3</MarginText>
<dt><br/>T'Terminated</dt>
<dd>Yields the value True if the task denoted by T is terminated, and False otherwise. The value of this attribute is of the predefined type Boolean. <br /></dd>
</dl>
<MarginText>4</MarginText>
<p>For a <code><a href="/docs/arm/AA-4/AA-4.1#S0093">prefix</a></code> E that denotes an entry of a task or protected unit, the following attribute is defined. This attribute is only allowed within the body of the task or protected unit, but excluding, in the case of an entry of a task unit, within any program unit that is, itself, inner to the body of the task unit. <br /></p>

<MarginText>5</MarginText>
<dt><br/>E'Count</dt>
<dl>
<dd>Yields the number of calls presently queued on the entry E of the current instance of the unit. The value of this attribute is of the type <i>universal{"_"}integer</i>.<br /></dd>
</dl>
<AnnotatedOnly>
<MarginText>6</MarginText>
<Admonition type="aarm" aarm="note">
NOTE 1   For the Count attribute, the entry can be either a single entry or an entry of a family. The name of the entry or entry family can be either a <code><a href="/docs/arm/AA-4/AA-4.1#S0092">direct_name</a></code> or an expanded name.<br />
</Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>7/5</MarginText>
<MarginInfo items={["AI12-0442-1"]} />
<Admonition type="aarm" aarm="note">
NOTE 2   Within task units, by interrogating the attribute E'Count an algorithm can allow for the increase of the value of this attribute for incoming entry calls, and its decrease, for example with <code><a href="/docs/arm/AA-9/AA-9.7#S0276">timed_entry_call</a></code>s. A <code><a href="/docs/arm/AA-9/AA-9.7#S0279">conditional_entry_call</a></code> can also briefly increase this value, even if the conditional call is not accepted.<br />
</Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>8/5</MarginText>
<MarginInfo items={["AI12-0442-1"]} />
<Admonition type="aarm" aarm="note">
NOTE 3   Within protected units, by interrogating the attribute E'Count in the <code><a href="/docs/arm/AA-9/AA-9.5#S0262">entry_barrier</a></code> for the entry E an algorithm can allow for the evaluation of the <code><a href="/docs/arm/AA-4/AA-4.5#S0150">condition</a></code> of the barrier both before and after queuing a given caller. <br />
</Admonition>
</AnnotatedOnly>
