<MarginText>1</MarginText>
<p>[An <code><a href="/docs/arm/AA-5/AA-5.7#S0193">exit_statement</a></code> is used to complete the execution of an enclosing <code><a href="/docs/arm/AA-5/AA-5.5#S0178">loop_statement</a></code>; the completion is conditional if the <code><a href="/docs/arm/AA-5/AA-5.7#S0193">exit_statement</a></code> includes a <code><a href="/docs/arm/AA-4/AA-4.5#S0150">condition</a></code>.] <br /></p>


#### Syntax

<MarginText>2</MarginText>

<CodeBlock>
<code>exit{"_"}statement</code><a id="S0193"></a><code> ::= </code><br />   <strong>exit</strong> [<em>loop{"_"}</em><code><a href="/docs/arm/AA-4/AA-4.1#S0091">name</a></code>] [<strong>when</strong> <code><a href="/docs/arm/AA-4/AA-4.5#S0150">condition</a></code>];<br />

</CodeBlock>

#### Name Resolution Rules

<MarginText>3</MarginText>
<p>The <em>loop{"_"}</em><code><a href="/docs/arm/AA-4/AA-4.1#S0091">name</a></code>, if any, in an <code><a href="/docs/arm/AA-5/AA-5.7#S0193">exit_statement</a></code> shall resolve to denote a <code><a href="/docs/arm/AA-5/AA-5.5#S0178">loop_statement</a></code>. <br /></p>


#### Legality Rules

<MarginText>4</MarginText>
<p>Each <code><a href="/docs/arm/AA-5/AA-5.7#S0193">exit{"_"}statement</a></code> <em>applies to</em> a <code><a href="/docs/arm/AA-5/AA-5.5#S0178">loop{"_"}statement</a></code>; this is the <code><a href="/docs/arm/AA-5/AA-5.5#S0178">loop{"_"}statement</a></code> being exited. An <code><a href="/docs/arm/AA-5/AA-5.7#S0193">exit{"_"}statement</a></code> with a <code><a href="/docs/arm/AA-4/AA-4.1#S0091">name</a></code> is only allowed within the <code><a href="/docs/arm/AA-5/AA-5.5#S0178">loop{"_"}statement</a></code> denoted by the <code><a href="/docs/arm/AA-4/AA-4.1#S0091">name</a></code>, and applies to that <code><a href="/docs/arm/AA-5/AA-5.5#S0178">loop{"_"}statement</a></code>. An <code><a href="/docs/arm/AA-5/AA-5.7#S0193">exit{"_"}statement</a></code> without a <code><a href="/docs/arm/AA-4/AA-4.1#S0091">name</a></code> is only allowed within a <code><a href="/docs/arm/AA-5/AA-5.5#S0178">loop{"_"}statement</a></code>, and applies to the innermost enclosing one. An <code><a href="/docs/arm/AA-5/AA-5.7#S0193">exit{"_"}statement</a></code> that applies to a given <code><a href="/docs/arm/AA-5/AA-5.5#S0178">loop{"_"}statement</a></code> shall not appear within a body or <code><a href="/docs/arm/AA-9/AA-9.5#S0258">accept{"_"}statement</a></code>, if this construct is itself enclosed by the given <code><a href="/docs/arm/AA-5/AA-5.5#S0178">loop_statement</a></code>. <br /></p>


#### Dynamic Semantics

<MarginText>5</MarginText>
<p>For the execution of an <code><a href="/docs/arm/AA-5/AA-5.7#S0193">exit_statement</a></code>, the <code><a href="/docs/arm/AA-4/AA-4.5#S0150">condition</a></code>, if present, is first evaluated. If the value of the <code><a href="/docs/arm/AA-4/AA-4.5#S0150">condition</a></code> is True, or if there is no <code><a href="/docs/arm/AA-4/AA-4.5#S0150">condition</a></code>, a transfer of control is done to complete the <code><a href="/docs/arm/AA-5/AA-5.5#S0178">loop{"_"}statement</a></code>. If the value of the <code><a href="/docs/arm/AA-4/AA-4.5#S0150">condition</a></code> is False, no transfer of control takes place. <br /></p>

<AnnotatedOnly>
<MarginText>6</MarginText>
<Admonition type="aarm" aarm="note">
NOTE   Several nested loops can be exited by an <code><a href="/docs/arm/AA-5/AA-5.7#S0193">exit_statement</a></code> that names the outer loop. <br />
</Admonition>
</AnnotatedOnly>


#### Examples

<MarginText>7</MarginText>
<p><em>Examples of loops with exit statements:</em> <br /></p>

<MarginText>8</MarginText>
<CodeBlock language="ada">
for N in 1 .. Max{"_"}Num{"_"}Items loop{"\n"}   Get{"_"}New{"_"}Item(New{"_"}Item);{"\n"}   Merge{"_"}Item(New{"_"}Item, Storage{"_"}File);{"\n"}   exit when New{"_"}Item = Terminal{"_"}Item;{"\n"}end loop;{"\n"}
<MarginText>9</MarginText>
Main{"_"}Cycle:{"\n"}   loop{"\n"}      --  initial statements{"\n"}      exit Main{"_"}Cycle when Found;{"\n"}      --  final statements{"\n"}   end loop Main{"_"}Cycle;{"\n"}

</CodeBlock>
