<MarginInfo items={["AI12-0442-1"]} />
<p>This Reference Manual sometimes gives advice about handling certain target machine dependences. Each Ada implementation is required to  document whether that advice is followed: <br /></p>

<AnnotatedOnly>
<MarginText>1.a/2</MarginText>
<Admonition type="aarm" aarm="ramification">
<strong></strong>Some of the items in this list require documentation only for implementations that conform to Specialized Needs Annexes. <br /></Admonition>
</AnnotatedOnly>

<MarginText>2/2</MarginText>
<ul>
<li>Program_Error should be raised when an unsupported Specialized Needs Annex feature is used at run time. See <a href="../AA-1/AA-1.1#Subclause_1.1.3">1.1.3</a>(20).<br /></li>
<MarginText>3/2</MarginText>
<li>Implementation-defined extensions to the functionality of a language-defined library unit should be provided by adding children to the library unit. See <a href="../AA-1/AA-1.1#Subclause_1.1.3">1.1.3</a>(21).<br /></li>
<MarginText>4/2</MarginText>
<li>If a bounded error or erroneous execution is detected, Program_Error should be raised. See <a href="../AA-1/AA-1.1#Subclause_1.1.5">1.1.5</a>(12).<br /></li>
<MarginText>5/2</MarginText>
<li>Implementation-defined pragmas should have no semantic effect for error-free programs. See <a href="../AA-2/AA-2.8">2.8</a>(16/3).<br /></li>
<MarginText>6/2</MarginText>
<li>Implementation-defined pragmas should not make an illegal program legal, unless they complete a declaration or configure the <code><a href="../AA-10/AA-10.1#S0287">library_item</a></code>s in an environment. See <a href="../AA-2/AA-2.8">2.8</a>(19).<br /></li>
<MarginText>7/2</MarginText>
<li>Long_Integer should be declared in Standard if the target supports 32-bit arithmetic. No other named integer subtypes should be declared in Standard. See <a href="../AA-3/AA-3.5#Subclause_3.5.4">3.5.4</a>(28).<br /></li>
<MarginText>8/2</MarginText>
<li>For a two's complement target, modular types with a binary modulus up to System.Max_Int*2+2 should be supported. A nonbinary modulus up to Integer'Last should be supported. See <a href="../AA-3/AA-3.5#Subclause_3.5.4">3.5.4</a>(29).<br /></li>
<MarginText>9/2</MarginText>
<li>Program_Error should be raised for the evaluation of S'Pos for an enumeration type, if the value of the operand does not correspond to the internal code for any enumeration literal of the type. See <a href="../AA-3/AA-3.5#Subclause_3.5.5">3.5.5</a>(8).<br /></li>
<MarginText>10/2</MarginText>
<li>Long_Float should be declared in Standard if the target supports 11 or more digits of precision. No other named float subtypes should be declared in Standard. See <a href="../AA-3/AA-3.5#Subclause_3.5.7">3.5.7</a>(17).<br /></li>
<MarginText>11/2</MarginText>
<li>Multidimensional arrays should be represented in row-major order, unless the array has convention Fortran. See <a href="../AA-3/AA-3.6#Subclause_3.6.2">3.6.2</a>(11/3).<br /></li>
<MarginText>12/3</MarginText>
<li>Tags.Internal_Tag should return the tag of a type, if one exists, whose innermost master is a master of the point of the function call. See <a href="../AA-3/AA-3.9">3.9</a>(26.1/3).<br /></li>
<MarginText>13/2</MarginText>
<li>A real static expression with a nonformal type that is not part of a larger static expression should be rounded the same as the target system. See <a href="../AA-4/AA-4.9">4.9</a>(38.1/2).<br /></li>
<MarginText>13.1/5</MarginText>
<li>For each language-defined private type T, T'Image should generate an image that would be meaningful based only on the relevant public interfaces. See <a href="../AA-4/AA-4.10">4.10</a>(56).<br /></li>
<MarginText>14/2</MarginText>
<li>The value of Duration'Small should be no greater than 100 microseconds. See <a href="../AA-9/AA-9.6">9.6</a>(30).<br /></li>
<MarginText>15/2</MarginText>
<li>The time base for <code><a href="../AA-9/AA-9.6#S0268">delay_relative_statement</a></code>s should be monotonic. See <a href="../AA-9/AA-9.6">9.6</a>(31/5).<br /></li>
<MarginText>16/2</MarginText>
<li>Leap seconds should be supported if the target system supports them. Otherwise, operations in Calendar.Formatting should return results consistent with no leap seconds. See <a href="../AA-9/AA-9.6#Subclause_9.6.1">9.6.1</a>(89/2).<br /></li>
<MarginText>17/5</MarginText>
<li><em>This paragraph was deleted.</em>  <br /></li>
<MarginText>18/2</MarginText>
<li>A type declared in a preelaborated package should have the same representation in every elaboration of a given version of the package. See <a href="../AA-10/AA-10.2#Subclause_10.2.1">10.2.1</a>(12).<br /></li>
<MarginText>19/2</MarginText>
<li>Exception_Information should provide information useful for debugging, and should include the Exception_Name and Exception_Message. See <a href="../AA-11/AA-11.4#Subclause_11.4.1">11.4.1</a>(19).<br /></li>
<MarginText>20/2</MarginText>
<li>Exception_Message by default should be short, provide information useful for debugging, and should not include the Exception_Name. See <a href="../AA-11/AA-11.4#Subclause_11.4.1">11.4.1</a>(19).<br /></li>
<MarginText>21/2</MarginText>
<li>Code executed for checks that have been suppressed should be minimized. See <a href="../AA-11/AA-11.5">11.5</a>(28).<br /></li>
<MarginText>22/2</MarginText>
<li>The recommended level of support for all representation items should be followed. See <a href="../AA-13/AA-13.1">13.1</a>(28/5).<br /></li>
<MarginText>23/2</MarginText>
<li>Storage allocated to objects of a packed type should be minimized. See <a href="../AA-13/AA-13.2">13.2</a>(6).<br /></li>
<MarginText>24/3</MarginText>
<li>The recommended level of support for the Pack aspect should be followed. See <a href="../AA-13/AA-13.2">13.2</a>(9).<br /></li>
<MarginText>25/2</MarginText>
<li>For an array X, X'Address should point at the first component of the array rather than the array bounds. See <a href="../AA-13/AA-13.3">13.3</a>(14).<br /></li>
<MarginText>26/2</MarginText>
<li>The recommended level of support for the Address attribute should be followed. See <a href="../AA-13/AA-13.3">13.3</a>(19).<br /></li>
<MarginText>26.1/3</MarginText>
<li>For any tagged specific subtype <em>S</em>, <em>S</em>'Class'Alignment should equal <em>S</em>'Alignment. See <a href="../AA-13/AA-13.3">13.3</a>(28).<br /></li>
<MarginText>27/2</MarginText>
<li>The recommended level of support for the Alignment attribute should be followed. See <a href="../AA-13/AA-13.3">13.3</a>(35).<br /></li>
<MarginText>28/2</MarginText>
<li>The Size of an array object should not include its bounds. See <a href="../AA-13/AA-13.3">13.3</a>(41.1/2).<br /></li>
<MarginText>29/5</MarginText>
<li>If the Size of a subtype is nonconfirming and allows for efficient independent addressability, then the Object_Size of the subtype (unless otherwise specified)  should equal the Size of the subtype. See <a href="../AA-13/AA-13.3">13.3</a>(52).<br /></li>
<MarginText>30/2</MarginText>
<li>A Size clause on a composite subtype should not affect the internal layout of components. See <a href="../AA-13/AA-13.3">13.3</a>(53).<br /></li>
<MarginText>31/2</MarginText>
<li>The recommended level of support for the Size attribute should be followed. See <a href="../AA-13/AA-13.3">13.3</a>(56).<br /></li>
<MarginText>31.1/5</MarginText>
<li>An Object_Size clause on a composite type should not affect the internal layout of components. See <a href="../AA-13/AA-13.3">13.3</a>(58).<br /></li>
<MarginText>31.2/5</MarginText>
<li>If S is a definite first subtype for which Object_Size is not specified, S'Object_Size should be the smallest multiple of the storage element size larger than or equal to S'Size that is consistent with the alignment of S. See <a href="../AA-13/AA-13.3">13.3</a>(58).<br /></li>
<MarginText>31.3/5</MarginText>
<li>The recommended level of support for the Object_Size attribute should be followed. See <a href="../AA-13/AA-13.3">13.3</a>(58).<br /></li>
<MarginText>31.4/5</MarginText>
<li>The Size of most objects of a subtype should equal the Object_Size of the subtype. See <a href="../AA-13/AA-13.3">13.3</a>(58).<br /></li>
<MarginText>32/2</MarginText>
<li>The recommended level of support for the Component_Size attribute should be followed. See <a href="../AA-13/AA-13.3">13.3</a>(73).<br /></li>
<MarginText>33/2</MarginText>
<li>The recommended level of support for <code><a href="../AA-13/AA-13.4#S0350">enumeration_representation_clause</a></code>s should be followed. See <a href="../AA-13/AA-13.4">13.4</a>(10).<br /></li>
<MarginText>34/2</MarginText>
<li>The recommended level of support for <code><a href="../AA-13/AA-13.5#S0352">record_representation_clause</a></code>s should be followed. See <a href="../AA-13/AA-13.5#Subclause_13.5.1">13.5.1</a>(22).<br /></li>
<MarginText>35/2</MarginText>
<li>If a component is represented using a pointer to the actual data of the component which is contiguous with the rest of the object, then the storage place attributes should reflect the place of the actual data. If a component is allocated discontiguously from the rest of the object, then a warning should be generated upon reference to one of its storage place attributes. See <a href="../AA-13/AA-13.5#Subclause_13.5.2">13.5.2</a>(5).<br /></li>
<MarginText>36/2</MarginText>
<li>The recommended level of support for the nondefault bit ordering should be followed. See <a href="../AA-13/AA-13.5#Subclause_13.5.3">13.5.3</a>(8).<br /></li>
<MarginText>37/2</MarginText>
<li>Type System.Address should be a private type. See <a href="../AA-13/AA-13.7">13.7</a>(37).<br /></li>
<MarginText>38/2</MarginText>
<li>Operations in System and its children should reflect the target environment; operations that do not make sense should raise Program_Error. See <a href="../AA-13/AA-13.7#Subclause_13.7.1">13.7.1</a>(16).<br /></li>
<MarginText>39/2</MarginText>
<li>Since the Size of an array object generally does not include its bounds, the bounds should not be part of the converted data in an instance of Unchecked_Conversion. See <a href="../AA-13/AA-13.9">13.9</a>(14/2).<br /></li>
<MarginText>40/2</MarginText>
<li>There should not be unnecessary runtime checks on the result of an Unchecked_Conversion; the result should be returned by reference when possible. Restrictions on Unchecked_Conversions should be avoided. See <a href="../AA-13/AA-13.9">13.9</a>(15).<br /></li>
<MarginText>41/2</MarginText>
<li>The recommended level of support for Unchecked_Conversion should be followed. See <a href="../AA-13/AA-13.9">13.9</a>(17).<br /></li>
<MarginText>42/2</MarginText>
<li>Any cases in which heap storage is dynamically allocated other than as part of the evaluation of an <code><a href="../AA-4/AA-4.8#S0164">allocator</a></code> should be documented. See <a href="../AA-13/AA-13.11">13.11</a>(23).<br /></li>
<MarginText>43/2</MarginText>
<li>A default storage pool for an access-to-constant type should not have overhead to support deallocation of individual objects. See <a href="../AA-13/AA-13.11">13.11</a>(24).<br /></li>
<MarginText>44/2</MarginText>
<li>Usually, a storage pool for an access discriminant or access parameter should be created at the point of an <code><a href="../AA-4/AA-4.8#S0164">allocator</a></code>, and be reclaimed when the designated object becomes inaccessible. For other anonymous access types, the pool should be created at the point where the type is elaborated and may have no mechanism for the  deallocation of individual objects. See <a href="../AA-13/AA-13.11">13.11</a>(25).<br /></li>
<MarginText>45/2</MarginText>
<li>For a standard storage pool, an instance of Unchecked_Deallocation should actually reclaim the storage. See <a href="../AA-13/AA-13.11#Subclause_13.11.2">13.11.2</a>(17).<br /></li>
<MarginText>45.1/3</MarginText>
<li>A call on an instance of Unchecked_Deallocation with a nonnull access value should raise Program_Error if the actual access type of the instance is a type for which the Storage_Size has been specified to be zero or is defined by the language to be zero. See <a href="../AA-13/AA-13.11#Subclause_13.11.2">13.11.2</a>(17.1/3).<br /></li>
<MarginText>45.2/5</MarginText>
<li>Streams.Storage.Bounded.Stream_Type objects should be implemented without implicit pointers or dynamic allocation. See <a href="../AA-13/AA-13.13#Subclause_13.13.1">13.13.1</a>(37).<br /></li>
<MarginText>46/2</MarginText>
<li>If not specified, the value of Stream_Size for an elementary type should be the number of bits that corresponds to the minimum number of stream elements required by the first subtype of the type, rounded up to the nearest factor or multiple of the word size that is also a multiple of the stream element size. See <a href="../AA-13/AA-13.13#Subclause_13.13.2">13.13.2</a>(1.6/2).<br /></li>
<MarginText>47/2</MarginText>
<li>The recommended level of support for the Stream_Size attribute should be followed. See <a href="../AA-13/AA-13.13#Subclause_13.13.2">13.13.2</a>(1.8/2).<br /></li>
<MarginText>48/2</MarginText>
<li>If an implementation provides additional named predefined integer types, then the names should end with &ldquo;Integer&rdquo;. If an implementation provides additional named predefined floating point types, then the names should end with &ldquo;Float&rdquo;. See <a href="../AA-A/AA-A.1">A.1</a>(52).<br /></li>
<MarginText>49/2</MarginText>
<li>Implementation-defined operations on Wide_Character, Wide_String, Wide_Wide_Character, and Wide_Wide_String should be child units of Wide_Characters or Wide_Wide_Characters. See <a href="../AA-A/AA-A.3#Subclause_A.3.1">A.3.1</a>(7/3).<br /></li>
<MarginText>49.1/3</MarginText>
<li>The string returned by Wide_Characters.Handling.Character_Set_Version should include either &ldquo;10646:&rdquo; or &ldquo;Unicode&rdquo;. See <a href="../AA-A/AA-A.3#Subclause_A.3.5">A.3.5</a>(62).<br /></li>
<MarginText>50/2</MarginText>
<li>Bounded string objects should not be implemented by implicit pointers and dynamic allocation. See <a href="../AA-A/AA-A.4#Subclause_A.4.4">A.4.4</a>(106).<br /></li>
<MarginText>51/2</MarginText>
<li>Strings.Hash should be good a hash function, returning a wide spread of values for different string values, and similar strings should rarely return the same value. See <a href="../AA-A/AA-A.4#Subclause_A.4.9">A.4.9</a>(12/2).<br /></li>
<MarginText>51.1/3</MarginText>
<li>If an implementation supports other string encoding schemes, a child of Ada.Strings similar to UTF_Encoding should be defined. See <a href="../AA-A/AA-A.4#Subclause_A.4.11">A.4.11</a>(107/3).<br /></li>
<MarginText>51.2/5</MarginText>
<li>Bounded buffer objects should be implemented without dynamic allocation. See <a href="../AA-A/AA-A.4#Subclause_A.4.12">A.4.12</a>(36).<br /></li>
<MarginText>52/2</MarginText>
<li>Any storage associated with an object of type Generator of the random number packages should be reclaimed on exit from the scope of the object. See <a href="../AA-A/AA-A.5#Subclause_A.5.2">A.5.2</a>(46).<br /></li>
<MarginText>53/2</MarginText>
<li>Each value of Initiator passed to Reset for the random number packages should initiate a distinct sequence of random numbers, or, if that is not possible, be at least a rapidly varying function of the initiator value. See <a href="../AA-A/AA-A.5#Subclause_A.5.2">A.5.2</a>(47).<br /></li>
<MarginText>54/2</MarginText>
<li>Get_Immediate should be implemented with unbuffered input; input should be available immediately; line-editing should be disabled. See <a href="../AA-A/AA-A.10#Subclause_A.10.7">A.10.7</a>(23).<br /></li>
<MarginText>55/2</MarginText>
<li>Package Directories.Information should be provided to retrieve other information about a file. See <a href="../AA-A/AA-A.16">A.16</a>(124/2).<br /></li>
<MarginText>56/3</MarginText>
<li>Directories.Start_Search and Directories.Search should raise Name_Error for malformed patterns. See <a href="../AA-A/AA-A.16">A.16</a>(125).<br /></li>
<MarginText>57/2</MarginText>
<li>Directories.Rename should be supported at least when both New_Name and Old_Name are simple names and New_Name does not identify an existing external file. See <a href="../AA-A/AA-A.16">A.16</a>(126/2).<br /></li>
<MarginText>57.1/3</MarginText>
<li>Directories.Hierarchical_File_Names should be provided for systems with hierarchical file naming, and should not be provided on other systems. See <a href="../AA-A/AA-A.16#Subclause_A.16.1">A.16.1</a>(36/3).<br /></li>
<MarginText>58/2</MarginText>
<li>If the execution environment supports subprocesses, the current environment variables should be used to initialize the environment variables of a subprocess. See <a href="../AA-A/AA-A.17">A.17</a>(32/2).<br /></li>
<MarginText>59/2</MarginText>
<li>Changes to the environment variables made outside the control of Environment_Variables should be reflected immediately. See <a href="../AA-A/AA-A.17">A.17</a>(33/2).<br /></li>
<MarginText>60/2</MarginText>
<li>Containers.Hash_Type'Modulus should be at least 2**32. Containers.Count_Type'Last should be at least 2**31&ndash;1. See <a href="../AA-A/AA-A.18#Subclause_A.18.1">A.18.1</a>(8/2).<br /></li>
<MarginText>61/2</MarginText>
<li>The worst-case time complexity of Element for Containers.Vector should be <em>O</em>(log <em>N</em>). See <a href="../AA-A/AA-A.18#Subclause_A.18.2">A.18.2</a>(256/2).<br /></li>
<MarginText>62/2</MarginText>
<li>The worst-case time complexity of Append with Count = 1 when <em>N</em> is less than the capacity for Containers.Vector should be <em>O</em>(log <em>N</em>). See <a href="../AA-A/AA-A.18#Subclause_A.18.2">A.18.2</a>(257/2).<br /></li>
<MarginText>63/2</MarginText>
<li>The worst-case time complexity of Prepend with Count = 1 and Delete_First with Count=1 for Containers.Vectors should be <em>O</em>(<em>N</em> log <em>N</em>). See <a href="../AA-A/AA-A.18#Subclause_A.18.2">A.18.2</a>(258/2).<br /></li>
<MarginText>64/2</MarginText>
<li>The worst-case time complexity of a call on procedure Sort of an instance of Containers.Vectors.Generic_Sorting should be <em>O</em>(<em>N</em>**2), and the average time complexity should be better than <em>O</em>(<em>N</em>**2). See <a href="../AA-A/AA-A.18#Subclause_A.18.2">A.18.2</a>(259/2).<br /></li>
<MarginText>65/2</MarginText>
<li>Containers.Vectors.Generic_Sorting.Sort and Containers.Vectors.Generic_Sorting.Merge should minimize copying of elements. See <a href="../AA-A/AA-A.18#Subclause_A.18.2">A.18.2</a>(260/2).<br /></li>
<MarginText>66/2</MarginText>
<li>Containers.Vectors.Move should not copy elements, and should minimize copying of internal data structures. See <a href="../AA-A/AA-A.18#Subclause_A.18.2">A.18.2</a>(261/2).<br /></li>
<MarginText>67/2</MarginText>
<li>If an exception is propagated from a vector operation, no storage should be lost, nor any elements removed from a vector unless specified by the operation. See <a href="../AA-A/AA-A.18#Subclause_A.18.2">A.18.2</a>(262/2).<br /></li>
<MarginText>68/2</MarginText>
<li>The worst-case time complexity of Element, Insert with Count=1, and Delete with Count=1 for Containers.Doubly_Linked_Lists should be <em>O</em>(log <em>N</em>). See <a href="../AA-A/AA-A.18#Subclause_A.18.3">A.18.3</a>(160/2).<br /></li>
<MarginText>69/2</MarginText>
<li>A call on procedure Sort of an instance of Containers.Doubly_Linked_Lists.Generic_Sorting should have an average time complexity better than <em>O</em>(<em>N</em>**2) and worst case no worse than <em>O</em>(<em>N</em>**2). See <a href="../AA-A/AA-A.18#Subclause_A.18.3">A.18.3</a>(161/2).<br /></li>
<MarginText>70/2</MarginText>
<li>Containers.Doubly_Linked_Lists.Move should not copy elements, and should minimize copying of internal data structures. See <a href="../AA-A/AA-A.18#Subclause_A.18.3">A.18.3</a>(162/2).<br /></li>
<MarginText>71/2</MarginText>
<li>If an exception is propagated from a list operation, no storage should be lost, nor any elements removed from a list unless specified by the operation. See <a href="../AA-A/AA-A.18#Subclause_A.18.3">A.18.3</a>(163/2).<br /></li>
<MarginText>72/2</MarginText>
<li>Move for a map should not copy elements, and should minimize copying of internal data structures. See <a href="../AA-A/AA-A.18#Subclause_A.18.4">A.18.4</a>(83/2).<br /></li>
<MarginText>73/2</MarginText>
<li>If an exception is propagated from a map operation, no storage should be lost, nor any elements removed from a map unless specified by the operation. See <a href="../AA-A/AA-A.18#Subclause_A.18.4">A.18.4</a>(84/2).<br /></li>
<MarginText>74/2</MarginText>
<li>The average time complexity of Element, Insert, Include, Replace, Delete, Exclude, and Find operations that take a key parameter for Containers.Hashed_Maps should be <em>O</em>(log <em>N</em>). The average time complexity of the subprograms of Containers.Hashed_Maps that take a cursor parameter should be <em>O</em>(1). The average time complexity of Containers.Hashed_Maps.Reserve_Capacity should be <em>O</em>(<em>N</em>). See <a href="../AA-A/AA-A.18#Subclause_A.18.5">A.18.5</a>(62/2).<br /></li>
<MarginText>75/2</MarginText>
<li>The worst-case time complexity of Element, Insert, Include, Replace, Delete, Exclude, and Find operations that take a key parameter for Containers.Ordered_Maps should be <em>O</em>((log <em>N</em>)**2) or better. The worst-case time complexity of the subprograms of Containers.Ordered_Maps that take a cursor parameter should be <em>O</em>(1). See <a href="../AA-A/AA-A.18#Subclause_A.18.6">A.18.6</a>(95/2).<br /></li>
<MarginText>76/2</MarginText>
<li>Move for sets should not copy elements, and should minimize copying of internal data structures. See <a href="../AA-A/AA-A.18#Subclause_A.18.7">A.18.7</a>(104/2).<br /></li>
<MarginText>77/2</MarginText>
<li>If an exception is propagated from a set operation, no storage should be lost, nor any elements removed from a set unless specified by the operation. See <a href="../AA-A/AA-A.18#Subclause_A.18.7">A.18.7</a>(105/2).<br /></li>
<MarginText>78/2</MarginText>
