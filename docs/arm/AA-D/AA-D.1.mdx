#### Dynamic Semantics

<MarginText>14/3</MarginText>
<MarginInfo items={["AI05-0229-1"]} />
<p>The Priority aspect has no effect if it is specified for a subprogram other than the main subprogram; the Priority value is not associated with any task.<br /></p>

<MarginText>15/5</MarginText>
<MarginInfo items={["AI12-0404-1"]} />
<p>A <em>task priority</em> is an integer value that indicates a degree of urgency and is the basis for resolving competing demands of tasks for resources. Unless otherwise specified, whenever tasks compete for processors or other implementation-defined resources, the resources are allocated to the task with the highest priority value. The <em>base priority</em> of a task is the priority with which it was created, or to which it was later set by Dynamic{"_"}Priorities.Set{"_"}Priority (see <a href="../AA-D/AA-D.5">D.5</a>). At all times, a task also has an <em>active priority</em>, which generally is its base priority unless it inherits a priority from other sources. <em>Priority inheritance</em> is the process by which the priority of a task or other entity (for example, a protected object; see <a href="../AA-D/AA-D.3">D.3</a>) is used in the evaluation of another task's active priority. <br /></p>

<AnnotatedOnly>
<MarginText>15.a</MarginText>
<Admonition type="aarm" aarm="implementation-defined">
<strong></strong>Implementation-defined execution resources.<br />
</Admonition>
</AnnotatedOnly>

<MarginText>16/3</MarginText>
<MarginInfo items={["AI05-0229-1"]} />
<p>The effect of specifying a Priority or Interrupt{"_"}Priority aspect for a protected type or <code><a href="../AA-9/AA-9.4#S0250">single_protected_declaration</a></code> is discussed in <a href="../AA-D/AA-D.3">D.3</a>.<br /></p>

<MarginText>17/4</MarginText>
<MarginInfo items={["AI05-0229-1", "AI12-0081-1"]} />
<p>The <code><a href="../AA-4/AA-4.4#S0132">expression</a></code> specified for the Priority or Interrupt{"_"}Priority aspect of a task type is evaluated each time an object of the task type is created (see <a href="../AA-9/AA-9.1">9.1</a>). For the Priority aspect, the value of the <code><a href="../AA-4/AA-4.4#S0132">expression</a></code> is converted to the subtype Priority; for the Interrupt{"_"}Priority aspect, this value is converted to the subtype Any{"_"}Priority. The priority value is then associated with the task object. <br /></p>

<MarginText>18/3</MarginText>
<MarginInfo items={["AI05-0229-1"]} />
<p>Likewise, the priority value is associated with the environment task if the aspect is specified for the main subprogram.<br /></p>

<MarginText>19/3</MarginText>
<MarginInfo items={["AI05-0229-1"]} />
<p>The initial value of a task's base priority is specified by default or by means of a Priority or Interrupt{"_"}Priority aspect. [After a task is created, its base priority can be changed only by a call to Dynamic{"_"}Priorities.Set{"_"}Priority (see <a href="../AA-D/AA-D.5">D.5</a>).] The initial base priority of a task in the absence of an aspect is the base priority of the task that creates it at the time of creation (see <a href="../AA-9/AA-9.1">9.1</a>). If the aspect Priority is not specified for the main subprogram, the initial base priority of the environment task is System.Default{"_"}Priority. [The task's active priority is used when the task competes for processors. Similarly, the task's active priority is used to determine the task's position in any queue when Priority{"_"}Queuing is specified (see <a href="../AA-D/AA-D.4">D.4</a>).]<br /></p>

<MarginText>20/2</MarginText>
<MarginInfo items={["AI95-00357-01"]} />
<p>At any time, the active priority of a task is the maximum of all the priorities the task is inheriting at that instant. For a task that is not held (see <a href="../AA-D/AA-D.11">D.11</a>), its base priority is a source of priority inheritance unless otherwise specified for a particular task dispatching policy. Other sources of priority inheritance are specified under the following conditions: <br /></p>

<AnnotatedOnly>
<MarginText>20.a</MarginText>
<Admonition type="aarm" aarm="discussion">
<strong></strong>Other parts of the annex, e.g. <a href="../AA-D/AA-D.11">D.11</a>, define other sources of priority inheritance. <br />
</Admonition>
</AnnotatedOnly>

<MarginText>21/1</MarginText>
<MarginInfo items={["AI95-00092-01"]} />
<ul>
<li>{"{"}<em>8652/0072</em>{"}"} During activation, a task being activated inherits the active priority that its activator (see <a href="../AA-9/AA-9.2">9.2</a>) had at the time the activation was initiated.<br /></li>
<MarginText>22/1</MarginText>
<MarginInfo items={["AI95-00092-01"]} />
<li>{"{"}<em>8652/0072</em>{"}"} During rendezvous, the task accepting the entry call inherits the priority of the entry call (see <a href="../AA-9/AA-9.5#Subclause_9.5.3">9.5.3</a> and <a href="../AA-D/AA-D.4">D.4</a>).<br /></li>
<MarginText>22.1/5</MarginText>
<MarginInfo items={["AI12-0276-1"]} />
<li>While starting a protected action on a protected object when the FIFO{"_"}Spinning admission policy is in effect, a task inherits the ceiling priority of the protected object (see <a href="../AA-9/AA-9.5">9.5</a>, <a href="../AA-D/AA-D.3">D.3</a>, and <a href="../AA-D/AA-D.4#Subclause_D.4.1">D.4.1</a>).<br /></li>
</ul>
<AnnotatedOnly>
<MarginText>22.a/5</MarginText>
<MarginInfo items={["AI12-0005-1", "AI12-0276-1"]} />
<Admonition type="aarm" aarm="reason">
<strong></strong>Priority inheritance is needed for FIFO{"_"}Spinning to ensure that lower priority tasks that initiate spin waiting earlier than other higher priority tasks continue to spin to ensure that they can be granted the resource when it becomes available in order to support FIFO ordering. Note that this rule only matters when tasks that can initiate a protected action on an object P can be on a different processor than P. In particular, this rule does not matter on a monoprocessor. <br />
</Admonition>
</AnnotatedOnly>

<MarginText>23/5</MarginText>
<MarginInfo items={["AI12-0404-1"]} />
<ul>
<li>While a task executes a protected action on a protected object, the task inherits the ceiling priority of the protected object (see <a href="../AA-9/AA-9.5">9.5</a> and <a href="../AA-D/AA-D.3">D.3</a>).<br /></li>
</ul>
<MarginText>24</MarginText>
<p>In all of these cases, the priority ceases to be inherited as soon as the condition calling for the inheritance no longer exists.<br /></p>


#### Implementation Requirements

<MarginText>25</MarginText>
<p>The range of System.Interrupt{"_"}Priority shall include at least one value.<br /></p>

<MarginText>26</MarginText>
<p>The range of System.Priority shall include at least 30 values.<br /></p>

<AnnotatedOnly>
<MarginText>27</MarginText>
<Admonition type="aarm" aarm="note">
NOTE 1   The priority expression can include references to discriminants of the enclosing type.<br />
</Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>28</MarginText>
<Admonition type="aarm" aarm="note">
NOTE 2   It is a consequence of the active priority rules that at the point when a task stops inheriting a priority from another source, its active priority is re-evaluated. This is in addition to other instances described in this Annex for such re-evaluation.<br />
</Admonition>
</AnnotatedOnly>

<MarginText>29/5</MarginText>

</Admonition>
