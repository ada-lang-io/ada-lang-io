<MarginText>1/4</MarginText>
<MarginInfo items={["AI12-0072-1"]} />
<p>[Specifying the aspect Discard_Names can be used to request a reduction in storage used for the names of entities with runtime name text.] <br /></p>


#### Static Semantics

<MarginText>1.1/4</MarginText>
<MarginInfo items={["AI12-0072-1"]} />
<p>An entity with <em>runtime name text</em> is a nonderived enumeration first subtype, a tagged first subtype, or an exception.<br /></p>

<MarginText>1.2/4</MarginText>
<MarginInfo items={["AI12-0072-1"]} />
<p>For an entity with runtime name text, the following language-defined representation aspect may be specified:<br /></p>

<MarginText>1.3/4</MarginText>
<dt><br/>Discard_Names</dt>
<dl>
<dd>The type of aspect Discard_Names is Boolean. If directly specified, the <code><a href="../AA-13/AA-13.1#S0348">aspect_definition</a></code> shall be a static expression. If not specified (including by inheritance), the aspect is False.<br /></dd>
</dl>
<AnnotatedOnly>
<MarginText>1.a.1/4</MarginText>
<Admonition type="aarm" aarm="note">
<strong>Aspect Description for </strong><strong>Discard_Names: </strong>Requests a reduction in storage for names associated with an entity.<br /></Admonition>
</AnnotatedOnly>


#### Syntax

<MarginText>2</MarginText>
<p>The form of a <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code> Discard_Names is as follows: <br /></p>

<MarginText>3</MarginText>
<p>  <strong>pragma</strong> Discard_Names[([On ={">"} ] <code><a href="../AA-13/AA-13.1#S0345">local_name</a></code>)];<br /></p>

<MarginText>4</MarginText>
<p>A <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code> Discard_Names is allowed only immediately within a <code><a href="../AA-3/AA-3.11#S0086">declarative_part</a></code>, immediately within a <code><a href="../AA-7/AA-7.1#S0230">package_specification</a></code>, or as a configuration pragma. <br /></p>


#### Legality Rules

<MarginText>5/4</MarginText>
<MarginInfo items={["AI12-0072-1"]} />
<p>The <code><a href="../AA-13/AA-13.1#S0345">local_name</a></code> (if present) shall denote an entity with runtime name text. The pragma specifies that the aspect Discard_Names for the type or exception has the value True. Without a <code><a href="../AA-13/AA-13.1#S0345">local_name</a></code>, the pragma specifies that all entities with runtime name text declared after the pragma, within the same declarative region have the value True for aspect Discard_Names. Alternatively, the pragma can be used as a configuration pragma. If the configuration pragma Discard_Names applies to a compilation unit, all entities with runtime name text declared in the compilation unit have the value True for the aspect Discard_Names.<br /></p>

<AnnotatedOnly>
<MarginText>5.a/4</MarginText>
<MarginInfo items={["AI12-0072-1"]} />
<Admonition type="aarm" aarm="ramification">
<strong></strong>If the aspect is specified for a type, then it is inherited by all descendants of the type. The aspect cannot be specified as False on a derived type (because specifying the aspect is not allowed on derived enumeration types, and by rule applying to all aspects for other types (see <a href="../AA-13/AA-13.1#Subclause_13.1.1">13.1.1</a>)). <br /></Admonition>
</AnnotatedOnly>


#### Static Semantics

<MarginText>6</MarginText>
<p>If a <code><a href="../AA-13/AA-13.1#S0345">local_name</a></code> is given, then a <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code> Discard_Names is a representation pragma.<br /></p>

<MarginText>7/4</MarginText>
<MarginInfo items={["AI95-00285-01", "AI95-00400-01", "AI12-0072-1"]} />
<p>If the aspect Discard_Names is True for an enumeration type, then the semantics of the Wide_Wide_Image and Wide_Wide_Value attributes are implementation defined for that type[; the semantics of Image, Wide_Image, Value, and Wide_Value are still defined in terms of Wide_Wide_Image and Wide_Wide_Value]. In addition, the semantics of Text_IO.Enumeration_IO are implementation defined. If the aspect Discard_Names is True for a tagged type, then the semantics of the Tags.Wide_Wide_Expanded_Name function are implementation defined for that type[; the semantics of Tags.Expanded_Name and Tags.Wide_Expanded_Name are still defined in terms of Tags.Wide_Wide_Expanded_Name]. If the aspect Discard_Names is True for an exception, then the semantics of the Exceptions.Wide_Wide_Exception_Name function are implementation defined for that exception[; the semantics of Exceptions.Exception_Name and Exceptions.Wide_Exception_Name are still defined in terms of Exceptions.Wide_Wide_Exception_Name].<br /></p>

<AnnotatedOnly>
<MarginText>7.a/4</MarginText>
<Admonition type="aarm" aarm="implementation-defined">
<strong></strong>The semantics of some attributes and functions of an entity for which aspect Discard_Names is True.<br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>7.b</MarginText>
<Admonition type="aarm" aarm="ramification">
<strong></strong>The Width attribute is still defined in terms of Image.<br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>7.c/4</MarginText>
<MarginInfo items={["AI95-00285-01", "AI12-0072-1"]} />
<Admonition type="aarm" aarm="note">
The semantics of S'Wide_Wide_Image and S'Wide_Wide_Value are implementation defined for any subtype of an enumeration type for which the aspect is True. (The pragma, if used, actually names the first subtype, of course.) <br /></Admonition>
</AnnotatedOnly>


#### Implementation Advice

<MarginText>8/4</MarginText>
<MarginInfo items={["AI12-0072-1"]} />
<p>If the aspect Discard_Names is True for an entity, then the implementation should reduce the amount of storage used for storing names associated with that entity. <br /></p>

<AnnotatedOnly>
<MarginText>8.a/4</MarginText>
<Admonition type="aarm" aarm="implementation-advice">
<strong></strong>If aspect Discard_Names is True for an entity, then the amount of storage used for storing names associated with that entity should be reduced.<br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>8.b</MarginText>
<Admonition type="aarm" aarm="reason">
<strong></strong>A typical implementation of the Image attribute for enumeration types is to store a table containing the names of all the enumeration literals. Aspect Discard_Names allows the implementation to avoid storing such a table without having to prove that the Image attribute is never used (which can be difficult in the presence of separate compilation).<br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>8.c</MarginText>
<Admonition type="aarm" aarm="note">
We did not specify the semantics of the Image attribute when aspect Discard_Names is True because different semantics might be desirable in different situations. In some cases, it might make sense to use the Image attribute to print out a useful value that can be used to identify the entity given information in compiler-generated listings. In other cases, it might make sense to get an error at compile time or at run time. In cases where memory is plentiful, the simplest implementation makes sense: ignore the aspect. Implementations that are capable of avoiding the extra storage in cases where the Image attribute is never used might also wish to ignore the aspect.<br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>8.d</MarginText>
<Admonition type="aarm" aarm="note">
The same applies to the Tags.Expanded_Name and Exceptions.Exception_Name functions. <br /></Admonition>
</AnnotatedOnly>
