<p>[This subclause specifies rules regarding access to machine instructions from within an Ada program.] <br /></p>

<MarginText>1.a/2</MarginText>
<Admonition type="aarm" aarm="implementation-defined">
<strong></strong>Implementation-defined intrinsic subprograms.<br />
</Admonition>
</AnnotatedOnly>


#### Implementation Requirements

<MarginText>2/5</MarginText>
<MarginInfo items={["AI12-0320-1"]} />
<p>The implementation shall support machine code insertions (see <a href="/docs/arm/AA-13/AA-13.8">13.8</a>) or intrinsic subprograms (see <a href="/docs/arm/AA-6/AA-6.3#Subclause_6.3.1">6.3.1</a>) (or both). The implementation shall allow the use of Ada entities as operands for such machine code insertions or intrinsic subprograms.<br /></p>

<AnnotatedOnly>
<MarginText>2.a/5</MarginText>
<MarginInfo items={["AI12-0320-1"]} />
<Admonition type="aarm" aarm="discussion">
<strong></strong>Ada entities could be used as operands in machine code insertions and/or intrinsic subprograms using language-defined attributes (such as address), implementation-defined attributes, or could be directly allowed. <br />
</Admonition>
</AnnotatedOnly>


#### Implementation Advice

<MarginText>3</MarginText>
<p>The machine code or intrinsics support should allow access to all operations normally available to assembly language programmers for the target environment, including privileged instructions, if any. <br /></p>

<AnnotatedOnly>
<MarginText>3.a.1/2</MarginText>
<Admonition type="aarm" aarm="implementation-advice">
<strong></strong>The machine code or intrinsics support should allow access to all operations normally available to assembly language programmers for the target environment.<br />
</Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>3.a</MarginText>
<Admonition type="aarm" aarm="ramification">
<strong></strong>Of course, on a machine with protection, an attempt to execute a privileged instruction in user mode will probably trap. Nonetheless, we want implementations to provide access to them so that Ada can be used to write systems programs that run in privileged mode. <br />
</Admonition>
</AnnotatedOnly>

<MarginText>4/3</MarginText>
<MarginInfo items={["AI05-0229-1"]} />
<p>The support for interfacing aspects (see <a href="/docs/arm/AA-B/">Annex B</a>) should include interface to assembler; the default assembler should be associated with the convention identifier Assembler. <br /></p>

<AnnotatedOnly>
<MarginText>4.a/2</MarginText>
<Admonition type="aarm" aarm="implementation-advice">
<strong></strong>Interface to assembler should be supported; the default assembler should be associated with the convention identifier Assembler.<br />
</Admonition>
</AnnotatedOnly>

<MarginText>5</MarginText>
<p>If an entity is exported to assembly language, then the implementation should allocate it at an addressable location, and should ensure that it is retained by the linking process, even if not otherwise referenced from the Ada code. The implementation should assume that any call to a machine code or assembler subprogram is allowed to read or update every object that is specified as exported. <br /></p>

<AnnotatedOnly>
<MarginText>5.a/5</MarginText>
<Admonition type="aarm" aarm="implementation-advice">
<strong></strong>If an entity is exported to assembly language, then the implementation should allocate it at an addressable location even if not otherwise referenced from the Ada code. A call to a machine code or assembler subprogram should be treated as if it can read or update every object that is specified as exported.<br />
</Admonition>
</AnnotatedOnly>


#### Documentation Requirements

<MarginText>6</MarginText>
<p>The implementation shall document the overhead associated with calling machine-code or intrinsic subprograms, as compared to a fully-inlined call, and to a regular out-of-line call. <br /></p>

<AnnotatedOnly>
<MarginText>6.a/2</MarginText>
<Admonition type="aarm" aarm="note">
<strong>Documentation Requirement: </strong>The overhead of calling machine-code or intrinsic subprograms.<br />
</Admonition>
</AnnotatedOnly>

<MarginText>7</MarginText>
<p>The implementation shall document the types of the package System.Machine{"_"}Code usable for machine code insertions, and the attributes to be used in machine code insertions for references to Ada entities. <br /></p>

<AnnotatedOnly>
<MarginText>7.a/2</MarginText>
<Admonition type="aarm" aarm="note">
<strong>Documentation Requirement: </strong>The types and attributes used in machine code insertions.<br />
</Admonition>
</AnnotatedOnly>

<MarginText>8/3</MarginText>
<MarginInfo items={["AI05-0229-1"]} />
<p>The implementation shall document the subprogram calling conventions associated with the convention identifiers available for use with the Convention aspect (Ada and Assembler, at a minimum), including register saving, exception propagation, parameter passing, and function value returning. <br /></p>

<AnnotatedOnly>
<MarginText>8.a/2</MarginText>
<Admonition type="aarm" aarm="note">
<strong>Documentation Requirement: </strong>The subprogram calling conventions for all supported convention identifiers.<br />
</Admonition>
</AnnotatedOnly>

<MarginText>9</MarginText>
<p>For exported and imported subprograms, the implementation shall document the mapping between the Link{"_"}Name string, if specified, or the Ada designator, if not, and the external link name used for such a subprogram. <br /></p>

#### Implementation Advice

<MarginText>10</MarginText>
<p>The implementation should ensure that little or no overhead is associated with calling intrinsic and machine-code subprograms. <br /></p>

<AnnotatedOnly>
<MarginText>10.a/2</MarginText>
<Admonition type="aarm" aarm="implementation-advice">
<strong></strong>Little or no overhead should be associated with calling intrinsic and machine-code subprograms.<br />
</Admonition>
</AnnotatedOnly>

<MarginText>11</MarginText>
<p>It is recommended that intrinsic subprograms be provided for convenient access to any machine operations that provide special capabilities or efficiency and that are not otherwise available through the language constructs. Examples of such instructions include: <br /></p>

<MarginText>12/5</MarginText>
<MarginInfo items={["AI12-0453-1"]} />
<ul>
<li>Atomic read-modify-write operations &mdash; for example, test and set, compare and swap, decrement and test, enqueue/dequeue.<br /></li>
<MarginText>13/5</MarginText>
<MarginInfo items={["AI12-0453-1"]} />
<li>Standard numeric functions &mdash; for example, <em>sin</em>, <em>log</em>.<br /></li>
<MarginText>14/5</MarginText>
<MarginInfo items={["AI12-0453-1"]} />
<li>String manipulation operations &mdash; for example, translate and test.<br /></li>
<MarginText>15/5</MarginText>
<MarginInfo items={["AI12-0453-1"]} />
<li>Vector operations &mdash; for example, compare vector against thresholds.<br /></li>
<MarginText>16</MarginText>
<li>Direct operations on I/O ports.<br /></li>
</ul>
<AnnotatedOnly>
<MarginText>16.a/2</MarginText>
<Admonition type="aarm" aarm="implementation-advice">
<strong></strong>Intrinsic subprograms should be provided to access any machine operations that provide special capabilities or efficiency not normally available.<br />
</Admonition>
</AnnotatedOnly>
