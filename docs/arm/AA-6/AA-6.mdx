<MarginText>1</MarginText>
<p>A subprogram is a program unit or intrinsic operation whose execution is invoked by a subprogram call. There are two forms of subprogram: procedures and functions. A procedure call is a <code><a href="../AA-5/AA-5.1#S0167">statement</a></code>; a function call is an expression and returns a value. The definition of a subprogram can be given in two parts: a subprogram declaration defining its interface, and a <code><a href="../AA-6/AA-6.3#S0216">subprogram_body</a></code> defining its execution. [Operators and enumeration literals are functions.] <br /></p>

<AnnotatedOnly>
<MarginText>1.a</MarginText>
<Admonition type="aarm" aarm="note">
<strong>To be honest: </strong>A function call is an expression, but more specifically it is a <code><a href="../AA-4/AA-4.1#S0091">name</a></code>. <br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>1.b/5</MarginText>
<Admonition type="aarm" aarm="note">
<strong>Term entry: </strong><strong>subprogram</strong> &mdash; unit of a program that can be brought into execution in various contexts, with the invocation being a subprogram call that can parameterize the effect of the subprogram through the passing of operands<br /><br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>1.c/5</MarginText>
<Admonition type="aarm" aarm="note">
<strong>Term entry: </strong><strong>function</strong> &mdash; form of subprogram that returns a result and can be called as part of an expression<br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>1.d/5</MarginText>
<Admonition type="aarm" aarm="note">
<strong>Term entry: </strong><strong>procedure</strong> &mdash; form of subprogram that does not return a result and can only be invoked by a statement<br /></Admonition>
</AnnotatedOnly>

<MarginText>2/3</MarginText>
<MarginInfo items={["AI05-0299-1"]} />
<p>A <em>callable entity</em> is a subprogram or entry (see Section 9). A callable entity is invoked by a <em>call</em>; that is, a subprogram call or entry call. A <em>callable construct</em> is a construct that defines the action of a call upon a callable entity: a <code><a href="../AA-6/AA-6.3#S0216">subprogram_body</a></code>, <code><a href="../AA-9/AA-9.5#S0260">entry_body</a></code>, or <code><a href="../AA-9/AA-9.5#S0258">accept_statement</a></code>. <br /></p>

<AnnotatedOnly>
<MarginText>2.a</MarginText>
<Admonition type="aarm" aarm="ramification">
<strong></strong>Note that &ldquo;callable entity&rdquo; includes predefined operators, enumeration literals, and abstract subprograms. &ldquo;Call&rdquo; includes calls of these things. They do not have callable constructs, since they don't have completions. <br /></Admonition>
</AnnotatedOnly>
