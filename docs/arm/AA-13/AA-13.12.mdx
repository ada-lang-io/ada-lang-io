<MarginText>5</MarginText>

<MarginText>6</MarginText>
<p>Unless otherwise specified for a particular restriction, the <code><a href="../AA-4/AA-4.4#S0132">expression</a></code> shall be static, and its value shall be nonnegative. <br /></p>

#### Post-Compilation Rules

<MarginText>8/3</MarginText>
<MarginInfo items={["AI05-0013-1"]} />
<p>A <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code> Restrictions is a configuration pragma. If a <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code> Restrictions applies to any compilation unit included in the partition, this may impose either (or both) of two kinds of requirements, as specified for the particular restriction:<br /></p>

<MarginText>8.1/3</MarginText>
<MarginInfo items={["AI05-0013-1"]} />
<ul>
<li>A restriction may impose requirements on some or all of the units comprising the partition. Unless otherwise specified for a particular restriction, such a requirement applies to all of the units comprising the partition and is enforced via a post-compilation check.<br /></li>
<MarginText>8.2/3</MarginText>
<MarginInfo items={["AI05-0013-1"]} />
<li>A restriction may impose requirements on the run-time behavior of the program, as indicated by the specification of run-time behavior associated with a violation of the requirement.<br /></li>
</ul>
<AnnotatedOnly>
<MarginText>8.a.1/3</MarginText>
<Admonition type="aarm" aarm="ramification">
<strong></strong>In this latter case, there is no post-compilation check needed for the requirement. <br /></Admonition>
</AnnotatedOnly>

<MarginText>8.3/1</MarginText>
<MarginInfo items={["AI95-00130-01"]} />
<p>{"{"}<em>8652/0042</em>{"}"} For the purpose of checking whether a partition contains constructs that violate any restriction (unless specified otherwise for a particular restriction):<br /></p>

<MarginText>8.4/1</MarginText>
<MarginInfo items={["AI95-00130-01"]} />
<ul>
<li>{"{"}<em>8652/0042</em>{"}"} Generic instances are logically expanded at the point of instantiation;<br /></li>
<MarginText>8.5/1</MarginText>
<MarginInfo items={["AI95-00130-01"]} />
<li>{"{"}<em>8652/0042</em>{"}"} If an object of a type is declared or allocated and not explicitly initialized, then all expressions appearing in the definition for the type and any of its ancestors are presumed to be used;<br /></li>
<MarginText>8.6/1</MarginText>
<MarginInfo items={["AI95-00130-01"]} />
<li>{"{"}<em>8652/0042</em>{"}"} A <code><a href="../AA-3/AA-3.7#S0063">default_expression</a></code> for a formal parameter or a generic formal object is considered to be used if and only if the corresponding actual parameter is not provided in a given call or instantiation. <br /></li>
</ul>

#### Implementation Permissions

<MarginText>8.7/3</MarginText>
<MarginInfo items={["AI05-0269-1"]} />
<p>An implementation may provide implementation-defined restrictions; the identifier for an implementation-defined restriction shall differ from those of the language-defined restrictions. <br /></p>

<AnnotatedOnly>
<MarginText>8.a.2/3</MarginText>
<Admonition type="aarm" aarm="implementation-defined">
<strong></strong>Implementation-defined restrictions allowed in a <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code> Restrictions.<br /></Admonition>
</AnnotatedOnly>

<MarginText>9</MarginText>
<p>An implementation may place limitations on the values of the <code><a href="../AA-4/AA-4.4#S0132">expression</a></code> that are supported, and limitations on the supported combinations of restrictions. The consequences of violating such limitations are implementation defined. <br /></p>

<AnnotatedOnly>
<MarginText>9.a</MarginText>
<Admonition type="aarm" aarm="implementation-defined">
<strong></strong>The consequences of violating limitations on Restrictions <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code>s.<br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>9.b</MarginText>
<Admonition type="aarm" aarm="ramification">
<strong></strong>Such limitations may be enforced at compile time or at run time. Alternatively, the implementation is allowed to declare violations of the restrictions to be erroneous, and not enforce them at all. <br /></Admonition>
</AnnotatedOnly>

<MarginText>9.1/1</MarginText>
<MarginInfo items={["AI95-00130-01"]} />
<p>{"{"}<em>8652/0042</em>{"}"} An implementation is permitted to omit restriction checks for code that is recognized at compile time to be unreachable and for which no code is generated.<br /></p>

<MarginText>9.2/1</MarginText>
<MarginInfo items={["AI95-00190-01"]} />
<p>{"{"}<em>8652/0043</em>{"}"} Whenever enforcement of a restriction is not required prior to execution, an implementation may nevertheless enforce the restriction prior to execution of a partition to which the restriction applies, provided that every execution of the partition would violate the restriction. <br /></p>


#### Syntax

<MarginText>10/3</MarginText>
<MarginInfo items={["AI95-00249-01", "AI05-0246-1"]} />
<p class="Indented2">The form of a <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code> Profile is as follows: <br /></p>

<MarginText>11/3</MarginText>
<p class="Indented2">  <strong>pragma</strong> Profile (<em>profile_</em><code><a href="../AA-2/AA-2.3#S0002">identifier</a></code> {"{"}, <em>profile_</em><code><a href="../AA-2/AA-2.8#S0020">pragma_argument_association</a></code>{"}"});<br /></p>


#### Legality Rules

<MarginText>12/3</MarginText>
<MarginInfo items={["AI95-00249-01", "AI05-0246-1"]} />
<p>The <em>profile_</em><code><a href="../AA-2/AA-2.3#S0002">identifier</a></code> shall be the name of a usage profile. The semantics of any <em>profile_</em><code><a href="../AA-2/AA-2.8#S0020">pragma_argument_association</a></code>s are defined by the usage profile specified by the <em>profile_</em><code><a href="../AA-2/AA-2.3#S0002">identifier</a></code>. <br /></p>


#### Static Semantics

<MarginText>13/3</MarginText>
<MarginInfo items={["AI95-00249-01", "AI05-0246-1"]} />
<p>A profile is equivalent to the set of configuration pragmas that is defined for each usage profile. <br /></p>


#### Post-Compilation Rules

<MarginText>14/3</MarginText>
<MarginInfo items={["AI95-00249-01"]} />
<p>A <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code> Profile is a configuration pragma. There may be more than one <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code> Profile for a partition. <br /></p>


#### Implementation Permissions

<MarginText>15/3</MarginText>
<MarginInfo items={["AI05-0269-1"]} />
<p>An implementation may provide implementation-defined usage profiles; the identifier for an implementation-defined usage profile shall differ from those of the language-defined usage profiles.<br /></p>

<AnnotatedOnly>
<MarginText>15.a.1/3</MarginText>
<Admonition type="aarm" aarm="implementation-defined">
<strong></strong>Implementation-defined usage profiles allowed in a <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code> Profile.<br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>16/2</MarginText>
<MarginInfo items={["AI95-00347-01"]} />
<Admonition type="aarm" aarm="note">
NOTE 1   Restrictions intended to facilitate the construction of efficient tasking run-time systems are defined in <a href="../AA-D/AA-D.7">D.7</a>. Restrictions intended for use when constructing high integrity systems are defined in <a href="../AA-H/AA-H.4">H.4</a>.<br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>17</MarginText>
<Admonition type="aarm" aarm="note">
NOTE 2   An implementation has to enforce the restrictions in cases where enforcement is required, even if it chooses not to take advantage of the restrictions in terms of efficiency. <br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>17.a</MarginText>
<Admonition type="aarm" aarm="discussion">
<strong></strong>It is not the intent that an implementation will support a different run-time system for every possible combination of restrictions. An implementation might support only two run-time systems, and document a set of restrictions that is sufficient to allow use of the more efficient and safe one. <br /></Admonition>
</AnnotatedOnly>

## 13.12.1  Language-Defined Restrictions and Profiles


#### Static Semantics

<MarginText>1/2</MarginText>
<MarginInfo items={["AI95-00257-01"]} />
<p>The following <em>restriction_</em><code><a href="../AA-2/AA-2.3#S0002">identifier</a></code>s are language defined (additional restrictions are defined in the Specialized Needs Annexes):<br /></p>

<MarginText>1.1/3</MarginText>
<MarginInfo items={["AI05-0241-1"]} />
<dt><br/>No_Implementation_Aspect_Specifications </dt>
<dl>
<dd>There are no implementation-defined aspects specified by an <code><a href="../AA-13/AA-13.1#S0346">aspect_specification</a></code>. This restriction applies only to the current compilation or environment, not the entire partition. <br /></dd>
</dl>
<AnnotatedOnly>
<MarginText>1.a/3</MarginText>
<MarginInfo items={["AI05-0241-1"]} />
<Admonition type="aarm" aarm="discussion">
<strong></strong>This restriction (as well as others below) applies only to the current compilation, because it is likely that the runtime (and possibly user-written low-level code) will need to use implementation-defined aspects. But a partition-wide restriction applies everywhere, including the runtime. <br /></Admonition>
</AnnotatedOnly>

<MarginText>2/2</MarginText>
<MarginInfo items={["AI95-00257-01"]} />
<dt><br/>No_Implementation_Attributes </dt>
<dl>
<dd>There are no implementation-defined attributes. This restriction applies only to the current compilation or environment, not the entire partition. <br /></dd>
<MarginText>2.1/3</MarginText>
<MarginInfo items={["AI05-0246-1", "AI05-0269-1"]} />
<dt><br/>No_Implementation_Identifiers </dt>
<dd>There are no usage names that denote declarations with implementation-defined identifiers that occur within language-defined packages or instances of language-defined generic packages. Such identifiers can arise as follows: <br /></dd>
</dl>
<MarginText>2.2/3</MarginText>
<ul>
<li>The following language-defined packages and generic packages allow implementation-defined identifiers:<br /></li>
</ul>
<MarginText>2.3/3</MarginText>
<ul>
<li>package System (see <a href="../AA-13/AA-13.7">13.7</a>);<br /></li>
<MarginText>2.4/3</MarginText>
<li>package Standard (see <a href="../AA-A/AA-A.1">A.1</a>);<br /></li>
<MarginText>2.5/3</MarginText>
<li>package Ada.Command_Line (see <a href="../AA-A/AA-A.15">A.15</a>);<br /></li>
<MarginText>2.6/3</MarginText>
<li>package Interfaces.C (see <a href="../AA-B/AA-B.3">B.3</a>);<br /></li>
<MarginText>2.7/3</MarginText>
<li>package Interfaces.C.Strings (see <a href="../AA-B/AA-B.3#Subclause_B.3.1">B.3.1</a>);<br /></li>
<MarginText>2.8/3</MarginText>
<li>package Interfaces.C.Pointers (see <a href="../AA-B/AA-B.3#Subclause_B.3.2">B.3.2</a>);<br /></li>
<MarginText>2.9/3</MarginText>
<li>package Interfaces.COBOL (see <a href="../AA-B/AA-B.4">B.4</a>);<br /></li>
<MarginText>2.10/3</MarginText>
<li>package Interfaces.Fortran (see <a href="../AA-B/AA-B.5">B.5</a>); <br /></li>
</ul>
<MarginText>2.11/3</MarginText>
<ul>
<li>The following language-defined packages contain only implementation-defined identifiers:<br /></li>
</ul>
<MarginText>2.12/3</MarginText>
<ul>
<li>package System.Machine_Code (see <a href="../AA-13/AA-13.8">13.8</a>);<br /></li>
<MarginText>2.13/3</MarginText>
<li>package Ada.Directories.Information (see <a href="../AA-A/AA-A.16">A.16</a>);<br /></li>
<MarginText>2.14/3</MarginText>
<li>nested Implementation packages of the Queue containers (see <a href="../AA-A/AA-A.18#Subclause_A.18.28">A.18.28</a>-31);<br /></li>
<MarginText>2.15/3</MarginText>
<li>package Interfaces (see <a href="../AA-B/AA-B.2">B.2</a>);<br /></li>
<MarginText>2.16/3</MarginText>
<li>package Ada.Interrupts.Names (see <a href="../AA-C/AA-C.3#Subclause_C.3.2">C.3.2</a>).<br /></li>
</ul>
<MarginText>2.17/3</MarginText>
<dl>
<dd>For package Standard, Standard.Long_Integer and Standard.Long_Float are considered language-defined identifiers, but identifiers such as Standard.Short_Short_Integer are considered implementation-defined.<br /></dd>
<MarginText>2.18/3</MarginText>
<dd>This restriction applies only to the current compilation or environment, not the entire partition.<br /></dd>
<MarginText>3/2</MarginText>
<MarginInfo items={["AI95-00257-01"]} />
<dt><br/>No_Implementation_Pragmas </dt>
<dd>There are no implementation-defined pragmas or pragma arguments. This restriction applies only to the current compilation or environment, not the entire partition.<br /></dd>
<MarginText>3.1/3</MarginText>
<MarginInfo items={["AI05-0242-1"]} />
<dt><br/>No_Implementation_Units </dt>
<dd>There is no mention in the <code><a href="../AA-10/AA-10.1#S0292">context_clause</a></code> of any implementation-defined descendants of packages Ada, Interfaces, or System. This restriction applies only to the current compilation or environment, not the entire partition.<br /></dd>
<MarginText>4/3</MarginText>
<MarginInfo items={["AI95-00368-01", "AI05-0229-1"]} />
<dt><br/>No_Obsolescent_Features </dt>
<dd>There is no use of language features defined in Annex J. It is implementation defined whether uses of the renamings of <a href="../AA-J/AA-J.1">J.1</a> and of the <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code>s of <a href="../AA-J/AA-J.15">J.15</a> are detected by this restriction. This restriction applies only to the current compilation or environment, not the entire partition. <br /></dd>
</dl>
<AnnotatedOnly>
<MarginText>4.a/2</MarginText>
<Admonition type="aarm" aarm="reason">
<strong></strong>A user could compile a rename like <br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>4.b/2</MarginText>
<CodeBlock language="ada">
with Ada.Text_IO;{"\n"}package Text_IO renames Ada.Text_IO;{"\n"}

</CodeBlock>
</AnnotatedOnly>
<AnnotatedOnly>
<MarginText>4.c/2</MarginText>
<Admonition type="aarm" aarm="note">
Such a rename must not be disallowed by this restriction, nor should the compilation of such a rename be restricted by an implementation. Many implementations implement the renames of <a href="../AA-J/AA-J.1">J.1</a> by compiling them normally; we do not want to require implementations to use a special mechanism to implement these renames.<br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>4.d/3</MarginText>
<MarginInfo items={["AI05-0229-1"]} />
<Admonition type="aarm" aarm="note">
The pragmas have the same functionality as the corresponding aspect (unlike the typical obsolescent feature), and rejecting them could be a significant portability problem for existing code. <br /></Admonition>
</AnnotatedOnly>

<MarginText>5/3</MarginText>
<MarginInfo items={["AI95-00381-01", "AI05-0241-1"]} />
<p>The following <em>restriction_parameter_</em><code><a href="../AA-2/AA-2.3#S0002">identifier</a></code>s are language defined:<br /></p>

<MarginText>6/2</MarginText>
<MarginInfo items={["AI95-00381-01"]} />
<dt><br/>No_Dependence </dt>
<dl>
<dd>Specifies a library unit on which there are no semantic dependences.<br /></dd>
<MarginText>6.1/3</MarginText>
<MarginInfo items={["AI05-0241-1"]} />
<dt><br/>No_Specification_of_Aspect </dt>
<dd>Identifies an aspect for which no <code><a href="../AA-13/AA-13.1#S0346">aspect_specification</a></code>, <code><a href="../AA-13/AA-13.3#S0349">attribute_definition_clause</a></code>, or <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code> is given.<br /></dd>
<MarginText>6.2/3</MarginText>
<MarginInfo items={["AI05-0272-1"]} />
<dt><br/>No_Use_Of_Attribute </dt>
<dd>Identifies an attribute for which no <code><a href="../AA-4/AA-4.1#S0100">attribute_reference</a></code> or <code><a href="../AA-13/AA-13.3#S0349">attribute_definition_clause</a></code> is given.<br /></dd>
<MarginText>6.3/3</MarginText>
<MarginInfo items={["AI05-0272-1"]} />
<dt><br/>No_Use_Of_Pragma </dt>
<dd>Identifies a <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code> which is not to be used.<br /></dd>
<MarginText>6.4/5</MarginText>
<MarginInfo items={["AI12-0389-1"]} />
<dt><br/>No_Unrecognized_Aspects </dt>
<dd>There are no <code><a href="../AA-13/AA-13.1#S0346">aspect_specification</a></code>s having an unrecognized <em>aspect_</em><code><a href="../AA-2/AA-2.3#S0002">identifier</a></code>. This restriction applies only to the current compilation or environment, not the entire partition.<br /></dd>
</dl>
<AnnotatedOnly>
<MarginText>6.a/5</MarginText>
<Admonition type="aarm" aarm="ramification">
<strong></strong>When this restriction is in effect, unrecognized aspects cannot be ignored in the current compilation; they violate the restriction. This is true despite the Implementation Permissions of <a href="../AA-13/AA-13.1#Subclause_13.1.1">13.1.1</a>. <br /></Admonition>
</AnnotatedOnly>

<MarginText>6.5/5</MarginText>
<MarginInfo items={["AI12-0389-1"]} />
<dt><br/>No_Unrecognized_Pragmas </dt>
<dl>
<dd>There are no <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code>s having an unrecognized pragma <code><a href="../AA-2/AA-2.3#S0002">identifier</a></code>. This restriction applies only to the current compilation or environment, not the entire partition.<br /></dd>
</dl>
<AnnotatedOnly>
<MarginText>6.b/5</MarginText>
<Admonition type="aarm" aarm="ramification">
<strong></strong>When this restriction is in effect, unrecognized pragmas cannot be ignored in the current compilation; they violate the restriction. This is true despite the rules of <a href="../AA-2/AA-2.8">2.8</a>. <br /></Admonition>
</AnnotatedOnly>


#### Legality Rules

<MarginText>7/5</MarginText>
<MarginInfo items={["AI95-00381-01", "AI12-0444-1"]} />
<p>The <code><a href="../AA-13/AA-13.12#S0360">restriction_parameter_argument</a></code> of a No_Dependence restriction shall be a <code><a href="../AA-4/AA-4.1#S0091">name</a></code>; the <code><a href="../AA-4/AA-4.1#S0091">name</a></code> shall have the form of a full expanded name of a library unit, but can be a name that has no corresponding  unit currently present in the environment.<br /></p>

<MarginText>7.a/2</MarginText>
<Admonition type="aarm" aarm="ramification">
<strong></strong>This <code><a href="../AA-4/AA-4.1#S0091">name</a></code> is not resolved. <br /></Admonition>
</AnnotatedOnly>

<MarginText>7.1/3</MarginText>
<MarginInfo items={["AI05-0241-1"]} />
<p>The <code><a href="../AA-13/AA-13.12#S0360">restriction_parameter_argument</a></code> of a No_Specification_of_Aspect restriction shall be an <code><a href="../AA-2/AA-2.3#S0002">identifier</a></code>; this is an identifier specific to a pragma (see <a href="../AA-2/AA-2.8">2.8</a>) and does not denote any declaration.<br /></p>

<AnnotatedOnly>
<MarginText>7.b/3</MarginText>
<Admonition type="aarm" aarm="ramification">
<strong></strong>This <code><a href="../AA-13/AA-13.12#S0360">restriction_parameter_argument</a></code> is not resolved as it is an identifier specific to a pragma. As for No_Dependence, there is no check that the aspect <code><a href="../AA-2/AA-2.3#S0002">identifier</a></code> is meaningful; it might refer to an implementation-defined aspect on one implementation, but nothing at all on another implementation. <br /></Admonition>
</AnnotatedOnly>

<MarginText>7.2/3</MarginText>
<MarginInfo items={["AI05-0272-1"]} />
<p>The <code><a href="../AA-13/AA-13.12#S0360">restriction_parameter_argument</a></code> of a No_Use_Of_Attribute restriction shall be an <code><a href="../AA-2/AA-2.3#S0002">identifier</a></code> or one of the reserved words Access, Delta, Digits, Mod, or Range; this is an identifier specific to a pragma.<br /></p>

<AnnotatedOnly>
<MarginText>7.c/3</MarginText>
<Admonition type="aarm" aarm="ramification">
<strong></strong>This <code><a href="../AA-13/AA-13.12#S0360">restriction_parameter_argument</a></code> is not resolved as it is an identifier specific to a pragma. There is no check that the attribute identifier refers to a known <code><a href="../AA-4/AA-4.1#S0101">attribute_designator</a></code>; it might refer to an implementation-defined attribute on one implementation, but nothing at all on another implementation. <br /></Admonition>
</AnnotatedOnly>

<MarginText>7.3/3</MarginText>
<MarginInfo items={["AI05-0272-1"]} />
<p>The <code><a href="../AA-13/AA-13.12#S0360">restriction_parameter_argument</a></code> of a No_Use_Of_Pragma restriction shall be an <code><a href="../AA-2/AA-2.3#S0002">identifier</a></code> or the reserved word Interface; this is an identifier specific to a pragma.<br /></p>

<AnnotatedOnly>
<MarginText>7.d/3</MarginText>
<Admonition type="aarm" aarm="ramification">
<strong></strong>This <code><a href="../AA-13/AA-13.12#S0360">restriction_parameter_argument</a></code> is not resolved as it is an identifier specific to a pragma. There is no check that the pragma identifier refers to a known <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code>; it might refer to an implementation-defined pragma on one implementation, but nothing at all on another implementation. <br /></Admonition>
</AnnotatedOnly>


#### Post-Compilation Rules

<MarginText>8/3</MarginText>
<MarginInfo items={["AI95-00381-01", "AI05-0241-1"]} />
<p>No compilation unit included in the partition shall depend semantically on the library unit identified by the <code><a href="../AA-4/AA-4.1#S0091">name</a></code> of a No_Dependence restriction.<br /></p>

<AnnotatedOnly>
<MarginText>8.a/5</MarginText>
<MarginInfo items={["AI12-0005-1"]} />
<Admonition type="aarm" aarm="ramification">
<strong></strong>There is no requirement that the library unit actually exists . One possible use of the pragma is to prevent the use of implementation-defined units; when the program is ported to a different compiler, it is perfectly reasonable that no unit with the name exists . <br /></Admonition>
</AnnotatedOnly>
