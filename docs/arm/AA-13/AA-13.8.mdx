<MarginText>1</MarginText>
<p>[ A machine code insertion can be achieved by a call to a subprogram whose <code><a href="../AA-5/AA-5.1#S0166">sequence_of_statements</a></code> contains <code><a href="../AA-13/AA-13.8#S0357">code_statement</a></code>s.] <br /></p>


#### Syntax

<MarginText>2</MarginText>

<CodeBlock>
<code>code_statement</code><a id="S0357"></a><code> ::= </code><code><a href="../AA-4/AA-4.7#S0163">qualified_expression</a></code>;<br />

</CodeBlock>
<MarginText>3</MarginText>
<p class="Indented2">A <code><a href="../AA-13/AA-13.8#S0357">code_statement</a></code> is only allowed in the <code><a href="../AA-11/AA-11.2#S0304">handled_sequence_of_statements</a></code> of a <code><a href="../AA-6/AA-6.3#S0216">subprogram_body</a></code>. If a <code><a href="../AA-6/AA-6.3#S0216">subprogram_body</a></code> contains any <code><a href="../AA-13/AA-13.8#S0357">code_statement</a></code>s, then within this <code><a href="../AA-6/AA-6.3#S0216">subprogram_body</a></code> the only allowed form of <code><a href="../AA-5/AA-5.1#S0167">statement</a></code> is a <code><a href="../AA-13/AA-13.8#S0357">code_statement</a></code> (labeled or not), the only allowed <code><a href="../AA-3/AA-3.11#S0087">declarative_item</a></code>s are <code><a href="../AA-8/AA-8.4#S0235">use_clause</a></code>s, and no <code><a href="../AA-11/AA-11.2#S0305">exception_handler</a></code> is allowed (<code><a href="../AA-2/AA-2.7#S0018">comment</a></code>s and <code><a href="../AA-2/AA-2.8#S0019">pragma</a></code>s are allowed as usual). <br /></p>


#### Name Resolution Rules

<MarginText>4</MarginText>
<p>The <code><a href="../AA-4/AA-4.7#S0163">qualified_expression</a></code> is expected to be of any type. <br /></p>


#### Legality Rules

<MarginText>5</MarginText>
<p>The <code><a href="../AA-4/AA-4.7#S0163">qualified_expression</a></code> shall be of a type declared in package System.Machine_Code. <br /></p>

<AnnotatedOnly>
<MarginText>5.a</MarginText>
<Admonition type="aarm" aarm="ramification">
<strong></strong>This includes types declared in children of System.Machine_Code. <br /></Admonition>
</AnnotatedOnly>

<MarginText>6</MarginText>
<p>A <code><a href="../AA-13/AA-13.8#S0357">code_statement</a></code> shall appear only within the scope of a <code><a href="../AA-10/AA-10.1#S0294">with_clause</a></code> that mentions package System.Machine_Code. <br /></p>

<AnnotatedOnly>
<MarginText>6.a</MarginText>
<Admonition type="aarm" aarm="ramification">
<strong></strong>Note that this is not a note; without this rule, it would be possible to write machine code in compilation units which depend on System.Machine_Code only indirectly. <br /></Admonition>
</AnnotatedOnly>


#### Static Semantics

<MarginText>7</MarginText>
<p>The contents of the library package System.Machine_Code (if provided) are implementation defined. The meaning of <code><a href="../AA-13/AA-13.8#S0357">code_statement</a></code>s is implementation defined. [Typically, each <code><a href="../AA-4/AA-4.7#S0163">qualified_expression</a></code> represents a machine instruction or assembly directive.] <br /></p>

<AnnotatedOnly>
<MarginText>7.a</MarginText>
<Admonition type="aarm" aarm="discussion">
<strong></strong>For example, an instruction might be a record with an Op_Code component and other components for the operands. <br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>7.b</MarginText>
<Admonition type="aarm" aarm="implementation-defined">
<strong></strong>The contents of the visible part of package System.Machine_Code, and the meaning of <code><a href="../AA-13/AA-13.8#S0357">code_statement</a></code>s.<br /></Admonition>
</AnnotatedOnly>


#### Implementation Permissions

<MarginText>8</MarginText>
<p>An implementation may place restrictions on <code><a href="../AA-13/AA-13.8#S0357">code_statement</a></code>s. An implementation is not required to provide package System.Machine_Code. <br /></p>

<AnnotatedOnly>
<MarginText>9/5</MarginText>
<MarginInfo items={["AI12-0440-1"]} />
<Admonition type="aarm" aarm="note">
NOTE 1   An implementation can  provide implementation-defined pragmas specifying register conventions and calling conventions.<br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>10/2</MarginText>
<MarginInfo items={["AI95-00318-02"]} />
<Admonition type="aarm" aarm="note">
NOTE 2   Machine code functions are exempt from the rule that a return statement is required. In fact, return statements are forbidden, since only <code><a href="../AA-13/AA-13.8#S0357">code_statement</a></code>s are allowed. <br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>10.a</MarginText>
<Admonition type="aarm" aarm="discussion">
<strong></strong>The idea is that the author of a machine code subprogram knows the calling conventions, and refers to parameters and results accordingly. The implementation should document where to put the result of a machine code function, for example, &ldquo;Scalar results are returned in register 0&rdquo;. <br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>11</MarginText>
<Admonition type="aarm" aarm="note">
NOTE 3   Intrinsic subprograms (see <a href="../AA-6/AA-6.3#Subclause_6.3.1">6.3.1</a>, &ldquo;<a href="../AA-6/AA-6.3#Subclause_6.3.1">Conformance Rules</a>&rdquo;) can also be used to achieve machine code insertions. Interface to assembly language can be achieved using the features in <a href="../AA-B/">Annex B</a>, &ldquo;<a href="../AA-B/">Interface to Other Languages</a>&rdquo;. <br /></Admonition>
</AnnotatedOnly>


#### Examples

<MarginText>12</MarginText>
<p><em>Example of a code statement:</em> <br /></p>

<MarginText>13/3</MarginText>
<MarginInfo items={["AI05-0229-1"]} />
<CodeBlock language="ada">
M : Mask;{"\n"}procedure Set_Mask{"\n"}  with Inline;{"\n"}
<MarginText>14</MarginText>
procedure Set_Mask is{"\n"}  use System.Machine_Code; -- assume &ldquo;with System.Machine_Code;&rdquo; appears somewhere above{"\n"}begin{"\n"}  SI_Format'(Code ={">"} SSM, B ={">"} M'Base_Reg, D ={">"} M'Disp);{"\n"}  --  Base_Reg and Disp are implementation-defined attributes{"\n"}end Set_Mask;{"\n"}

</CodeBlock>
<AnnotatedOnly>

#### Extensions to Ada 83

</AnnotatedOnly>
<AnnotatedOnly>
<MarginText>14.a</MarginText>
<Admonition type="aarm" aarm="note">
Machine code functions are allowed in Ada 95; in Ada 83, only procedures were allowed. <br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>

#### Wording Changes from Ada 83

</AnnotatedOnly>
<AnnotatedOnly>
<MarginText>14.b</MarginText>
<Admonition type="aarm" aarm="note">
The syntax for <code><a href="../AA-13/AA-13.8#S0357">code_statement</a></code> is changed to say &ldquo;<code><a href="../AA-4/AA-4.7#S0163">qualified_expression</a></code>&rdquo; instead of &ldquo;<code><a href="../AA-3/AA-3.2#S0028">subtype_mark</a></code>'<code><a href="../AA-4/AA-4.3#S0107">record_aggregate</a></code>&rdquo;. Requiring the type of each instruction to be a record type is overspecification. <br /></Admonition>
</AnnotatedOnly>
