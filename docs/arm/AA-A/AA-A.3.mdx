<p>This subclause presents the packages related to character processing: an empty declared pure package Characters and child packages Characters.Handling and Characters.Latin_1. The package Characters.Handling provides classification and conversion functions for Character data, and some simple functions for dealing with Wide_Character and Wide_Wide_Character data. The child package Characters.Latin_1 declares a set of constants initialized to values of type Character. <br /></p>

<p>In the description below for each function that returns a Boolean result, the effect is described in terms of the conditions under which the value True is returned. If these conditions are not met, then the function returns False.<br /></p>

<MarginText>21</MarginText>
<p>Each of the following classification functions has a formal Character parameter, Item, and returns a Boolean result. <br /></p>

<MarginText>22</MarginText>
<dt><br/>Is_Control </dt>
<dl>
<dd>True if Item is a control character. A <em>control character</em> is a character whose position is in one of the ranges 0..31 or 127..159.<br /></dd>
<MarginText>23</MarginText>
<dt><br/>Is_Graphic </dt>
<dd>True if Item is a graphic character. A <em>graphic character</em> is a character whose position is in one of the ranges 32..126 or 160..255.<br /></dd>
<MarginText>24</MarginText>
<dt><br/>Is_Letter </dt>
<dd>True if Item is a letter. A <em>letter</em> is a character that is in one of the ranges 'A'..'Z' or 'a'..'z', or whose position is in one of the ranges 192..214, 216..246, or 248..255.<br /></dd>
<MarginText>25</MarginText>
<dt><br/>Is_Lower </dt>
<dd>True if Item is a lower-case letter. A <em>lower-case letter</em> is a character that is in the range 'a'..'z', or whose position is in one of the ranges 223..246 or 248..255.<br /></dd>
<MarginText>26</MarginText>
<dt><br/>Is_Upper</dt>
<dd>True if Item is an upper-case letter. An <em>upper-case letter</em> is a character that is in the range 'A'..'Z' or whose position is in one of the ranges 192..214 or 216.. 222.<br /></dd>
<MarginText>27</MarginText>
<dt><br/>Is_Basic </dt>
<dd>True if Item is a basic letter. A <em>basic letter</em> is a character that is in one of the ranges 'A'..'Z' and 'a'..'z', or that is one of the following: 'Æ', 'æ', 'Ð', 'ð', 'Þ', 'þ', or 'ß'.<br /></dd>
<MarginText>28</MarginText>
<dt><br/>Is_Digit </dt>
<dd>True if Item is a decimal digit. A <em>decimal digit</em> is a character in the range '0'..'9'.<br /></dd>
<MarginText>29</MarginText>
<dt><br/>Is_Decimal_Digit </dt>
<dd>A renaming of Is_Digit.<br /></dd>
<MarginText>30</MarginText>
<dt><br/>Is_Hexadecimal_Digit </dt>
<dd>True if Item is a hexadecimal digit. A <em>hexadecimal digit</em> is a character that is either a decimal digit or that is in one of the ranges 'A' .. 'F' or 'a' .. 'f'.<br /></dd>
<MarginText>31</MarginText>
<dt><br/>Is_Alphanumeric </dt>
<dd>True if Item is an alphanumeric character. An <em>alphanumeric character</em> is a character that is either a letter or a decimal digit.<br /></dd>
<MarginText>32</MarginText>
<dt><br/>Is_Special </dt>
<dd>True if Item is a special graphic character. A <em>special graphic character</em> is a graphic character that is not alphanumeric.<br /></dd>
<MarginText>32.1/3</MarginText>
<MarginInfo items={["AI05-0185-1"]} />
<dt><br/>Is_Line_Terminator</dt>
<dd>True if Item is a character with position 10 .. 13 (Line_Feed, Line_Tabulation, Form_Feed, Carriage_Return) or 133 (Next_Line).<br /></dd>
<MarginText>32.2/3</MarginText>
<MarginInfo items={["AI05-0185-1"]} />
<dt><br/>Is_Mark</dt>
<dd>Never True (no value of type Character has categories Mark, Non-Spacing or Mark, Spacing Combining).<br /></dd>
<MarginText>32.3/3</MarginText>
<MarginInfo items={["AI05-0185-1"]} />
<dt><br/>Is_Other_Format</dt>
<dd>True if Item is a character with position 173 (Soft_Hyphen).<br /></dd>
<MarginText>32.4/3</MarginText>
<MarginInfo items={["AI05-0185-1"]} />
<dt><br/>Is_Punctuation_Connector</dt>
<dd>True if Item is a character with position 95 ('_', known as Low_Line or Underscore).<br /></dd>
<MarginText>32.5/3</MarginText>
<MarginInfo items={["AI05-0185-1"]} />
<dt><br/>Is_Space</dt>
<dd>True if Item is a character with position 32 (' ') or 160 (No_Break_Space).<br /></dd>
<MarginText>32.6/5</MarginText>
<MarginInfo items={["AI12-0004-1", "AI12-0263-1", "AI12-0439-1", "AI12-0450-1"]} />
<dt><br/>Is_NFKC</dt>
<dd>True if Item can be present in a string normalized to Normalization Form KC (as defined by Clause 22 of ISO/IEC 10646:2020); this includes all characters except those with positions 160, 168, 170, 175, 178, 179, 180, 181, 184, 185, 186, 188, 189, and 190. <br /></dd>
</dl>
<MarginText>33</MarginText>
<p>Each of the names To_Lower, To_Upper, and To_Basic refers to two functions: one that converts from Character to Character, and the other that converts from String to String. The result of each Character-to-Character function is described below, in terms of the conversion applied to Item, its formal Character parameter. The result of each String-to-String conversion is obtained by applying to each element of the function's String parameter the corresponding Character-to-Character conversion; the result is the null String if the value of the formal parameter is the null String. The lower bound of the result String is 1. <br /></p>

<MarginText>34</MarginText>
<dt><br/>To_Lower</dt>
<dl>
<dd>Returns the corresponding lower-case value for Item if Is_Upper(Item), and returns Item otherwise.<br /></dd>
<MarginText>35</MarginText>
<dt><br/>To_Upper</dt>
<dd>Returns the corresponding upper-case value for Item if Is_Lower(Item) and Item has an upper-case form, and returns Item otherwise. The lower case letters 'ß' and 'ÿ' do not have upper case forms.<br /></dd>
<MarginText>36</MarginText>
<dt><br/>To_Basic</dt>
<dd>Returns the letter corresponding to Item but with no diacritical mark, if Item is a letter but not a basic letter; returns Item otherwise. <br /></dd>
</dl>
<MarginText>37</MarginText>
<p>The following set of functions test for membership in the ISO 646 character range, or convert between ISO 646 and Character. <br /></p>

<MarginText>38</MarginText>
<dt><br/>Is_ISO_646</dt>
<dl>
<dd>The function whose formal parameter, Item, is of type Character returns True if Item is in the subtype ISO_646.<br /></dd>
<MarginText>39</MarginText>
<dt><br/>Is_ISO_646</dt>
<dd>The function whose formal parameter, Item, is of type String returns True if Is_ISO_646(Item(I)) is True for each I in Item'Range.<br /></dd>
<MarginText>40</MarginText>
<dt><br/>To_ISO_646</dt>
<dd>The function whose first formal parameter, Item, is of type Character returns Item if Is_ISO_646(Item), and returns the Substitute ISO_646 character otherwise.<br /></dd>
<MarginText>41</MarginText>
<dt><br/>To_ISO_646</dt>
<dd>The function whose first formal parameter, Item, is of type String returns the String whose Range is 1..Item'Length and each of whose elements is given by To_ISO_646 of the corresponding element in Item. <br /></dd>
</dl>

<MarginText>62.a/5</MarginText>
<MarginInfo items={["AI12-0263-1", "AI12-0450-1"]} />
<Admonition type="aarm" aarm="discussion">
<strong></strong>The intent is that the returned string include the year for 10646 (as in "10646:2020 "), and the version number for Unicode (as in "Unicode 13.0 "). We don't try to specify that further so we don't need to decide how to represent Corrigenda for 10646, nor which of these is preferred. (Giving a Unicode version is more accurate, as the case folding and mapping rules always come from a Unicode version [10646 just tells one to look at Unicode to get those], and the character classifications ought to be the same for equivalent versions, but we don't want to talk about non-ISO standards in an ISO standard.) <br /></Admonition>
</AnnotatedOnly>
