<p>This subclause presents the packages related to character processing: an empty declared pure package Characters and child packages Characters.Handling and Characters.Latin{"_"}1. The package Characters.Handling provides classification and conversion functions for Character data, and some simple functions for dealing with Wide{"_"}Character and Wide{"_"}Wide{"_"}Character data. The child package Characters.Latin{"_"}1 declares a set of constants initialized to values of type Character. <br /></p>

<MarginText>21_A.3.2</MarginText>
<p>Each of the following classification functions has a formal Character parameter, Item, and returns a Boolean result. <br /></p>

<MarginText>22_A.3.2</MarginText>
<dt><br/>Is{"_"}Control </dt>
<dl>
<dd>True if Item is a control character. A <i>control character</i> is a character whose position is in one of the ranges 0..31 or 127..159.<br /></dd>
<MarginText>23_A.3.2</MarginText>
<dt><br/>Is{"_"}Graphic </dt>
<dd>True if Item is a graphic character. A <i>graphic character</i> is a character whose position is in one of the ranges 32..126 or 160..255.<br /></dd>
<MarginText>24_A.3.2</MarginText>
<dt><br/>Is{"_"}Letter </dt>
<dd>True if Item is a letter. A <i>letter</i> is a character that is in one of the ranges 'A'..'Z' or 'a'..'z', or whose position is in one of the ranges 192..214, 216..246, or 248..255.<br /></dd>
<MarginText>25_A.3.2</MarginText>
<dt><br/>Is{"_"}Lower </dt>
<dd>True if Item is a lower-case letter. A <i>lower-case letter</i> is a character that is in the range 'a'..'z', or whose position is in one of the ranges 223..246 or 248..255.<br /></dd>
<MarginText>26_A.3.2</MarginText>
<dt><br/>Is{"_"}Upper</dt>
<dd>True if Item is an upper-case letter. An <i>upper-case letter</i> is a character that is in the range 'A'..'Z' or whose position is in one of the ranges 192..214 or 216.. 222.<br /></dd>
<MarginText>27_A.3.2</MarginText>
<dt><br/>Is{"_"}Basic </dt>
<dd>True if Item is a basic letter. A <i>basic letter</i> is a character that is in one of the ranges 'A'..'Z' and 'a'..'z', or that is one of the following: 'Æ', 'æ', 'Ð', 'ð', 'Þ', 'þ', or 'ß'.<br /></dd>
<MarginText>28_A.3.2</MarginText>
<dt><br/>Is{"_"}Digit </dt>
<dd>True if Item is a decimal digit. A <i>decimal digit</i> is a character in the range '0'..'9'.<br /></dd>
<MarginText>29_A.3.2</MarginText>
<dt><br/>Is{"_"}Decimal{"_"}Digit </dt>
<dd>A renaming of Is{"_"}Digit.<br /></dd>
<MarginText>30_A.3.2</MarginText>
<dt><br/>Is{"_"}Hexadecimal{"_"}Digit </dt>
<dd>True if Item is a hexadecimal digit. A <i>hexadecimal digit</i> is a character that is either a decimal digit or that is in one of the ranges 'A' .. 'F' or 'a' .. 'f'.<br /></dd>
<MarginText>31_A.3.2</MarginText>
<dt><br/>Is{"_"}Alphanumeric </dt>
<dd>True if Item is an alphanumeric character. An <i>alphanumeric character</i> is a character that is either a letter or a decimal digit.<br /></dd>
<MarginText>32_A.3.2</MarginText>
<dt><br/>Is{"_"}Special </dt>
<dd>True if Item is a special graphic character. A <i>special graphic character</i> is a graphic character that is not alphanumeric.<br /></dd>
<MarginText>32.1/3_A.3.2</MarginText>
<MarginInfo items={["AI05-0185-1"]} />
<dt><br/>Is{"_"}Line{"_"}Terminator</dt>
<dd>True if Item is a character with position 10 .. 13 (Line{"_"}Feed, Line{"_"}Tabulation, Form{"_"}Feed, Carriage{"_"}Return) or 133 (Next{"_"}Line).<br /></dd>
<MarginText>32.2/3_A.3.2</MarginText>
<MarginInfo items={["AI05-0185-1"]} />
<dt><br/>Is{"_"}Mark</dt>
<dd>Never True (no value of type Character has categories Mark, Non-Spacing or Mark, Spacing Combining).<br /></dd>
<MarginText>32.3/3_A.3.2</MarginText>
<MarginInfo items={["AI05-0185-1"]} />
<dt><br/>Is{"_"}Other{"_"}Format</dt>
<dd>True if Item is a character with position 173 (Soft{"_"}Hyphen).<br /></dd>
<MarginText>32.4/3_A.3.2</MarginText>
<MarginInfo items={["AI05-0185-1"]} />
<dt><br/>Is{"_"}Punctuation{"_"}Connector</dt>
<dd>True if Item is a character with position 95 ('{"_"}', known as Low{"_"}Line or Underscore).<br /></dd>
<MarginText>32.5/3_A.3.2</MarginText>
<MarginInfo items={["AI05-0185-1"]} />
<dt><br/>Is{"_"}Space</dt>
<dd>True if Item is a character with position 32 (' ') or 160 (No{"_"}Break{"_"}Space).<br /></dd>
<MarginText>32.6/5_A.3.2</MarginText>
<MarginInfo items={["AI12-0004-1", "AI12-0263-1", "AI12-0439-1", "AI12-0450-1"]} />
<dt><br/>Is{"_"}NFKC</dt>
<dd>True if Item can be present in a string normalized to Normalization Form KC (as defined by Clause 22 of ISO/IEC 10646:2020); this includes all characters except those with positions 160, 168, 170, 175, 178, 179, 180, 181, 184, 185, 186, 188, 189, and 190. <br /></dd>
</dl>
<MarginText>33_A.3.2</MarginText>
<p>Each of the names To{"_"}Lower, To{"_"}Upper, and To{"_"}Basic refers to two functions: one that converts from Character to Character, and the other that converts from String to String. The result of each Character-to-Character function is described below, in terms of the conversion applied to Item, its formal Character parameter. The result of each String-to-String conversion is obtained by applying to each element of the function's String parameter the corresponding Character-to-Character conversion; the result is the null String if the value of the formal parameter is the null String. The lower bound of the result String is 1. <br /></p>

<MarginText>34_A.3.2</MarginText>
<dt><br/>To{"_"}Lower</dt>
<dl>
<dd>Returns the corresponding lower-case value for Item if Is{"_"}Upper(Item), and returns Item otherwise.<br /></dd>
<MarginText>35_A.3.2</MarginText>
<dt><br/>To{"_"}Upper</dt>
<dd>Returns the corresponding upper-case value for Item if Is{"_"}Lower(Item) and Item has an upper-case form, and returns Item otherwise. The lower case letters 'ß' and 'ÿ' do not have upper case forms.<br /></dd>
<MarginText>36_A.3.2</MarginText>
<dt><br/>To{"_"}Basic</dt>
<dd>Returns the letter corresponding to Item but with no diacritical mark, if Item is a letter but not a basic letter; returns Item otherwise. <br /></dd>
</dl>
<MarginText>37_A.3.2</MarginText>
<p>The following set of functions test for membership in the ISO 646 character range, or convert between ISO 646 and Character. <br /></p>

<MarginText>38_A.3.2</MarginText>
<dt><br/>Is{"_"}ISO{"_"}646</dt>
<dl>
<dd>The function whose formal parameter, Item, is of type Character returns True if Item is in the subtype ISO{"_"}646.<br /></dd>
<MarginText>39_A.3.2</MarginText>
<dt><br/>Is{"_"}ISO{"_"}646</dt>
<dd>The function whose formal parameter, Item, is of type String returns True if Is{"_"}ISO{"_"}646(Item(I)) is True for each I in Item'Range.<br /></dd>
<MarginText>40_A.3.2</MarginText>
<dt><br/>To{"_"}ISO{"_"}646</dt>
<dd>The function whose first formal parameter, Item, is of type Character returns Item if Is{"_"}ISO{"_"}646(Item), and returns the Substitute ISO{"_"}646 character otherwise.<br /></dd>
<MarginText>41_A.3.2</MarginText>
<dt><br/>To{"_"}ISO{"_"}646</dt>
<dd>The function whose first formal parameter, Item, is of type String returns the String whose Range is 1..Item'Length and each of whose elements is given by To{"_"}ISO{"_"}646 of the corresponding element in Item. <br /></dd>
</dl>

<MarginInfo items={["AI12-0263-1", "AI12-0450-1"]} />
<Admonition type="aarm" aarm="discussion">
<b></b>The intent is that the returned string include the year for 10646 (as in "10646:2020"), and the version number for Unicode (as in "Unicode 13.0"). We don't try to specify that further so we don't need to decide how to represent Corrigenda for 10646, nor which of these is preferred. (Giving a Unicode version is more accurate, as the case folding and mapping rules always come from a Unicode version [10646 just tells one to look at Unicode to get those], and the character classifications ought to be the same for equivalent versions, but we don't want to talk about non-ISO standards in an ISO standard.) <br />
</Admonition>
</AnnotatedOnly>
