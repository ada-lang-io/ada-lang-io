<MarginText>1</MarginText>
<p>An <em>array</em> object is a composite object consisting of components which all have the same subtype. The name for a component of an array uses one or more index values belonging to specified discrete types. The value of an array object is a composite value consisting of the values of the components. <br /></p>

#### Name Resolution Rules

<MarginText>8</MarginText>
<p>For a <code><a href="../AA-3/AA-3.6#S0055">discrete_subtype_definition</a></code> that is a <code><a href="../AA-3/AA-3.5#S0037">range</a></code>, the <code><a href="../AA-3/AA-3.5#S0037">range</a></code> shall resolve to be of some specific discrete type[; which discrete type shall be determined without using any context other than the bounds of the <code><a href="../AA-3/AA-3.5#S0037">range</a></code> itself (plus the preference for <em>root_integer</em> &mdash; see <a href="../AA-8/AA-8.6">8.6</a>).] <br /></p>


#### Legality Rules

<MarginText>9</MarginText>
<p>Each <code><a href="../AA-3/AA-3.6#S0053">index_subtype_definition</a></code> or <code><a href="../AA-3/AA-3.6#S0055">discrete_subtype_definition</a></code> in an <code><a href="../AA-3/AA-3.6#S0051">array_type_definition</a></code> defines an <em>index subtype</em>; its type (the <em>index type</em>) shall be discrete. <br /></p>

<AnnotatedOnly>
<MarginText>9.a</MarginText>
<Admonition type="aarm" aarm="discussion">
<strong></strong>An <em>index</em> is a discrete quantity used to select along a given dimension of an array. A component is selected by specifying corresponding values for each of the indices. <br /></Admonition>
</AnnotatedOnly>

<MarginText>10</MarginText>
<p>The subtype defined by the <code><a href="../AA-3/AA-3.2#S0027">subtype_indication</a></code> of a <code><a href="../AA-3/AA-3.6#S0056">component_definition</a></code> (the <em>component subtype</em>) shall be a definite subtype. <br /></p>

<AnnotatedOnly>
<MarginText>10.a</MarginText>
<Admonition type="aarm" aarm="ramification">
<strong></strong>This applies to all uses of <code><a href="../AA-3/AA-3.6#S0056">component_definition</a></code>, including in <code><a href="../AA-3/AA-3.8#S0066">record_type_definition</a></code>s and <code><a href="../AA-9/AA-9.4#S0251">protected_definition</a></code>s.<br /></Admonition>

#### Static Semantics

<MarginText>12</MarginText>
<p>An array is characterized by the number of indices (the <em>dimensionality</em> of the array), the type and position of each index, the lower and upper bounds for each index, and the subtype of the components. The order of the indices is significant.<br /></p>

<MarginText>13</MarginText>
<p>A one-dimensional array has a distinct component for each possible index value. A multidimensional array has a distinct component for each possible sequence of index values that can be formed by selecting one value for each index position (in the given order). The possible values for a given index are all the values between the lower and upper bounds, inclusive; this range of values is called the <em>index range</em>. The <em>bounds</em> of an array are the bounds of its index ranges. The <em>length</em> of a dimension of an array is the number of values of the index range of the dimension (zero for a null range). The <em>length</em> of a one-dimensional array is the length of its only dimension.<br /></p>

<MarginText>14</MarginText>
<p>An <code><a href="../AA-3/AA-3.6#S0051">array_type_definition</a></code> defines an array type and its first subtype. For each object of this array type, the number of indices, the type and position of each index, and the subtype of the components are as in the type definition[; the values of the lower and upper bounds for each index belong to the corresponding index subtype of its type, except for null arrays (see <a href="../AA-3/AA-3.6#Subclause_3.6.1">3.6.1</a>)].<br /></p>

<MarginText>15/5</MarginText>
<MarginInfo items={["AI12-0444-1"]} />
<p>An <code><a href="../AA-3/AA-3.6#S0052">unconstrained_array_definition</a></code> defines an array type with an unconstrained first subtype. Each <code><a href="../AA-3/AA-3.6#S0053">index_subtype_definition</a></code> defines the corresponding index subtype to be the subtype denoted by the <code><a href="../AA-3/AA-3.2#S0028">subtype_mark</a></code>. [ The compound delimiter {"<"}{">"} (called a <em>box</em>) of an <code><a href="../AA-3/AA-3.6#S0053">index_subtype_definition</a></code> stands for an undefined range (different objects of the type can  have different  bounds).]<br /></p>

<MarginText>16</MarginText>
<p>A <code><a href="../AA-3/AA-3.6#S0054">constrained_array_definition</a></code> defines an array type with a constrained first subtype. Each <code><a href="../AA-3/AA-3.6#S0055">discrete_subtype_definition</a></code> defines the corresponding index subtype, as well as the corresponding index range for the constrained first subtype. The <em>constraint</em> of the first subtype consists of the bounds of the index ranges. <br /></p>

<AnnotatedOnly>
<MarginText>16.a/3</MarginText>
<MarginInfo items={["AI05-0005-1"]} />
<Admonition type="aarm" aarm="discussion">
<strong></strong>Although there is no nameable unconstrained array subtype in this case, the predefined slicing and concatenation operations can operate on and yield values that do not necessarily belong to the first array subtype. This is also true for Ada 83. <br /></Admonition>
</AnnotatedOnly>

<MarginText>17</MarginText>
<p>The discrete subtype defined by a <code><a href="../AA-3/AA-3.6#S0055">discrete_subtype_definition</a></code> is either that defined by the <code><a href="../AA-3/AA-3.2#S0027">subtype_indication</a></code>, or a subtype determined by the <code><a href="../AA-3/AA-3.5#S0037">range</a></code> as follows: <br /></p>

<MarginText>18</MarginText>
<ul>
<li>If the type of the <code><a href="../AA-3/AA-3.5#S0037">range</a></code> resolves to <em>root_integer</em>, then the <code><a href="../AA-3/AA-3.6#S0055">discrete_subtype_definition</a></code> defines a subtype of the predefined type Integer with bounds given by a conversion to Integer of the bounds of the <code><a href="../AA-3/AA-3.5#S0037">range</a></code>; <br /></li>
</ul>
<AnnotatedOnly>
<MarginText>18.a</MarginText>
<Admonition type="aarm" aarm="reason">
<strong></strong>This ensures that indexing over the discrete subtype can be performed with regular Integers, rather than only <em>universal_integer</em>s. <br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>18.b</MarginText>
<Admonition type="aarm" aarm="discussion">
<strong></strong>We considered doing this by simply creating a &ldquo;preference&rdquo; for Integer when resolving the <code><a href="../AA-3/AA-3.5#S0037">range</a></code>. However, this can introduce <em>Beaujolais</em> effects when the <code><a href="../AA-4/AA-4.4#S0138">simple_expression</a></code>s involve calls on functions visible due to <strong>use</strong> clauses. <br /></Admonition>
</AnnotatedOnly>

<MarginText>19</MarginText>
<ul>
<li>Otherwise, the <code><a href="../AA-3/AA-3.6#S0055">discrete_subtype_definition</a></code> defines a subtype of the type of the <code><a href="../AA-3/AA-3.5#S0037">range</a></code>, with the bounds given by the <code><a href="../AA-3/AA-3.5#S0037">range</a></code>. <br /></li>
</ul>
<MarginText>20</MarginText>
<p>The <code><a href="../AA-3/AA-3.6#S0056">component_definition</a></code> of an <code><a href="../AA-3/AA-3.6#S0051">array_type_definition</a></code> defines the nominal subtype of the components. If the reserved word <strong>aliased</strong> appears in the <code><a href="../AA-3/AA-3.6#S0056">component_definition</a></code>, then each component of the array is aliased (see <a href="../AA-3/AA-3.10">3.10</a>). <br /></p>


#### Dynamic Semantics

<MarginText>21</MarginText>
<p>The elaboration of an <code><a href="../AA-3/AA-3.6#S0051">array_type_definition</a></code> creates the array type and its first subtype, and consists of the elaboration of any <code><a href="../AA-3/AA-3.6#S0055">discrete_subtype_definition</a></code>s and the <code><a href="../AA-3/AA-3.6#S0056">component_definition</a></code>.<br /></p>

<MarginText>22/2</MarginText>
<MarginInfo items={["AI95-00171-01", "AI95-00230-01"]} />
<p>{"{"}<em>8652/0002</em>{"}"} The elaboration of a <code><a href="../AA-3/AA-3.6#S0055">discrete_subtype_definition</a></code> that does not contain any per-object expressions creates the discrete subtype, and consists of the elaboration of the <code><a href="../AA-3/AA-3.2#S0027">subtype_indication</a></code> or the evaluation of the <code><a href="../AA-3/AA-3.5#S0037">range</a></code>. The elaboration of a <code><a href="../AA-3/AA-3.6#S0055">discrete_subtype_definition</a></code> that contains one or more per-object expressions is defined in <a href="../AA-3/AA-3.8">3.8</a>. The elaboration of a <code><a href="../AA-3/AA-3.6#S0056">component_definition</a></code> in an <code><a href="../AA-3/AA-3.6#S0051">array_type_definition</a></code> consists of the elaboration of the <code><a href="../AA-3/AA-3.2#S0027">subtype_indication</a></code> or <code><a href="../AA-3/AA-3.10#S0084">access_definition</a></code>. The elaboration of any <code><a href="../AA-3/AA-3.6#S0055">discrete_subtype_definition</a></code>s and the elaboration of the <code><a href="../AA-3/AA-3.6#S0056">component_definition</a></code> are performed in an arbitrary order. <br /></p>


#### Static Semantics

<MarginText>22.1/3</MarginText>
<MarginInfo items={["AI05-0228-1"]} />
<p>For an array type with a scalar component type, the following language-defined representation aspect may be specified with an <code><a href="../AA-13/AA-13.1#S0346">aspect_specification</a></code> (see <a href="../AA-13/AA-13.1#Subclause_13.1.1">13.1.1</a>): <br /></p>

<MarginText>22.2/3</MarginText>
<dt><br/>Default_Component_Value</dt>
<dl>
<dd>This aspect shall be specified by a static expression, and that expression shall be explicit, even if the aspect has a boolean type. Default_Component_Value shall be specified only on a <code><a href="../AA-3/AA-3.2#S0024">full_type_declaration</a></code>. <br /></dd>
</dl>
<AnnotatedOnly>
<MarginText>22.a/3</MarginText>
<Admonition type="aarm" aarm="reason">
<strong></strong>The part about requiring an explicit expression is to disallow omitting the value for this aspect, which would otherwise be allowed by the rules of <a href="../AA-13/AA-13.1#Subclause_13.1.1">13.1.1</a>.<br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>22.b/3</MarginText>
<Admonition type="aarm" aarm="note">
This is a representation attribute in order to disallow specifying it on a derived type that has inherited primitive subprograms; that is necessary as the sizes of <strong>out</strong> parameters could be different whether or not a Default_Value is specified (see <a href="../AA-6/AA-6.4#Subclause_6.4.1">6.4.1</a>). <br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>22.c/3</MarginText>
<Admonition type="aarm" aarm="note">
<strong>Aspect Description for </strong><strong>Default_Component_Value: </strong>Default value for the components of an array-of-scalar subtype.<br /></Admonition>
</AnnotatedOnly>

<MarginText>22.3/5</MarginText>
<MarginInfo items={["AI05-0228-1", "AI12-0427-1"]} />
<p>If a derived type  inherits a boolean Default_Component_Value aspect, the aspect may be specified to have any value for the derived type. <br /></p>

<AnnotatedOnly>
<MarginText>22.d/3</MarginText>
<Admonition type="aarm" aarm="reason">
<strong></strong>This overrides the <a href="../AA-13/AA-13.1#Subclause_13.1.1">13.1.1</a> rule that says that a boolean aspect with a value True cannot be changed. <br /></Admonition>
</AnnotatedOnly>


#### Name Resolution Rules

<MarginText>22.4/3</MarginText>
<MarginInfo items={["AI05-0228-1"]} />
<p>The expected type for the <code><a href="../AA-4/AA-4.4#S0132">expression</a></code> specified for the Default_Component_Value aspect is the component type of the array type defined by the <code><a href="../AA-3/AA-3.2#S0024">full_type_declaration</a></code> on which it appears. <br /></p>

<AnnotatedOnly>
<MarginText>23</MarginText>
<Admonition type="aarm" aarm="note">
NOTE 1   All components of an array have the same subtype. In particular, for an array of components that are one-dimensional arrays, this means that all components have the same bounds and hence the same length.<br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>24</MarginText>
<Admonition type="aarm" aarm="note">
NOTE 2   Each elaboration of an <code><a href="../AA-3/AA-3.6#S0051">array_type_definition</a></code> creates a distinct array type. A consequence of this is that each object whose <code><a href="../AA-3/AA-3.3#S0032">object_declaration</a></code> contains an <code><a href="../AA-3/AA-3.6#S0051">array_type_definition</a></code> is of its own unique type. <br /></Admonition>
</AnnotatedOnly>

## 3.6.1  Index Constraints and Discrete Ranges

<MarginText>1</MarginText>
<p>An <code><a href="../AA-3/AA-3.6#S0057">index_constraint</a></code> determines the range of possible values for every index of an array subtype, and thereby the corresponding array bounds. <br /></p>


#### Syntax

<MarginText>2</MarginText>

<CodeBlock>
<code>index_constraint</code><a id="S0057"></a><code> ::= </code> (<code><a href="../AA-3/AA-3.6#S0058">discrete_range</a></code> {"{"}, <code><a href="../AA-3/AA-3.6#S0058">discrete_range</a></code>{"}"})<br />

</CodeBlock>
<MarginText>3</MarginText>

<CodeBlock>
<code>discrete_range</code><a id="S0058"></a><code> ::= </code><em>discrete_</em><code><a href="../AA-3/AA-3.2#S0027">subtype_indication</a></code> | <code><a href="../AA-3/AA-3.5#S0037">range</a></code><br />

</CodeBlock>

#### Name Resolution Rules

<MarginText>4</MarginText>
<p>The type of a <code><a href="../AA-3/AA-3.6#S0058">discrete_range</a></code> is the type of the subtype defined by the <code><a href="../AA-3/AA-3.2#S0027">subtype_indication</a></code>, or the type of the <code><a href="../AA-3/AA-3.5#S0037">range</a></code>. For an <code><a href="../AA-3/AA-3.6#S0057">index_constraint</a></code>, each <code><a href="../AA-3/AA-3.6#S0058">discrete_range</a></code> shall resolve to be of the type of the corresponding index. <br /></p>

<AnnotatedOnly>
<MarginText>4.a</MarginText>
<Admonition type="aarm" aarm="discussion">
<strong></strong>In Ada 95, <code><a href="../AA-3/AA-3.6#S0057">index_constraint</a></code>s only appear in a <code><a href="../AA-3/AA-3.2#S0027">subtype_indication</a></code>; they no longer appear in <code><a href="../AA-3/AA-3.6#S0054">constrained_array_definition</a></code>s. <br /></Admonition>
</AnnotatedOnly>


#### Legality Rules

<MarginText>5</MarginText>
<p>An <code><a href="../AA-3/AA-3.6#S0057">index_constraint</a></code> shall appear only in a <code><a href="../AA-3/AA-3.2#S0027">subtype_indication</a></code> whose <code><a href="../AA-3/AA-3.2#S0028">subtype_mark</a></code> denotes either an unconstrained array subtype, or an unconstrained access subtype whose designated subtype is an unconstrained array subtype; in either case, the <code><a href="../AA-3/AA-3.6#S0057">index_constraint</a></code> shall provide a <code><a href="../AA-3/AA-3.6#S0058">discrete_range</a></code> for each index of the array type. <br /></p>


#### Static Semantics

<MarginText>6</MarginText>
<p>A <code><a href="../AA-3/AA-3.6#S0058">discrete_range</a></code> defines a range whose bounds are given by the <code><a href="../AA-3/AA-3.5#S0037">range</a></code>, or by the range of the subtype defined by the <code><a href="../AA-3/AA-3.2#S0027">subtype_indication</a></code>. <br /></p>


#### Dynamic Semantics

<MarginText>7</MarginText>
<p>An <code><a href="../AA-3/AA-3.6#S0057">index_constraint</a></code> is <em>compatible</em> with an unconstrained array subtype if and only if the index range defined by each <code><a href="../AA-3/AA-3.6#S0058">discrete_range</a></code> is compatible (see <a href="../AA-3/AA-3.5">3.5</a>) with the corresponding index subtype. If any of the <code><a href="../AA-3/AA-3.6#S0058">discrete_range</a></code>s defines a null range, any array thus constrained is a <em>null array</em>, having no components. An array value <em>satisfies</em> an <code><a href="../AA-3/AA-3.6#S0057">index_constraint</a></code> if at each index position the array value and the <code><a href="../AA-3/AA-3.6#S0057">index_constraint</a></code> have the same index bounds. <br /></p>

<AnnotatedOnly>
<MarginText>7.a</MarginText>
<Admonition type="aarm" aarm="ramification">
<strong></strong>There is no need to define compatibility with a constrained array subtype, because one is not allowed to constrain it again.<br /></Admonition>
</AnnotatedOnly>

<MarginText>8</MarginText>
<p>The elaboration of an <code><a href="../AA-3/AA-3.6#S0057">index_constraint</a></code> consists of the evaluation of the <code><a href="../AA-3/AA-3.6#S0058">discrete_range</a></code>(s), in an arbitrary order. The evaluation of a <code><a href="../AA-3/AA-3.6#S0058">discrete_range</a></code> consists of the elaboration of the <code><a href="../AA-3/AA-3.2#S0027">subtype_indication</a></code> or the evaluation of the <code><a href="../AA-3/AA-3.5#S0037">range</a></code>. <br /></p>

<AnnotatedOnly>
<MarginText>9</MarginText>
<Admonition type="aarm" aarm="note">
NOTE 1   The elaboration of a <code><a href="../AA-3/AA-3.2#S0027">subtype_indication</a></code> consisting of a <code><a href="../AA-3/AA-3.2#S0028">subtype_mark</a></code> followed by an <code><a href="../AA-3/AA-3.6#S0057">index_constraint</a></code> checks the compatibility of the <code><a href="../AA-3/AA-3.6#S0057">index_constraint</a></code> with the <code><a href="../AA-3/AA-3.2#S0028">subtype_mark</a></code> (see <a href="../AA-3/AA-3.2#Subclause_3.2.2">3.2.2</a>).<br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>10</MarginText>
<Admonition type="aarm" aarm="note">
NOTE 2   Even if an array value does not satisfy the index constraint of an array subtype, Constraint_Error is not raised on conversion to the array subtype, so long as the length of each dimension of the array value and the array subtype match. See <a href="../AA-4/AA-4.6">4.6</a>. <br /></Admonition>
</AnnotatedOnly>

## 3.6.2  Operations of Array Types


#### Legality Rules

<MarginText>1</MarginText>
<p>[The argument N used in the <code><a href="../AA-4/AA-4.1#S0101">attribute_designator</a></code>s for the N-th dimension of an array shall be a static <code><a href="../AA-4/AA-4.4#S0132">expression</a></code> of some integer type.] The value of N shall be positive (nonzero) and no greater than the dimensionality of the array. <br /></p>


#### Static Semantics

<MarginText>2/1</MarginText>
<MarginInfo items={["AI95-00030-01"]} />
<p>{"{"}<em>8652/0006</em>{"}"} The following attributes are defined for a <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> A that is of an array type [(after any implicit dereference)], or denotes a constrained array subtype: <br /></p>

<AnnotatedOnly>
<MarginText>2.a</MarginText>
<Admonition type="aarm" aarm="ramification">
<strong></strong>These attributes are not defined if A is a subtype-mark for an access-to-array subtype. They are defined (by implicit dereference) for access-to-array values.<br /></Admonition>
</AnnotatedOnly>

<MarginText>3</MarginText>
<dt><br/>A'First</dt>
<dl>
<dd>A'First denotes the lower bound of the first index range; its type is the corresponding index type.<br /></dd>
<MarginText>4</MarginText>
<dt><br/>A'First(N)</dt>
<dd>A'First(N) denotes the lower bound of the N-th index range; its type is the corresponding index type.<br /></dd>
<MarginText>5</MarginText>
<dt><br/>A'Last</dt>
<dd>A'Last denotes the upper bound of the first index range; its type is the corresponding index type.<br /></dd>
<MarginText>6</MarginText>
<dt><br/>A'Last(N)</dt>
<dd>A'Last(N) denotes the upper bound of the N-th index range; its type is the corresponding index type.<br /></dd>
<MarginText>7</MarginText>
<dt><br/>A'Range</dt>
<dd>A'Range is equivalent to the range A'First .. A'Last, except that the <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> A is only evaluated once.<br /></dd>
<MarginText>8</MarginText>
<dt><br/>A'Range(N)</dt>
<dd>A'Range(N) is equivalent to the range A'First(N) .. A'Last(N), except that the <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> A is only evaluated once.<br /></dd>
<MarginText>9</MarginText>
<dt><br/>A'Length</dt>
<dd>A'Length denotes the number of values of the first index range (zero for a null range); its type is <em>universal_integer</em>.<br /></dd>
<MarginText>10</MarginText>
<dt><br/>A'Length(N)</dt>
<dd>A'Length(N) denotes the number of values of the N-th index range (zero for a null range); its type is <em>universal_integer</em>. <br /></dd>
</dl>

#### Implementation Advice

<MarginText>11/3</MarginText>
<MarginInfo items={["AI05-0229-1"]} />
<p>An implementation should normally represent multidimensional arrays in row-major order, consistent with the notation used for multidimensional array aggregates (see <a href="../AA-4/AA-4.3#Subclause_4.3.3">4.3.3</a>). However, if convention Fortran is specified for a multidimensional array type, then column-major order should be used instead (see <a href="../AA-B/AA-B.5">B.5</a>, &ldquo;<a href="../AA-B/AA-B.5">Interfacing with Fortran</a>&rdquo;). <br /></p>

<AnnotatedOnly>
<MarginText>11.a/2</MarginText>
<Admonition type="aarm" aarm="implementation-advice">
<strong></strong>Multidimensional arrays should be represented in row-major order, unless the array has convention Fortran.<br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>12</MarginText>
<Admonition type="aarm" aarm="note">
NOTE 1   The <code><a href="../AA-4/AA-4.1#S0100">attribute_reference</a></code>s A'First and A'First(1) denote the same value. A similar relation exists for the <code><a href="../AA-4/AA-4.1#S0100">attribute_reference</a></code>s A'Last, A'Range, and A'Length. The following relation is satisfied (except for a null array) by the above attributes if the index type is an integer type: <br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>13</MarginText>
<CodeBlock language="ada">
   A'Length(N) = A'Last(N) - A'First(N) + 1{"\n"}

</CodeBlock>
</AnnotatedOnly>
<AnnotatedOnly>
<MarginText>14</MarginText>
<Admonition type="aarm" aarm="note">
NOTE 2   An array type is limited if its component type is limited (see <a href="../AA-7/AA-7.5">7.5</a>).<br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>15</MarginText>
<Admonition type="aarm" aarm="note">
NOTE 3   The predefined operations of an array type include the membership tests, qualification, and explicit conversion. If the array type is not limited, they also include assignment and the predefined equality operators. For a one-dimensional array type, they include the predefined concatenation operators (if nonlimited) and, if the component type is discrete, the predefined relational operators; if the component type is boolean, the predefined logical operators are also included.<br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>16/2</MarginText>
<MarginInfo items={["AI95-00287-01"]} />
<Admonition type="aarm" aarm="note">
NOTE 4   A component of an array can be named with an <code><a href="../AA-4/AA-4.1#S0096">indexed_component</a></code>. A value of an array type can be specified with an <code><a href="../AA-4/AA-4.3#S0113">array_aggregate</a></code>. For a one-dimensional array type, a slice of the array can be named; also, string literals are defined if the component type is a character type. <br /></Admonition>
</AnnotatedOnly>

## 3.6.3  String Types


#### Static Semantics

<MarginText>1</MarginText>
<p>A one-dimensional array type whose component type is a character type is called a <em>string type</em>.<br /></p>

<MarginText>2/2</MarginText>
<MarginInfo items={["AI95-00285-01"]} />
<p>[There are three predefined string types, String, Wide_String, and Wide_Wide_String, each indexed by values of the predefined subtype Positive; these are declared in the visible part of package Standard:] <br /></p>

<MarginText>3</MarginText>
<CodeBlock language="ada">
[subtype Positive is Integer range 1 .. Integer'Last;{"\n"}
<MarginText>4/2</MarginText>
<MarginInfo items={["AI95-00285-01"]} />
type String is array(Positive range {"<"}{">"}) of Character;{"\n"}type Wide_String is array(Positive range {"<"}{">"}) of Wide_Character;{"\n"}type Wide_Wide_String is array(Positive range {"<"}{">"}) of Wide_Wide_Character;{"\n"}]{"\n"}

</CodeBlock>
<AnnotatedOnly>
<MarginText>5</MarginText>
<Admonition type="aarm" aarm="note">
NOTE   String literals (see <a href="../AA-2/AA-2.6">2.6</a> and <a href="../AA-4/AA-4.2">4.2</a>) are defined for all string types. The concatenation operator & is predefined for string types, as for all nonlimited one-dimensional array types. The ordering operators {"<"}, {"<"}=, {">"}, and {">"}= are predefined for string types, as for all one-dimensional discrete array types; these ordering operators correspond to lexicographic order (see <a href="../AA-4/AA-4.5#Subclause_4.5.2">4.5.2</a>).<br /></Admonition>
</AnnotatedOnly>


#### Examples

<MarginText>6</MarginText>

Ask_Twice  : String  := Question & Question; -- constrained to (1..40){"\n"}Ninety_Six : constant Roman   := "XCVI";     -- see <a href="../AA-3/AA-3.5#Subclause_3.5.2">3.5.2</a> and <a href="../AA-3/AA-3.6">3.6</a>{"\n"}
