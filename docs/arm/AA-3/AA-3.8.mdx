<MarginText>1</MarginText>
<p>A record object is a composite object consisting of named components. The value of a record object is a composite value consisting of the values of the components. <br /></p>

<MarginText>7</MarginText>
<p>The expected type for the <code><a href="../AA-3/AA-3.7#S0063">default_expression</a></code>, if any, in a <code><a href="../AA-3/AA-3.8#S0070">component_declaration</a></code> is the type of the component. <br /></p>


#### Legality Rules

<MarginText>8/2</MarginText>
<MarginInfo items={["AI95-00287-01"]} />
<p><em>This paragraph was deleted.</em>{"{"}<em>{"AI95-00287-01"}</em>{"}"} <br /></p>

<MarginText>9/2</MarginText>
<MarginInfo items={["AI95-00366-01"]} />
<p>{"{"}<em>{"AI95-00366-01"}</em>{"}"} <br />Each <code><a href="../AA-3/AA-3.8#S0070">component_declaration</a></code> declares a component of the record type. Besides components declared by <code><a href="../AA-3/AA-3.8#S0070">component_declaration</a></code>s, the components of a record type include any components declared by <code><a href="../AA-3/AA-3.7#S0062">discriminant_specification</a></code>s of the record type declaration. [The identifiers of all components of a record type shall be distinct.] <br /></p>

<MarginText>9.a/3</MarginText>
<MarginInfo items={["AI05-0299-1"]} />
<Admonition type="aarm" aarm="proof" title="Proof: ">
<strong></strong>{"{"}<em>{"AI05-0299-1"}</em>{"}"} <br />The identifiers of all components of a record type have to be distinct because they are all declared immediately within the same declarative region. See Clause <a href="../AA-8/">8</a>. <br /></Admonition>

<MarginText>10</MarginText>
<p>Within a <code><a href="../AA-3/AA-3.2#S0023">type_declaration</a></code>, a <code><a href="../AA-4/AA-4.1#S0091">name</a></code> that denotes a component, protected subprogram, or entry of the type is allowed only in the following cases:<br /></p>

<MarginText>11/3</MarginText>
<MarginInfo items={["AI05-0004-1", "AI05-0295-1"]} />
<ul>
<li>{"{"}<em>{"AI05-0004-1"}</em>{"}"} {"{"}<em>{"AI05-0295-1"}</em>{"}"} <br />A <code><a href="../AA-4/AA-4.1#S0091">name</a></code> that denotes any component, protected subprogram, or entry is allowed within an <code><a href="../AA-13/AA-13.1#S0346">aspect_specification</a></code>, an operational item, or a representation item that occurs within the declaration of the composite type.<br /></li>
<MarginText>12/3</MarginText>
<MarginInfo items={["AI05-0264-1"]} />
<li>{"{"}<em>{"AI05-0264-1"}</em>{"}"} <br />A <code><a href="../AA-4/AA-4.1#S0091">name</a></code> that denotes a noninherited discriminant is allowed within the declaration of the type, but not within the <code><a href="../AA-3/AA-3.7#S0059">discriminant_part</a></code>. If the discriminant is used to define the constraint of a component, the bounds of an entry family, or the constraint of the parent subtype in a <code><a href="../AA-3/AA-3.4#S0035">derived_type_definition</a></code>, then its name shall appear alone as a <code><a href="../AA-4/AA-4.1#S0092">direct_name</a></code> (not as part of a larger expression or expanded name). A discriminant shall not be used to define the constraint of a scalar component. <br /></li>
</ul>
<MarginText>12.a</MarginText>
<Admonition type="aarm" aarm="reason" title="Reason: ">
<strong></strong>The penultimate restriction simplifies implementation, and allows the outer discriminant and the inner discriminant or bound to possibly share storage. <br /></Admonition>

<MarginText>12.b</MarginText>
<Admonition type="aarm" aarm="ramification" title="Ramification: ">
<strong></strong>Other rules prevent such a discriminant from being an inherited one. <br /></Admonition>

<MarginText>12.c</MarginText>
<Admonition type="aarm" aarm="reason" title="Reason: ">
<strong></strong>The last restriction is inherited from Ada 83. The restriction is not really necessary from a language design point of view, but we did not remove it, in order to avoid unnecessary changes to existing compilers. <br /></Admonition>

<MarginText>12.d</MarginText>
<Admonition type="aarm" aarm="discussion" title="Discussion: ">
<strong></strong>Note that a discriminant can be used to define the constraint for a component that is of an access-to-composite type. <br /></Admonition>

<MarginText>12.e/2</MarginText>
<MarginInfo items={["AI95-00373-01"]} />
<Admonition type="aarm" aarm="reason" title="Reason: ">
<strong></strong>{"{"}<em>{"AI95-00373-01"}</em>{"}"} <br />The above rules, and a similar one in <a href="../AA-6/AA-6.1">6.1</a> for formal parameters, are intended to allow initializations of components or parameters to occur in a (nearly) arbitrary order &mdash; whatever order is most efficient (subject to the restrictions of <a href="../AA-3/AA-3.3#Subclause_3.3.1">3.3.1</a>), since one <code><a href="../AA-3/AA-3.7#S0063">default_expression</a></code> cannot depend on the value of another one. They also prevent circularities. <br /></Admonition>

<MarginText>12.f/3</MarginText>
<MarginInfo items={["AI05-0295-1"]} />
<Admonition type="aarm" aarm="ramification" title="Ramification: ">
<strong></strong>{"{"}<em>{"AI05-0295-1"}</em>{"}"} <br />Inherited discriminants are not allowed to be denoted, except within <code><a href="../AA-13/AA-13.1#S0346">aspect_specification</a></code>s and representation items. However, the <em>discriminant_</em><code><a href="../AA-4/AA-4.1#S0099">selector_name</a></code> of the parent <code><a href="../AA-3/AA-3.2#S0027">subtype_indication</a></code> is allowed to denote a discriminant of the parent. <br /></Admonition>

<MarginText>13</MarginText>
<p>If the name of the current instance of a type (see <a href="../AA-8/AA-8.6">8.6</a>) is used to define the constraint of a component, then it shall appear as a <code><a href="../AA-4/AA-4.1#S0092">direct_name</a></code> that is the <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> of an <code><a href="../AA-4/AA-4.1#S0100">attribute_reference</a></code> whose result is of an access type, and the <code><a href="../AA-4/AA-4.1#S0100">attribute_reference</a></code> shall appear alone. <br /></p>

<MarginText>13.a</MarginText>
<Admonition type="aarm" aarm="reason" title="Reason: ">
<strong></strong>This rule allows T'Access or T'Unchecked_Access, but disallows, for example, a range constraint (1..T'Size). Allowing things like (1..T'Size) would mean that a per-object constraint could affect the size of the object, which would be bad. <br /></Admonition>


#### Static Semantics

<MarginText>13.1/3</MarginText>
<MarginInfo items={["AI95-00318-02", "AI05-0004-1"]} />
<p>{"{"}<em>{"AI95-00318-02"}</em>{"}"} {"{"}<em>{"AI05-0004-1"}</em>{"}"} <br />If a <code><a href="../AA-3/AA-3.8#S0066">record_type_definition</a></code> includes the reserved word <strong>limited</strong>, the type is called an <em>explicitly limited record</em> type.<br /></p>

<MarginText>14</MarginText>
<p>The <code><a href="../AA-3/AA-3.6#S0056">component_definition</a></code> of a <code><a href="../AA-3/AA-3.8#S0070">component_declaration</a></code> defines the (nominal) subtype of the component. If the reserved word <strong>aliased</strong> appears in the <code><a href="../AA-3/AA-3.6#S0056">component_definition</a></code>, then the component is aliased (see <a href="../AA-3/AA-3.10">3.10</a>). <br /></p>

<MarginText>15</MarginText>
<p>If the <code><a href="../AA-3/AA-3.8#S0068">component_list</a></code> of a record type is defined by the reserved word <strong>null</strong> and there are no discriminants, then the record type has no components and all records of the type are <em>null records</em>. A <code><a href="../AA-3/AA-3.8#S0067">record_definition</a></code> of <strong>null record</strong> is equivalent to <strong>record null; end record</strong>. <br /></p>

<MarginText>15.a/5</MarginText>
<MarginInfo items={["AI12-0426-1"]} />
<Admonition type="aarm" aarm="ramification" title="Ramification: ">
<strong></strong>{"{"}<em>{"AI12-0426-1"}</em>{"}"} <br />This shorthand  is available both for declaring a record type and a record extension &mdash; see <a href="../AA-3/AA-3.9#Subclause_3.9.1">3.9.1</a>. <br /></Admonition>


#### Dynamic Semantics

<MarginText>16</MarginText>
<p>The elaboration of a <code><a href="../AA-3/AA-3.8#S0066">record_type_definition</a></code> creates the record type and its first subtype, and consists of the elaboration of the <code><a href="../AA-3/AA-3.8#S0067">record_definition</a></code>. The elaboration of a <code><a href="../AA-3/AA-3.8#S0067">record_definition</a></code> consists of the elaboration of its <code><a href="../AA-3/AA-3.8#S0068">component_list</a></code>, if any.<br /></p>

<MarginText>17</MarginText>
<p>The elaboration of a <code><a href="../AA-3/AA-3.8#S0068">component_list</a></code> consists of the elaboration of the <code><a href="../AA-3/AA-3.8#S0069">component_item</a></code>s and <code><a href="../AA-3/AA-3.8#S0071">variant_part</a></code>, if any, in the order in which they appear. The elaboration of a <code><a href="../AA-3/AA-3.8#S0070">component_declaration</a></code> consists of the elaboration of the <code><a href="../AA-3/AA-3.6#S0056">component_definition</a></code>. <br /></p>

<MarginText>17.a</MarginText>
<Admonition type="aarm" aarm="discussion" title="Discussion: ">
<strong></strong>If the <code><a href="../AA-3/AA-3.3#S0033">defining_identifier_list</a></code> has more than one <code><a href="../AA-3/AA-3.1#S0022">defining_identifier</a></code>, we presume here that the transformation explained in <a href="../AA-3/AA-3.3#Subclause_3.3.1">3.3.1</a> has already taken place. Alternatively, we could say that the <code><a href="../AA-3/AA-3.6#S0056">component_definition</a></code> is elaborated once for each <code><a href="../AA-3/AA-3.1#S0022">defining_identifier</a></code> in the list. <br /></Admonition>

<MarginText>18/2</MarginText>
<MarginInfo items={["AI95-00171-01", "AI95-00230-01"]} />
<p>{"{"}<em>8652/0002</em>{"}"} {"{"}<em>{"AI95-00171-01"}</em>{"}"} {"{"}<em>{"AI95-00230-01"}</em>{"}"} <br />Within the definition of a composite type, if a <code><a href="../AA-3/AA-3.6#S0056">component_definition</a></code> or <code><a href="../AA-3/AA-3.6#S0055">discrete_subtype_definition</a></code> (see <a href="../AA-9/AA-9.5#Subclause_9.5.2">9.5.2</a>) includes a <code><a href="../AA-4/AA-4.1#S0091">name</a></code> that denotes a discriminant of the type, or that is an <code><a href="../AA-4/AA-4.1#S0100">attribute_reference</a></code> whose <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> denotes the current instance of the type, the expression containing the <code><a href="../AA-4/AA-4.1#S0091">name</a></code> is called a <em>per-object expression</em>, and the <code><a href="../AA-3/AA-3.2#S0029">constraint</a></code> or <code><a href="../AA-3/AA-3.5#S0037">range</a></code> being defined is called a <em>per-object constraint</em>. For the elaboration of a <code><a href="../AA-3/AA-3.6#S0056">component_definition</a></code> of a <code><a href="../AA-3/AA-3.8#S0070">component_declaration</a></code> or the <code><a href="../AA-3/AA-3.6#S0055">discrete_subtype_definition</a></code> of an <code><a href="../AA-9/AA-9.5#S0257">entry_declaration</a></code> for an entry family (see <a href="../AA-9/AA-9.5#Subclause_9.5.2">9.5.2</a>), if the component subtype is defined by an <code><a href="../AA-3/AA-3.10#S0084">access_definition</a></code> or if the <code><a href="../AA-3/AA-3.2#S0029">constraint</a></code> or <code><a href="../AA-3/AA-3.5#S0037">range</a></code> of the <code><a href="../AA-3/AA-3.2#S0027">subtype_indication</a></code> or <code><a href="../AA-3/AA-3.6#S0055">discrete_subtype_definition</a></code> is not a per-object constraint, then the <code><a href="../AA-3/AA-3.10#S0084">access_definition</a></code>, <code><a href="../AA-3/AA-3.2#S0027">subtype_indication</a></code>, or <code><a href="../AA-3/AA-3.6#S0055">discrete_subtype_definition</a></code> is elaborated. On the other hand, if the <code><a href="../AA-3/AA-3.2#S0029">constraint</a></code> or <code><a href="../AA-3/AA-3.5#S0037">range</a></code> is a per-object constraint, then the elaboration consists of the evaluation of any included expression that is not part of a per-object expression. Each such expression is evaluated once unless it is part of a named association in a discriminant constraint, in which case it is evaluated once for each associated discriminant.<br /></p>

<MarginText>18.1/1</MarginText>
<MarginInfo items={["AI95-00171-01"]} />
<p>{"{"}<em>8652/0002</em>{"}"} {"{"}<em>{"AI95-00171-01"}</em>{"}"} <br />When a per-object constraint is elaborated [(as part of creating an object)], each per-object expression of the constraint is evaluated. For other expressions, the values determined during the elaboration of the <code><a href="../AA-3/AA-3.6#S0056">component_definition</a></code> or <code><a href="../AA-9/AA-9.5#S0257">entry_declaration</a></code> are used. Any checks associated with the enclosing <code><a href="../AA-3/AA-3.2#S0027">subtype_indication</a></code> or <code><a href="../AA-3/AA-3.6#S0055">discrete_subtype_definition</a></code> are performed[, including the subtype compatibility check (see <a href="../AA-3/AA-3.2#Subclause_3.2.2">3.2.2</a>),] and the associated subtype is created. <br /></p>

<MarginText>18.a</MarginText>
<Admonition type="aarm" aarm="discussion" title="Discussion: ">
<strong></strong>The evaluation of other expressions that appear in <code><a href="../AA-3/AA-3.6#S0056">component_definition</a></code>s and <code><a href="../AA-3/AA-3.6#S0055">discrete_subtype_definition</a></code>s is performed when the type definition is elaborated. The evaluation of expressions that appear as <code><a href="../AA-3/AA-3.7#S0063">default_expression</a></code>s is postponed until an object is created. Expressions in representation items that appear within a composite type definition are evaluated according to the rules of the particular representation item. <br /></Admonition>

#### Extensions to Ada 83

<MarginText>31.a</MarginText>
<Admonition type="aarm" aarm="note" title="Note: ">
The syntax rule for <code><a href="../AA-3/AA-3.8#S0070">component_declaration</a></code> is modified to use <code><a href="../AA-3/AA-3.6#S0056">component_definition</a></code> (instead of <code>component_subtype_definition</code>). The effect of this change is to allow the reserved word <strong>aliased</strong> before the <code>component_subtype_definition</code>.<br /></Admonition>

<MarginText>31.b</MarginText>
<Admonition type="aarm" aarm="note" title="Note: ">
A short-hand is provided for defining a null record type (and a null record extension), as these will be more common for abstract root types (and derived types without additional components).<br /></Admonition>

<MarginText>31.c</MarginText>
<Admonition type="aarm" aarm="note" title="Note: ">
The syntax rule for <code><a href="../AA-3/AA-3.8#S0066">record_type_definition</a></code> is modified to allow the reserved words <strong>tagged</strong> and <strong>limited</strong>. Tagging is new. Limitedness is now orthogonal to privateness. In Ada 83 the syntax implied that limited private was sort of more private than private. However, limitedness really has nothing to do with privateness; limitedness simply indicates the lack of assignment capabilities, and makes perfect sense for nonprivate types such as record types. <br /></Admonition>


#### Wording Changes from Ada 83

<MarginText>31.d/1</MarginText>
<MarginInfo items={["AI95-00137-01"]} />
<Admonition type="aarm" aarm="note" title="Note: ">
{"{"}<em>8652/0009</em>{"}"} {"{"}<em>{"AI95-00137-01"}</em>{"}"} <br />The syntax rules now allow <code><a href="../AA-13/AA-13.1#S0343">aspect_clause</a></code>s to appear in a <code><a href="../AA-3/AA-3.8#S0067">record_definition</a></code>. This is not a language extension, because Legality Rules prevent all language-defined representation clauses from appearing there. ## 3.8.1  Variant Parts and Discrete Choices

<MarginText>1</MarginText>
<p>A record type with a <code><a href="../AA-3/AA-3.8#S0071">variant_part</a></code> specifies alternative lists of components. Each <code><a href="../AA-3/AA-3.8#S0072">variant</a></code> defines the components for the value or values of the discriminant covered by its <code><a href="../AA-3/AA-3.8#S0073">discrete_choice_list</a></code>. <br /></p>

<MarginText>1.a</MarginText>
<Admonition type="aarm" aarm="discussion" title="Discussion: ">
<strong></strong><code><a href="../AA-3/AA-3.8#S0073">Discrete_choice_list</a></code>s and <code><a href="../AA-3/AA-3.8#S0074">discrete_choice</a></code>s are said to <em>cover</em> values as defined below; which <code><a href="../AA-3/AA-3.8#S0073">discrete_choice_list</a></code> covers a value determines which of various alternatives is chosen. These are used in <code><a href="../AA-3/AA-3.8#S0071">variant_part</a></code>s, <code><a href="../AA-4/AA-4.3#S0113">array_aggregate</a></code>s, and <code><a href="../AA-5/AA-5.4#S0176">case_statement</a></code>s. <br /></Admonition>


#### Language Design Principles

<MarginText>1.b</MarginText>
<Admonition type="aarm" aarm="note" title="Note: ">
The definition of &ldquo;cover&rdquo; in this subclause and the rules about discrete choices are designed so that they are also appropriate for array aggregates and case statements.<br /></Admonition>

<MarginText>1.c</MarginText>
<Admonition type="aarm" aarm="note" title="Note: ">
The rules of this subclause intentionally parallel those for case statements. <br /></Admonition>

#### Name Resolution Rules

<MarginText>6</MarginText>
<p>The <em>discriminant_</em><code><a href="../AA-4/AA-4.1#S0092">direct_name</a></code> shall resolve to denote a discriminant (called the <em>discriminant of the <code><a href="../AA-3/AA-3.8#S0071">variant_part</a></code></em>) specified in the <code><a href="../AA-3/AA-3.7#S0061">known_discriminant_part</a></code> of the <code><a href="../AA-3/AA-3.2#S0024">full_type_declaration</a></code> that contains the <code><a href="../AA-3/AA-3.8#S0071">variant_part</a></code>. The expected type for each <code><a href="../AA-3/AA-3.8#S0074">discrete_choice</a></code> in a <code><a href="../AA-3/AA-3.8#S0072">variant</a></code> is the type of the discriminant of the <code><a href="../AA-3/AA-3.8#S0071">variant_part</a></code>. <br /></p>

<MarginText>6.a</MarginText>
<Admonition type="aarm" aarm="ramification" title="Ramification: ">
<strong></strong>A <code><a href="../AA-3/AA-3.2#S0024">full_type_declaration</a></code> with a <code><a href="../AA-3/AA-3.8#S0071">variant_part</a></code> has to have a (new) <code><a href="../AA-3/AA-3.7#S0061">known_discriminant_part</a></code>; the discriminant of the <code><a href="../AA-3/AA-3.8#S0071">variant_part</a></code> cannot be an inherited discriminant. <br /></Admonition>


#### Legality Rules

<MarginText>7</MarginText>
<p>The discriminant of the <code><a href="../AA-3/AA-3.8#S0071">variant_part</a></code> shall be of a discrete type. <br /></p>

<MarginText>7.a</MarginText>
<Admonition type="aarm" aarm="ramification" title="Ramification: ">
<strong></strong>It shall not be of an access type, named or anonymous.<br /></Admonition>

<MarginText>8/3</MarginText>
<MarginInfo items={["AI05-0153-3"]} />
<p>{"{"}<em>{"AI05-0153-3"}</em>{"}"} <br />The <code><a href="../AA-4/AA-4.4#S0133">choice_expression</a></code>s, <code><a href="../AA-3/AA-3.2#S0027">subtype_indication</a></code>s, and <code><a href="../AA-3/AA-3.5#S0037">range</a></code>s given as <code><a href="../AA-3/AA-3.8#S0074">discrete_choice</a></code>s in a <code><a href="../AA-3/AA-3.8#S0071">variant_part</a></code> shall be static. The <code><a href="../AA-3/AA-3.8#S0074">discrete_choice</a></code> <strong>others</strong> shall appear alone in a <code><a href="../AA-3/AA-3.8#S0073">discrete_choice_list</a></code>, and such a <code><a href="../AA-3/AA-3.8#S0073">discrete_choice_list</a></code>, if it appears, shall be the last one in the enclosing construct.<br /></p>

<MarginText>9</MarginText>
<p>A <code><a href="../AA-3/AA-3.8#S0074">discrete_choice</a></code> is defined to <em>cover a value</em> in the following cases: <br /></p>

<MarginText>10/3</MarginText>
<MarginInfo items={["AI05-0262-1"]} />
<ul>
<li>{"{"}<em>{"AI05-0262-1"}</em>{"}"} <br />A <code><a href="../AA-3/AA-3.8#S0074">discrete_choice</a></code> that is a <code><a href="../AA-4/AA-4.4#S0133">choice_expression</a></code> covers a value if the value equals the value of the <code><a href="../AA-4/AA-4.4#S0133">choice_expression</a></code> converted to the expected type.<br /></li>
<MarginText>10.1/4</MarginText>
<MarginInfo items={["AI05-0153-3", "AI05-0262-1", "AI12-0071-1"]} />
<li>{"{"}<em>{"AI05-0153-3"}</em>{"}"} {"{"}<em>{"AI05-0262-1"}</em>{"}"} {"{"}<em>{"AI12-0071-1"}</em>{"}"} <br />A <code><a href="../AA-3/AA-3.8#S0074">discrete_choice</a></code> that is a <code><a href="../AA-3/AA-3.2#S0027">subtype_indication</a></code> covers all values (possibly none) that belong to the subtype and that satisfy the static predicates of the subtype (see <a href="../AA-3/AA-3.2#Subclause_3.2.4">3.2.4</a>).<br /></li>
</ul>
<MarginText>10.a/3</MarginText>
<MarginInfo items={["AI05-0262-1"]} />
<Admonition type="aarm" aarm="ramification" title="Ramification: ">
<strong></strong>{"{"}<em>{"AI05-0262-1"}</em>{"}"} <br />A dynamic predicate is never allowed in this case (for <code><a href="../AA-3/AA-3.8#S0072">variant</a></code>s, <code><a href="../AA-5/AA-5.4#S0176">case_statement</a></code>s, and <code><a href="../AA-4/AA-4.5#S0151">case_expression</a></code>s, a subtype with a dynamic predicate isn't static and thus isn't allowed in a <code><a href="../AA-3/AA-3.8#S0074">discrete_choice</a></code>, and for a choice in an <code><a href="../AA-4/AA-4.3#S0113">array_aggregate</a></code>, a dynamic predicate is explicitly disallowed &mdash; see <a href="../AA-3/AA-3.2#Subclause_3.2.4">3.2.4</a>). <br /></Admonition>

<MarginText>11/3</MarginText>
<MarginInfo items={["AI05-0153-3"]} />
<ul>
<li>{"{"}<em>{"AI05-0153-3"}</em>{"}"} <br />A <code><a href="../AA-3/AA-3.8#S0074">discrete_choice</a></code> that is a <code><a href="../AA-3/AA-3.5#S0037">range</a></code> covers all values (possibly none) that belong to the range.<br /></li>
<MarginText>12</MarginText>
<li>The <code><a href="../AA-3/AA-3.8#S0074">discrete_choice</a></code> <strong>others</strong> covers all values of its expected type that are not covered by previous <code><a href="../AA-3/AA-3.8#S0073">discrete_choice_list</a></code>s of the same construct. <br /></li>
</ul>
<MarginText>12.a</MarginText>
<Admonition type="aarm" aarm="ramification" title="Ramification: ">
<strong></strong>For <code><a href="../AA-5/AA-5.4#S0176">case_statement</a></code>s, this includes values outside the range of the static subtype (if any) to be covered by the choices. It even includes values outside the base range of the case expression's type, since values of numeric types (and undefined values of any scalar type?) can be outside their base range. <br /></Admonition>

<MarginText>13</MarginText>
<p>A <code><a href="../AA-3/AA-3.8#S0073">discrete_choice_list</a></code> covers a value if one of its <code><a href="../AA-3/AA-3.8#S0074">discrete_choice</a></code>s covers the value.<br /></p>

<MarginText>14</MarginText>
<p>The possible values of the discriminant of a <code><a href="../AA-3/AA-3.8#S0071">variant_part</a></code> shall be covered as follows: <br /></p>

<MarginText>15/4</MarginText>
<MarginInfo items={["AI05-0153-3", "AI05-0188-1", "AI05-0262-1", "AI12-0071-1"]} />
<ul>
<li>{"{"}<em>{"AI05-0153-3"}</em>{"}"} {"{"}<em>{"AI05-0188-1"}</em>{"}"} {"{"}<em>{"AI05-0262-1"}</em>{"}"} {"{"}<em>{"AI12-0071-1"}</em>{"}"} <br />If the discriminant is of a static constrained scalar subtype then, except within an instance of a generic unit, each non-<strong>others</strong> <code><a href="../AA-3/AA-3.8#S0074">discrete_choice</a></code> shall cover only values in that subtype that satisfy its predicates, and each value of that subtype that satisfies its predicates shall be covered by some <code><a href="../AA-3/AA-3.8#S0074">discrete_choice</a></code> [(either explicitly or by <strong>others</strong>)];<br /></li>
</ul>
<MarginText>15.a/3</MarginText>
<MarginInfo items={["AI05-0188-1"]} />
<Admonition type="aarm" aarm="reason" title="Reason: ">
<strong></strong>{"{"}<em>{"AI05-0188-1"}</em>{"}"} <br />The exemption for a discriminated type declared in an instance allows the following example: <br /></Admonition>

<MarginText>15.b/3</MarginText>
<CodeBlock language="ada">
generic{"\n"}   type T is new Integer;{"\n"}package G is{"\n"}   type Rec (Discrim : T) is record{"\n"}      case Discrim is{"\n"}         when -10 .. -1 ={">"}{"\n"}            Foo : Float;{"\n"}         when others ={">"}{"\n"}            null;{"\n"}      end case;{"\n"}   end record;{"\n"}end G;{"\n"}
<MarginText>15.c/3</MarginText>
package I is new G (Natural); -- Legal{"\n"}

</CodeBlock>
<MarginText>16/3</MarginText>
<MarginInfo items={["AI05-0264-1"]} />
<ul>
<li>{"{"}<em>{"AI05-0264-1"}</em>{"}"} <br />If the type of the discriminant is a descendant of a generic formal scalar type, then the <code><a href="../AA-3/AA-3.8#S0071">variant_part</a></code> shall have an <strong>others</strong> <code><a href="../AA-3/AA-3.8#S0074">discrete_choice</a></code>; <br /></li>
</ul>
<MarginText>16.a</MarginText>
<Admonition type="aarm" aarm="reason" title="Reason: ">
<strong></strong>The base range is not known statically in this case. <br /></Admonition>

<MarginText>17</MarginText>
<ul>
<li>Otherwise, each value of the base range of the type of the discriminant shall be covered [(either explicitly or by <strong>others</strong>)]. <br /></li>
</ul>
<MarginText>18</MarginText>
<p>Two distinct <code><a href="../AA-3/AA-3.8#S0074">discrete_choice</a></code>s of a <code><a href="../AA-3/AA-3.8#S0071">variant_part</a></code> shall not cover the same value.<br /></p>


#### Static Semantics

<MarginText>19</MarginText>
<p>If the <code><a href="../AA-3/AA-3.8#S0068">component_list</a></code> of a <code><a href="../AA-3/AA-3.8#S0072">variant</a></code> is specified by <strong>null</strong>, the variant has no components.<br /></p>

<MarginText>20</MarginText>
<p>The discriminant of a <code><a href="../AA-3/AA-3.8#S0071">variant_part</a></code> is said to <em>govern</em> the <code><a href="../AA-3/AA-3.8#S0071">variant_part</a></code> and its <code><a href="../AA-3/AA-3.8#S0072">variant</a></code>s. In addition, the discriminant of a derived type governs a <code><a href="../AA-3/AA-3.8#S0071">variant_part</a></code> and its <code><a href="../AA-3/AA-3.8#S0072">variant</a></code>s if it corresponds (see <a href="../AA-3/AA-3.7">3.7</a>) to the discriminant of the <code><a href="../AA-3/AA-3.8#S0071">variant_part</a></code>.<br /></p>


#### Dynamic Semantics

<MarginText>21</MarginText>
<p>A record value contains the values of the components of a particular <code><a href="../AA-3/AA-3.8#S0072">variant</a></code> only if the value of the discriminant governing the <code><a href="../AA-3/AA-3.8#S0072">variant</a></code> is covered by the <code><a href="../AA-3/AA-3.8#S0073">discrete_choice_list</a></code> of the <code><a href="../AA-3/AA-3.8#S0072">variant</a></code>. This rule applies in turn to any further <code><a href="../AA-3/AA-3.8#S0072">variant</a></code> that is, itself, included in the <code><a href="../AA-3/AA-3.8#S0068">component_list</a></code> of the given <code><a href="../AA-3/AA-3.8#S0072">variant</a></code>.<br /></p>

<MarginText>21.1/3</MarginText>
<MarginInfo items={["AI05-0290-1"]} />
<p>{"{"}<em>{"AI05-0290-1"}</em>{"}"} <br />When an object of a discriminated type <em>T</em> is initialized by default, Constraint_Error is raised if no <code><a href="../AA-3/AA-3.8#S0073">discrete_choice_list</a></code> of any <code><a href="../AA-3/AA-3.8#S0072">variant</a></code> of a <code><a href="../AA-3/AA-3.8#S0071">variant_part</a></code> of <em>T</em> covers the value of the discriminant that governs the <code><a href="../AA-3/AA-3.8#S0071">variant_part</a></code>. When a <code><a href="../AA-3/AA-3.8#S0071">variant_part</a></code> appears in the <code><a href="../AA-3/AA-3.8#S0068">component_list</a></code> of another <code><a href="../AA-3/AA-3.8#S0072">variant</a></code> <em>V</em>, this test is only applied if the value of the discriminant governing <em>V</em> is covered by the <code><a href="../AA-3/AA-3.8#S0073">discrete_choice_list</a></code> of <em>V</em>. <br /></p>

<MarginText>21.a/3</MarginText>
<Admonition type="aarm" aarm="implementation-note" title="Implementation Note: ">
<strong></strong>This is not a &ldquo;check&rdquo;; it cannot be suppressed. However, in most cases it is not necessary to generate any code to raise this exception. A test is needed (and can fail) in the case where the discriminant subtype has a Static_Predicate specified, it also has predicate checking disabled, and the discriminant governs a <code><a href="../AA-3/AA-3.8#S0071">variant_part</a></code> which lacks a <strong>when others</strong> choice.<br /></Admonition>

<MarginText>21.b/3</MarginText>
<Admonition type="aarm" aarm="note" title="Note: ">
The test also could fail for a static discriminant subtype with range checking suppressed and the discriminant governs a <code><a href="../AA-3/AA-3.8#S0071">variant_part</a></code> which lacks a <strong>when others</strong> choice. But execution is erroneous if a range check that would have failed is suppressed (see <a href="../AA-11/AA-11.5">11.5</a>), so an implementation does not have to generate code to check this case. (An unchecked failed predicate does not cause erroneous execution, so the test is required in that case.)<br /></Admonition>

<MarginText>21.c/3</MarginText>
<Admonition type="aarm" aarm="note" title="Note: ">
Like the checks associated with a per-object constraint, this test is not made during the elaboration of a <code><a href="../AA-3/AA-3.2#S0027">subtype_indication</a></code>. <br /></Admonition>

<MarginText>22</MarginText>
<p>The elaboration of a <code><a href="../AA-3/AA-3.8#S0071">variant_part</a></code> consists of the elaboration of the <code><a href="../AA-3/AA-3.8#S0068">component_list</a></code> of each <code><a href="../AA-3/AA-3.8#S0072">variant</a></code> in the order in which they appear. <br /></p>

</CodeBlock>

</CodeBlock>

#### Extensions to Ada 83

<MarginText>29.a</MarginText>
<Admonition type="aarm" aarm="note" title="Note: ">
In Ada 83, the discriminant of a <code><a href="../AA-3/AA-3.8#S0071">variant_part</a></code> is not allowed to be of a generic formal type. This restriction is removed in Ada 95; an <strong>others</strong> <code><a href="../AA-3/AA-3.8#S0074">discrete_choice</a></code> is required in this case. <br /></Admonition>


#### Wording Changes from Ada 83

<MarginText>29.b</MarginText>
<Admonition type="aarm" aarm="note" title="Note: ">
The syntactic category <code>choice</code> is removed. The syntax rules for <code><a href="../AA-3/AA-3.8#S0072">variant</a></code>, <code><a href="../AA-4/AA-4.3#S0113">array_aggregate</a></code>, and <code><a href="../AA-5/AA-5.4#S0176">case_statement</a></code> now use <code><a href="../AA-3/AA-3.8#S0073">discrete_choice_list</a></code> or <code><a href="../AA-3/AA-3.8#S0074">discrete_choice</a></code> instead. #### Incompatibilities With Ada 2005

<MarginText>29.e/3</MarginText>
<MarginInfo items={["AI05-0158-1"]} />
<Admonition type="aarm" aarm="note" title="Note: ">
{"{"}<em>{"AI05-0158-1"}</em>{"}"} <br />Membership tests are no longer allowed as a <code><a href="../AA-3/AA-3.8#S0074">discrete_choice</a></code>, in order that those tests can be expanded to allow multiple tests in a single expression without ambiguity. Since a membership test has a boolean type, they are very unlikely to be used as a <code><a href="../AA-3/AA-3.8#S0074">discrete_choice</a></code>. <br /></Admonition>
