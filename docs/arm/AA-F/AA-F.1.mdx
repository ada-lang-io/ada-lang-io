#### Static Semantics

<MarginText>1/5</MarginText>
<MarginInfo items={["AI12-0272-1"]} />
<p>The representation attribute Machine{"_"}Radix may be specified for a decimal first subtype (see <a href="/docs/arm/AA-3/AA-3.5#Subclause_3.5.9">3.5.9</a>) via an <code><a href="/docs/arm/AA-13/AA-13.3#S0349">attribute_definition_clause</a></code>; the expression of such a clause shall be static, and its value shall be 2 or 10. A value of 2 implies a binary base range; a value of 10 implies a decimal base range. <br /></p>

<AnnotatedOnly>
<MarginText>1.a</MarginText>
<Admonition type="aarm" aarm="ramification">
<b></b>In the absence of a Machine{"_"}Radix clause, the choice of 2 versus 10 for S'Machine{"_"}Radix is not specified. <br />
</Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>1.b/3</MarginText>
<Admonition type="aarm" aarm="note">
<b>Aspect Description for </b><b>Machine{"_"}Radix: </b>Radix (2 or 10) that is used to represent a decimal fixed point type.<br />
</Admonition>
</AnnotatedOnly>


#### Implementation Advice

<MarginText>2</MarginText>
<p>Packed decimal should be used as the internal representation for objects of subtype S when S'Machine{"_"}Radix = 10. <br /></p>

<AnnotatedOnly>
<MarginText>2.a/2</MarginText>
<Admonition type="aarm" aarm="implementation-advice">
<b></b>Packed decimal should be used as the internal representation for objects of subtype <i>S</i> when <i>S</i>'Machine{"_"}Radix = 10.<br />
</Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>2.b/3</MarginText>
<MarginInfo items={["AI05-0229-1"]} />
<Admonition type="aarm" aarm="discussion">
<b></b>The intent of a decimal Machine{"_"}Radix attribute definition clause is to allow the programmer to declare an Ada decimal data object whose representation matches a particular COBOL implementation's representation of packed decimal items. The Ada object may then be passed to an interfaced COBOL program that takes a packed decimal data item as a parameter, assuming that convention COBOL has been specified for the Ada object's type with an aspect Convention.<br />
</Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>2.c</MarginText>
<Admonition type="aarm" aarm="note">
Additionally, the Ada compiler may choose to generate arithmetic instructions that exploit the packed decimal representation.<br />
</Admonition>
</AnnotatedOnly>


#### Examples

<MarginText>3</MarginText>
<p><i>Example of Machine{"_"}Radix attribute definition clause:</i> <br /></p>

<MarginText>4</MarginText>
<CodeBlock language="ada">
type Money is delta 0.01 digits 15;{"\n"}for Money'Machine{"_"}Radix use 10;{"\n"}

</CodeBlock>
