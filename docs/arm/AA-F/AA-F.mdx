<MarginText>1</MarginText>
<p>This Annex provides a set of facilities relevant to Information Systems programming. These fall into several categories: <br /></p>

<MarginText>2</MarginText>
<ul>
<li>an attribute definition clause specifying Machine{"_"}Radix for a decimal subtype;<br /></li>
<MarginText>3</MarginText>
<li>the package Decimal, which declares a set of constants defining the implementation's capacity for decimal types, and a generic procedure for decimal division; and<br /></li>
<MarginText>4/2</MarginText>
<MarginInfo items={["AI95-00285-01"]} />
<li>the child packages Text{"_"}IO.Editing, Wide{"_"}Text{"_"}IO.Editing, and Wide{"_"}Wide{"_"}Text{"_"}IO.Editing, which support formatted and localized output of decimal data, based on &ldquo;picture String&rdquo; values. <br /></li>
</ul>
<MarginText>5/5</MarginText>
<MarginInfo items={["AI95-00434-01", "AI12-0449-1"]} />
<p>Other relevant facilities can be found in: <a href="/docs/arm/AA-3/AA-3.5#Subclause_3.5.9">3.5.9</a>, &ldquo;<a href="/docs/arm/AA-3/AA-3.5#Subclause_3.5.9">Fixed Point Types</a>&rdquo;; <a href="/docs/arm/AA-3/AA-3.5#Subclause_3.5.10">3.5.10</a>, &ldquo;<a href="/docs/arm/AA-3/AA-3.5#Subclause_3.5.10">Operations of Fixed Point Types</a>&rdquo;; <a href="/docs/arm/AA-4/AA-4.6">4.6</a>, &ldquo;<a href="/docs/arm/AA-4/AA-4.6">Type Conversions</a>&rdquo;; <a href="/docs/arm/AA-13/AA-13.3">13.3</a>, &ldquo;<a href="/docs/arm/AA-13/AA-13.3">Operational and Representation Attributes</a>&rdquo;; <a href="/docs/arm/AA-A/AA-A.10#Subclause_A.10.9">A.10.9</a>, &ldquo;<a href="/docs/arm/AA-A/AA-A.10#Subclause_A.10.9">Input-Output for Real Types</a>&rdquo;; <a href="/docs/arm/AA-B/AA-B.3">B.3</a>, &ldquo;<a href="/docs/arm/AA-B/AA-B.3">Interfacing with C and C++</a>&rdquo;; <a href="/docs/arm/AA-B/AA-B.4">B.4</a>, &ldquo;<a href="/docs/arm/AA-B/AA-B.4">Interfacing with COBOL</a>&rdquo;; <a href="/docs/arm/AA-G/">Annex G</a>, &ldquo;<a href="/docs/arm/AA-G/">Numerics</a>&rdquo;.<br /></p>

<p>The character and string handling packages in <a href="/docs/arm/AA-A/">Annex A</a>, &ldquo;<a href="/docs/arm/AA-A/">Predefined Language Environment</a>&rdquo; are also relevant for Information Systems. <br /></p>


#### Implementation Advice

<MarginText>7/3</MarginText>
<MarginInfo items={["AI05-0229-1"]} />
<p>If COBOL (respectively, C) is widely supported in the target environment, implementations supporting the Information Systems Annex should provide the child package Interfaces.COBOL (respectively, Interfaces.C) specified in <a href="/docs/arm/AA-B/">Annex B</a> and should support a <i>convention{"_"}</i><code><a href="/docs/arm/AA-2/AA-2.3#S0002">identifier</a></code> of COBOL (respectively, C) for the Convention aspect (see <a href="/docs/arm/AA-B/">Annex B</a>), thus allowing Ada programs to interface with programs written in that language. <br /></p>

<AnnotatedOnly>
<MarginText>7.a/2</MarginText>
<Admonition type="aarm" aarm="implementation-advice">
<b></b>If COBOL (respectively, C) is supported in the target environment, then interfacing to COBOL (respectively, C) should be supported as specified in <a href="/docs/arm/AA-B/">Annex B</a>.<br />
</Admonition>
</AnnotatedOnly>
