<AnnotatedOnly>
<MarginText>
7/5
</MarginText>

<p>{"{"}<em>{"AI12-0079-3"}</em>{"}"} {"{"}<em>{"AI12-0431-1"}</em>{"}"} {"{"}<em>{"AI12-0439-1"}</em>{"}"} <br />The Global aspect for a subtype identifies the global variables that can be referenced during default initialization, adjustment as part of assignment, finalization of an object of the subtype, or conversion to the subtype, including the evaluation of any assertion expressions that apply. If not specified for the first subtype of a derived type, the aspect defaults to that of the ancestor subtype; if not specified for a nonderived composite first subtype the aspect defaults to that of the enclosing library unit; if not specified for a nonderived elementary first subtype (or scalar base subtype), the aspect defaults to <strong>null</strong> in the absence of a predicate (or when the predicate is statically True), and to that of the enclosing library unit otherwise. If not specified for a nonfirst subtype <em>S</em>, the Global aspect defaults to that of the subtype identified in the <code><a href="../AA-3/AA-3.2#S0027">subtype_indication</a></code> defining <em>S</em>.<br /></p>

<AnnotatedOnly>
<MarginText>
8/5
</MarginText>
</AnnotatedOnly>
<p>{"{"}<em>{"AI12-0079-3"}</em>{"}"} <br />The Global'Class aspect may be specified for the first subtype of a tagged type <em>T</em>, indicating an upper bound on the Global aspect of any descendant of <em>T</em>. If not specified, it defaults to Unspecified. <br /></p>

<AnnotatedOnly>
<MarginText>

#### Static Semantics

<AnnotatedOnly>
<MarginText>
13/5
</MarginText>
</AnnotatedOnly>
<p>{"{"}<em>{"AI12-0380-1"}</em>{"}"} {"{"}<em>{"AI12-0439-1"}</em>{"}"} <br />The <em>formal parameter set</em> is identified by a set of <code><a href="../AA-H/AA-H.7#S0364">formal_parameter_name</a></code>s. Alternatively, the reserved word <strong>null</strong> may be used to indicate none of the generic formal parameters, or <strong>all</strong> to indicate all of the generic formal parameters, of any enclosing generic unit (or visible formal package) can be used within the execution of the operation. If there is no formal parameter set specified for an entity declared within a generic unit, it defaults to <strong>all</strong>.<br /></p>

<AnnotatedOnly>
<MarginText>
14/5
</MarginText>
</AnnotatedOnly>
<p>{"{"}<em>{"AI12-0380-1"}</em>{"}"} {"{"}<em>{"AI12-0404-1"}</em>{"}"} {"{"}<em>{"AI12-0444-1"}</em>{"}"} <br />The <em>dispatching operation set</em> is identified by a set of <code><a href="../AA-H/AA-H.7#S0366">dispatching_operation_specifier</a></code>s. It indicates that the Nonblocking and global effects of dispatching calls that match one of the specifiers, rather than being accounted for by the Nonblocking or global aspect, are instead to be accounted for by the invoker of the operation. A dispatching call matches a <code><a href="../AA-H/AA-H.7#S0366">dispatching_operation_specifier</a></code> if the <code><a href="../AA-4/AA-4.1#S0091">name</a></code> or <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> of the call statically denotes the same operation(s) as that of the <code><a href="../AA-H/AA-H.7#S0366">dispatching_operation_specifier</a></code>, and at least one of the objects controlling the call is denoted by, or designated by, a <code><a href="../AA-4/AA-4.1#S0091">name</a></code> that statically names the same object as that denoted by the <em>object_</em><code><a href="../AA-4/AA-4.1#S0091">name</a></code> of the <code><a href="../AA-H/AA-H.7#S0366">dispatching_operation_specifier</a></code>.<br /></p>

<AnnotatedOnly>
<MarginText>
14.a/5
</MarginText>
</AnnotatedOnly>
<Admonition type="aarm" aarm="ramification" title="Ramification: ">
<strong></strong>The object "controlling the call" is not necessarily a controlling parameter of the call if the call is a function with a controlling result or has parameters that is such a function. It is one of the objects that provide the dispatching tag used for the call; that could, for example, be a parameter of a function used as a parameter to the call, or an object being assigned to, or a parameter of an enclosing call. <br /></Admonition>

<AnnotatedOnly>
<MarginText>
15/5
</MarginText>
</AnnotatedOnly>
<p>{"{"}<em>{"AI12-0380-1"}</em>{"}"} {"{"}<em>{"AI12-0404-1"}</em>{"}"} <br />In the absence of any <code><a href="../AA-H/AA-H.7#S0366">dispatching_operation_specifier</a></code>s, or if none of them match a dispatching call <em>C</em> within an operation <em>P</em>, Nonblocking and global aspects checks are performed at the point of the call <em>C</em> within <em>P</em> using the Nonblocking and Global'Class aspects that apply to the dispatching operation named in call <em>C</em>. If there is a match, any global access or potential blocking within the subprogram body invoked by the call <em>C</em> is ignored at the point of call within <em>P</em>. Instead, when the operation <em>P</em> itself is invoked, Nonblocking and global aspect checks are performed presuming each named dispatching operation is called at least once (with the named object controlling the call), but similarly ignoring those dispatching calls that would match a <code><a href="../AA-H/AA-H.7#S0366">dispatching_operation_specifier</a></code> applicable at the point of invocation of <em>P</em>.<br /></p>


#### Legality Rules

<AnnotatedOnly>
<MarginText>
16/5
</MarginText>
</AnnotatedOnly>
<p>{"{"}<em>{"AI12-0380-1"}</em>{"}"} <br />Within an operation to which a Use_Formal aspect applies, if the formal parameter set is anything but <strong>all</strong>, then the only generic formal subtypes that may be used, the only formal subprograms that may be called, and the only formal objects of an anonymous access-to-subprogram type that may be dereferenced as part of a call or passed as the actual for an access parameter, are those included in the formal parameter set.<br /></p>

<AnnotatedOnly>
<MarginText>
17/5
</MarginText>
</AnnotatedOnly>
<p>{"{"}<em>{"AI12-0380-1"}</em>{"}"} <br />When an operation (or instance thereof) to which a Use_Formal aspect applies is invoked, Nonblocking and global aspect checks are performed presuming each generic formal parameter (or corresponding actual parameter) of the formal parameter set is used at least once. <br /></p>
