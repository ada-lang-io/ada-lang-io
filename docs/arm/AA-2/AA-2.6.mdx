<p>[A <code><a href="../AA-2/AA-2.6#S0016">string_literal</a></code> is formed by a sequence of graphic characters (possibly none) enclosed between two quotation marks used as string brackets. They are used to represent <code><a href="../AA-6/AA-6.1#S0202">operator_symbol</a></code>s (see <a href="../AA-6/AA-6.1">6.1</a>), values of a string type (see <a href="../AA-4/AA-4.2">4.2</a>), and array subaggregates (see <a href="../AA-4/AA-4.3#Subclause_4.3.3">4.3.3</a>). ]<br /></p>

<p>A <code><a href="../AA-2/AA-2.6#S0017">string_element</a></code> is either a pair of quotation marks (""), or a single <code>graphic_character</code> other than a quotation mark. <br /></p>


#### Static Semantics

<AnnotatedOnly>
<MarginText>
5
</MarginText>
</AnnotatedOnly>
<p>The <em>sequence of characters</em> of a <code><a href="../AA-2/AA-2.6#S0016">string_literal</a></code> is formed from the sequence of <code><a href="../AA-2/AA-2.6#S0017">string_element</a></code>s between the bracketing quotation marks, in the given order, with a <code><a href="../AA-2/AA-2.6#S0017">string_element</a></code> that is "" becoming a single quotation mark in the sequence of characters, and any other <code><a href="../AA-2/AA-2.6#S0017">string_element</a></code> being reproduced in the sequence.<br /></p>

<AnnotatedOnly>
<MarginText>
6
</MarginText>
</AnnotatedOnly>
<p>A <em>null string literal</em> is a <code><a href="../AA-2/AA-2.6#S0016">string_literal</a></code> with no <code><a href="../AA-2/AA-2.6#S0017">string_element</a></code>s between the quotation marks.<br /></p>

<AnnotatedOnly>
<MarginText>
7
</MarginText>
</AnnotatedOnly>
<Admonition type="aarm" aarm="note" title="Note: ">
NOTE 1   An end of line cannot appear in a <code><a href="../AA-2/AA-2.6#S0016">string_literal</a></code>.<br /></Admonition>

<AnnotatedOnly>
<MarginText>

</MarginText>
</AnnotatedOnly>

#### Wording Changes from Ada 83

<AnnotatedOnly>
<MarginText>
9.a
</MarginText>
</AnnotatedOnly>
<Admonition type="aarm" aarm="note" title="Note: ">
The wording has been changed to be strictly lexical. No mention is made of string or character values, since <code><a href="../AA-2/AA-2.6#S0016">string_literal</a></code>s are also used to represent <code><a href="../AA-6/AA-6.1#S0202">operator_symbol</a></code>s, which don't have a defined value.<br /></Admonition>

<AnnotatedOnly>
<MarginText>
9.b
</MarginText>
</AnnotatedOnly>
<Admonition type="aarm" aarm="note" title="Note: ">
The syntax is described differently. <br /></Admonition>

</MarginText>
