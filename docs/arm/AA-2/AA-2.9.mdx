<p class="Indented2">The following are the <i>reserved words</i>. Within a program, some or all of the letters of a reserved word may be in upper case. <br /></p>

<AnnotatedOnly>
<MarginText>2.a</MarginText>
<Admonition type="aarm" aarm="discussion">
<b></b>Reserved words have special meaning in the syntax. In addition, certain reserved words are used as attribute names.<br />
</Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>2.b</MarginText>
<Admonition type="aarm" aarm="note">
The syntactic category <code><a href="/docs/arm/AA-2/AA-2.3#S0002">identifier</a></code> no longer allows reserved words. We have added the few reserved words that are legal explicitly to the syntax for <code><a href="/docs/arm/AA-4/AA-4.1#S0100">attribute_reference</a></code>. Allowing identifier to include reserved words has been a source of confusion for some users, and differs from the way they are treated in the C and Pascal language definitions.<br />
</Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>2.c/5</MarginText>
<MarginInfo items={["AI12-0005-1"]} />
<Admonition type="aarm" aarm="note">
Reserved words are sometimes called <i>keywords</i>  in informal material. <br />
</Admonition>
</AnnotatedOnly>

#### Incompatibilities With Ada 83

</AnnotatedOnly>
<AnnotatedOnly>
<MarginText>3.a</MarginText>
<Admonition type="aarm" aarm="note">
The following words are not reserved in Ada 83, but are reserved in Ada 95: <b>abstract</b>, <b>aliased</b>, <b>protected</b>, <b>requeue</b>, <b>tagged</b>, <b>until</b>. <br />
</Admonition>
</AnnotatedOnly>

#### Incompatibilities With Ada 95

</AnnotatedOnly>
<AnnotatedOnly>
<MarginText>3.c/2</MarginText>
<MarginInfo items={["AI95-00284-02"]} />
<Admonition type="aarm" aarm="note">
The following words are not reserved in Ada 95, but are reserved in Ada 2005: <b>interface</b>, <b>overriding</b>, <b>synchronized</b>. A special allowance is made for <b>pragma</b> Interface (see <a href="/docs/arm/AA-J/AA-J.12">J.12</a>). Uses of these words as identifiers will need to be changed, but we do not expect them to be common. <br />
</Admonition>
</AnnotatedOnly>

<AnnotatedOnly>

#### Wording Changes from Ada 95

</AnnotatedOnly>
<AnnotatedOnly>
<MarginText>3.d/2</MarginText>
<MarginInfo items={["AI95-00395-01"]} />
<Admonition type="aarm" aarm="note">
The definition of upper case equivalence has been modified to allow identifiers using all of the characters of ISO 10646. This change has no effect on the character sequences that are reserved words, but does make some unusual sequences of characters illegal. <br />
</Admonition>
</AnnotatedOnly>

<AnnotatedOnly>

#### Incompatibilities With Ada 2005

</AnnotatedOnly>
<AnnotatedOnly>
<MarginText>3.e/3</MarginText>
<MarginInfo items={["AI05-0091-1"]} />
<Admonition type="aarm" aarm="correction">
<b></b> Removed <code>other{"_"}format</code> characters from reserved words in order to be compatible with the latest Unicode recommendations. This change can only affect programs written for original Ada 2005, and there is little reason to put <code>other{"_"}format</code> characters into reserved words in the first place, so there should be very few such programs.<br />
</Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>3.f/3</MarginText>
<MarginInfo items={["AI05-0176-1"]} />
<Admonition type="aarm" aarm="note">
The following word is not reserved in Ada 2005, but is reserved in Ada 2012: <b>some</b>. Uses of this word as an identifier will need to be changed, but we do not expect them to be common. <br />
</Admonition>
</AnnotatedOnly>
