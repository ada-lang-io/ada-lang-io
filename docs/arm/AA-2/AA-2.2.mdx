#### Static Semantics

<MarginText>1</MarginText>
<p>The text of a program consists of the texts of one or more <code><a href="../AA-10/AA-10.1#S0285">compilation</a></code>s. The text of each <code><a href="../AA-10/AA-10.1#S0285">compilation</a></code> is a sequence of separate <em>lexical elements</em>. Each lexical element is formed from a sequence of characters, and is either a delimiter, an <code><a href="../AA-2/AA-2.3#S0002">identifier</a></code>, a reserved word, a <code><a href="../AA-2/AA-2.4#S0006">numeric_literal</a></code>, a <code><a href="../AA-2/AA-2.5#S0015">character_literal</a></code>, a <code><a href="../AA-2/AA-2.6#S0016">string_literal</a></code>, or a comment. The meaning of a program depends only on the particular sequences of lexical elements that form its <code><a href="../AA-10/AA-10.1#S0285">compilation</a></code>s, excluding <code><a href="../AA-2/AA-2.7#S0018">comment</a></code>s.<br /></p>

<MarginText>2/3</MarginText>
<MarginInfo items={["AI95-00285-01", "AI05-0262-1"]} />
<p>{"{"}<em>{"AI95-00285-01"}</em>{"}"} {"{"}<em>{"AI05-0262-1"}</em>{"}"} <br />The text of a <code><a href="../AA-10/AA-10.1#S0285">compilation</a></code> is divided into <em>lines</em>. In general, the representation for an end of line is implementation defined. However, a sequence of one or more <code>format_effector</code>s other than the character whose code point is 16#09# (CHARACTER TABULATION) signifies at least one end of line. <br /></p>

<MarginText>2.a</MarginText>
<Admonition type="aarm" aarm="implementation-defined" title="Implementation defined: ">
<strong></strong>The representation for an end of line.<br /></Admonition>

<p>{"{"}<em>{"AI95-00285-01"}</em>{"}"} <br />[In some cases an explicit <em>separator</em> is required to separate adjacent lexical elements.] A separator is any of a <code>separator_space</code>, a <code>format_effector</code>, or the end of a line, as follows: <br /></p>

<MarginText>4/2</MarginText>
<MarginInfo items={["AI95-00285-01"]} />
<ul>
<li>{"{"}<em>{"AI95-00285-01"}</em>{"}"} <br />A <code>separator_space</code> is a separator except within a <code><a href="../AA-2/AA-2.7#S0018">comment</a></code>, a <code><a href="../AA-2/AA-2.6#S0016">string_literal</a></code>, or a <code><a href="../AA-2/AA-2.5#S0015">character_literal</a></code>.<br /></li>
<MarginText>5/3</MarginText>
<MarginInfo items={["AI95-00285-01", "AI05-0262-1"]} />
<li>{"{"}<em>{"AI95-00285-01"}</em>{"}"} {"{"}<em>{"AI05-0262-1"}</em>{"}"} <br />The character whose code point is 16#09# (CHARACTER TABULATION) is a separator except within a <code><a href="../AA-2/AA-2.7#S0018">comment</a></code>.<br /></li>
<MarginText>6</MarginText>
<li>The end of a line is always a separator. <br /></li>
</ul>
<MarginText>7</MarginText>
<p>One or more separators are allowed between any two adjacent lexical elements, before the first of each <code><a href="../AA-10/AA-10.1#S0285">compilation</a></code>, or after the last. At least one separator is required between an <code><a href="../AA-2/AA-2.3#S0002">identifier</a></code>, a reserved word, or a <code><a href="../AA-2/AA-2.4#S0006">numeric_literal</a></code> and an adjacent <code><a href="../AA-2/AA-2.3#S0002">identifier</a></code>, reserved word, or <code><a href="../AA-2/AA-2.4#S0006">numeric_literal</a></code>.<br /></p>

<MarginText>7.1/3</MarginText>
<MarginInfo items={["AI05-0079-1"]} />
