#### Static Semantics

<MarginText>2</MarginText>
<p>A <code><a href="../AA-13/AA-13.5#S0352">record_representation_clause</a></code> of the form: <br /></p>

<MarginText>3/3</MarginText>
<MarginInfo items={["AI05-0092-1"]} />
<CodeBlock language="ada">
for r use{"\n"}    record at mod a;{"\n"}        ...{"\n"}    end record;{"\n"}

</CodeBlock>
<MarginText>4</MarginText>
<p>is equivalent to: <br /></p>

<MarginText>5</MarginText>
<CodeBlock language="ada">
for r'Alignment use a;{"\n"}for r use{"\n"}    record{"\n"}        ...{"\n"}    end record;{"\n"}

</CodeBlock>
<AnnotatedOnly>
<MarginText>5.a</MarginText>
<Admonition type="aarm" aarm="reason">
<strong></strong>The preferred syntax for specifying the alignment of an entity is an <code><a href="../AA-13/AA-13.3#S0349">attribute_definition_clause</a></code> specifying the Alignment attribute. Therefore, the special-purpose <code><a href="../AA-J/AA-J.8#S0369">mod_clause</a></code> syntax is now obsolete.<br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>5.b</MarginText>
<Admonition type="aarm" aarm="note">
The above equivalence implies, for example, that it is illegal to give both a <code><a href="../AA-J/AA-J.8#S0369">mod_clause</a></code> and an <code><a href="../AA-13/AA-13.3#S0349">attribute_definition_clause</a></code> specifying the Alignment attribute for the same type. <br /></Admonition>
</AnnotatedOnly>

<AnnotatedOnly>

#### Wording Changes from Ada 83

</AnnotatedOnly>
<AnnotatedOnly>
<MarginText>5.c</MarginText>
<Admonition type="aarm" aarm="note">
Ada 83's <code>alignment_clause</code> is now called a <code><a href="../AA-J/AA-J.8#S0369">mod_clause</a></code> to avoid confusion with the new term &ldquo;Alignment clause&rdquo; (that is, an <code><a href="../AA-13/AA-13.3#S0349">attribute_definition_clause</a></code> for the Alignment attribute). <br /></Admonition>
</AnnotatedOnly>
