<MarginText>1</MarginText>
<p>A <code><a href="/docs/arm/AA-3/AA-3.5#S0050">digits_constraint</a></code> may be used to define a floating point subtype with a new value for its requested decimal precision, as reflected by its Digits attribute. Similarly, a <code><a href="/docs/arm/AA-J/AA-J.3#S0367">delta_constraint</a></code> may be used to define an ordinary fixed point subtype with a new value for its <i>delta</i>, as reflected by its Delta attribute. <br /></p>

<AnnotatedOnly>
<MarginText>1.a</MarginText>
<Admonition type="aarm" aarm="discussion">
<b></b>It might be more direct to make these attributes specifiable via an <code><a href="/docs/arm/AA-13/AA-13.3#S0349">attribute_definition_clause</a></code>, and eliminate the syntax for these <code>{"_"}constraint</code>s. <br />
</Admonition>
</AnnotatedOnly>

#### Legality Rules

<MarginText>4/4</MarginText>

<MarginText>6</MarginText>

#### Static Semantics

<MarginText>7/4</MarginText>
<MarginInfo items={["AI12-0152-1"]} />
<p>A <code><a href="/docs/arm/AA-3/AA-3.2#S0027">subtype_indication</a></code> with a <code><a href="/docs/arm/AA-3/AA-3.2#S0028">subtype_mark</a></code> that denotes an ordinary fixed point subtype and a <code><a href="/docs/arm/AA-J/AA-J.3#S0367">delta_constraint</a></code> defines an ordinary fixed point subtype with a <i>delta</i> given by the value of the <code><a href="/docs/arm/AA-4/AA-4.4#S0138">simple_expression</a></code> of the <code><a href="/docs/arm/AA-J/AA-J.3#S0367">delta_constraint</a></code>. If the <code><a href="/docs/arm/AA-J/AA-J.3#S0367">delta_constraint</a></code> includes a <code><a href="/docs/arm/AA-3/AA-3.5#S0036">range{"_"}constraint</a></code>, then the ordinary fixed point subtype is constrained by the <code><a href="/docs/arm/AA-3/AA-3.5#S0036">range{"_"}constraint</a></code>.<br /></p>

<MarginText>8/4</MarginText>
<MarginInfo items={["AI12-0152-1"]} />
<p>A <code><a href="/docs/arm/AA-3/AA-3.2#S0027">subtype_indication</a></code> with a <code><a href="/docs/arm/AA-3/AA-3.2#S0028">subtype_mark</a></code> that denotes a floating point subtype and a <code><a href="/docs/arm/AA-3/AA-3.5#S0050">digits_constraint</a></code> defines a floating point subtype with a requested decimal precision (as reflected by its Digits attribute) given by the value of the <code><a href="/docs/arm/AA-4/AA-4.4#S0138">simple_expression</a></code> of the <code><a href="/docs/arm/AA-3/AA-3.5#S0050">digits_constraint</a></code>. If the <code><a href="/docs/arm/AA-3/AA-3.5#S0050">digits_constraint</a></code> includes a <code><a href="/docs/arm/AA-3/AA-3.5#S0036">range{"_"}constraint</a></code>, then the floating point subtype is constrained by the <code><a href="/docs/arm/AA-3/AA-3.5#S0036">range{"_"}constraint</a></code>. <br /></p>


#### Dynamic Semantics

<MarginText>9/4</MarginText>
<MarginInfo items={["AI12-0152-1"]} />
<p>A <code><a href="/docs/arm/AA-J/AA-J.3#S0367">delta_constraint</a></code> is <i>compatible</i> with an ordinary fixed point subtype if the value of the <code><a href="/docs/arm/AA-4/AA-4.4#S0138">simple_expression</a></code> is no less than the <i>delta</i> of the subtype, and the <code><a href="/docs/arm/AA-3/AA-3.5#S0036">range_constraint</a></code>, if any, is compatible with the subtype.<br /></p>

<MarginText>10/4</MarginText>
<MarginInfo items={["AI12-0152-1"]} />
<p>A <code><a href="/docs/arm/AA-3/AA-3.5#S0050">digits_constraint</a></code> is <i>compatible</i> with a floating point subtype if the value of the <code><a href="/docs/arm/AA-4/AA-4.4#S0138">simple_expression</a></code> is no greater than the requested decimal precision of the subtype, and the <code><a href="/docs/arm/AA-3/AA-3.5#S0036">range_constraint</a></code>, if any, is compatible with the subtype.<br /></p>

<MarginText>11</MarginText>
<p>The elaboration of a <code><a href="/docs/arm/AA-J/AA-J.3#S0367">delta_constraint</a></code> consists of the elaboration of the <code><a href="/docs/arm/AA-3/AA-3.5#S0036">range_constraint</a></code>, if any. <br /></p>

<AnnotatedOnly>
<MarginText>11.a</MarginText>
<Admonition type="aarm" aarm="reason">
<b></b>A numeric subtype is considered &ldquo;constrained&rdquo; only if a range constraint applies to it. The only effect of a <code><a href="/docs/arm/AA-3/AA-3.5#S0050">digits_constraint</a></code> or a <code><a href="/docs/arm/AA-J/AA-J.3#S0367">delta_constraint</a></code> without a <code><a href="/docs/arm/AA-3/AA-3.5#S0036">range_constraint</a></code> is to specify the value of the corresponding Digits or Delta attribute in the new subtype. The set of values of the subtype is not &ldquo;constrained&rdquo; in any way by such <code>{"_"}constraint</code>s. <br />
</Admonition>
</AnnotatedOnly>

#### Wording Changes from Ada 83

</AnnotatedOnly>
<AnnotatedOnly>
<MarginText>11.b</MarginText>
<Admonition type="aarm" aarm="note">
In Ada 83, a <code><a href="/docs/arm/AA-J/AA-J.3#S0367">delta_constraint</a></code> is called a fixed{"_"}point{"_"}constraint, and a <code><a href="/docs/arm/AA-3/AA-3.5#S0050">digits_constraint</a></code> is called a floating{"_"}point{"_"}constraint. We have adopted other terms because <code><a href="/docs/arm/AA-3/AA-3.5#S0050">digits_constraint</a></code>s apply primarily to decimal fixed point types now (they apply to floating point types only as an obsolescent feature). <br />
</Admonition>
</AnnotatedOnly>
