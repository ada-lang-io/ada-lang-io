<MarginText>1/3</MarginText>
<MarginInfo items={["AI05-0229-1"]} />
<p>Pragmas can be used as an alternative to aspect{"_"}specifications to specify certain aspects. <br /></p>


#### Name Resolution Rules

<MarginText>2/5</MarginText>
<MarginInfo items={["AI12-0417-1"]} />
<p>Certain <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code>s are defined to be <em>program unit pragmas</em>. A <code><a href="/docs/arm/AA-4/AA-4.1#S0091">name</a></code> given as the argument of a program unit pragma shall resolve to denote the declarations or renamings of one or more program units that occur immediately within the declarative region or <code><a href="/docs/arm/AA-10/AA-10.1#S0285">compilation</a></code> in which the <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code> immediately occurs, or it shall resolve to denote the declaration of the immediately enclosing program unit (if any); the <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code> applies to the denoted program unit(s). If there are no <code><a href="/docs/arm/AA-4/AA-4.1#S0091">name</a></code>s given as arguments, the <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code> applies to the immediately enclosing program unit. <br /></p>

<AnnotatedOnly>
<MarginText>2.a/5</MarginText>
<Admonition type="aarm" aarm="ramification">
<strong></strong>The fact that this is a Name Resolution Rule means that the <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code> will not apply to declarations from outer declarative regions. <br />
</Admonition>
</AnnotatedOnly>


#### Legality Rules

<MarginText>3/5</MarginText>
<MarginInfo items={["AI12-0417-1"]} />
<p>A program unit pragma shall appear in one of these places: <br /></p>

<MarginText>4/5</MarginText>
<MarginInfo items={["AI12-0417-1"]} />
<ul>
<li>At the place of a <code><a href="/docs/arm/AA-10/AA-10.1#S0286">compilation_unit</a></code>, in which case the <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code> shall immediately follow in the same <code><a href="/docs/arm/AA-10/AA-10.1#S0285">compilation</a></code> (except for other <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code>s) a <code><a href="/docs/arm/AA-10/AA-10.1#S0288">library{"_"}unit{"_"}declaration</a></code> that is a <code><a href="/docs/arm/AA-6/AA-6.1#S0195">subprogram{"_"}declaration</a></code>, <code><a href="/docs/arm/AA-12/AA-12.1#S0311">generic{"_"}subprogram{"_"}declaration</a></code>, or <code><a href="/docs/arm/AA-12/AA-12.3#S0315">generic{"_"}instantiation</a></code>, and the <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code> shall have an argument that is a <code><a href="/docs/arm/AA-4/AA-4.1#S0091">name</a></code> denoting that declaration. <br /></li>
</ul>
<AnnotatedOnly>
<MarginText>4.a/5</MarginText>
<Admonition type="aarm" aarm="ramification">
<strong></strong>The <code><a href="/docs/arm/AA-4/AA-4.1#S0091">name</a></code> has to denote the immediately preceding <code><a href="/docs/arm/AA-10/AA-10.1#S0288">library_unit_declaration</a></code>. <br />
</Admonition>
</AnnotatedOnly>

<MarginText>5/5</MarginText>
<MarginInfo items={["AI95-00136-01", "AI12-0417-1"]} />
<ul>
<li>{"{"}<em>8652/0033</em>{"}"} Immediately within the visible part of a program unit and before any nested declaration (but not within a generic formal part), in which case the argument, if any, shall be a <code><a href="/docs/arm/AA-4/AA-4.1#S0092">direct_name</a></code> that denotes the immediately enclosing program unit declaration. <br /></li>
</ul>
<AnnotatedOnly>
<MarginText>5.a/5</MarginText>
<Admonition type="aarm" aarm="ramification">
<strong></strong>The argument is optional in this case. <br />
</Admonition>
</AnnotatedOnly>

<MarginText>6/5</MarginText>
<MarginInfo items={["AI12-0417-1"]} />
<ul>
<li>At the place of a declaration other than the first, of a <code><a href="/docs/arm/AA-3/AA-3.11#S0086">declarative_part</a></code> or program unit declaration, in which case the <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code> shall have an argument, which shall be a <code><a href="/docs/arm/AA-4/AA-4.1#S0092">direct_name</a></code> that denotes one or more of the following (and nothing else): a <code><a href="/docs/arm/AA-6/AA-6.1#S0195">subprogram{"_"}declaration</a></code>, a <code><a href="/docs/arm/AA-12/AA-12.1#S0311">generic{"_"}subprogram{"_"}declaration</a></code>, or a <code><a href="/docs/arm/AA-12/AA-12.3#S0315">generic{"_"}instantiation</a></code>, of the same <code><a href="/docs/arm/AA-3/AA-3.11#S0086">declarative{"_"}part</a></code> or program unit declaration. <br /></li>
</ul>
<AnnotatedOnly>
<MarginText>6.a/5</MarginText>
<Admonition type="aarm" aarm="ramification">
<strong></strong>If you want to denote a <code><a href="/docs/arm/AA-6/AA-6.3#S0216">subprogram_body</a></code> that is not a completion, or a <code><a href="/docs/arm/AA-7/AA-7.1#S0229">package_declaration</a></code>, for example, you have to put the <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code> inside. <br />
</Admonition>
</AnnotatedOnly>

<MarginText>7/5</MarginText>
<MarginInfo items={["AI05-0132-1", "AI12-0417-1"]} />
<p>Certain program unit pragmas are defined to be <em>library unit pragmas</em>. If a library unit pragma applies to a program unit, the program unit shall be a library unit. <br /></p>

<AnnotatedOnly>
<MarginText>7.a/5</MarginText>
<Admonition type="aarm" aarm="ramification">
<strong></strong>This, together with the rules for program unit pragmas above, implies that if a library unit pragma applies to a <code><a href="/docs/arm/AA-6/AA-6.1#S0195">subprogram_declaration</a></code> (and similar things), it has to appear immediately after the <code><a href="/docs/arm/AA-10/AA-10.1#S0286">compilation_unit</a></code>, whereas if the <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code> applies to a <code><a href="/docs/arm/AA-7/AA-7.1#S0229">package_declaration</a></code>, a <code><a href="/docs/arm/AA-6/AA-6.3#S0216">subprogram_body</a></code> that is not a completion (and similar things), it has to appear inside, as the first <code><a href="/docs/arm/AA-3/AA-3.11#S0087">declarative_item</a></code>. <br />
</Admonition>
</AnnotatedOnly>


#### Static Semantics

<MarginText>8/5</MarginText>
<MarginInfo items={["AI95-00041-01", "AI12-0417-1"]} />
<p>{"{"}<em>8652/0034</em>{"}"} A library unit pragma that applies to a generic unit does not apply to its instances, unless a specific rule for the pragma specifies the contrary. <br /></p>


#### Implementation Advice

<MarginText>9/5</MarginText>
<MarginInfo items={["AI95-00041-01", "AI12-0417-1"]} />
<p>{"{"}<em>8652/0034</em>{"}"} When applied to a generic unit, a program unit pragma that is not a library unit pragma should apply to each instance of the generic unit for which there is not an overriding pragma applied directly to the instance. <br /></p>

<AnnotatedOnly>
<MarginText>9.a/5</MarginText>
<Admonition type="aarm" aarm="implementation-advice">
<strong></strong>When applied to a generic unit, a program unit pragma that is not a library unit pragma should apply to each instance of the generic unit for which there is not an overriding pragma applied directly to the instance.<br />
</Admonition>
</AnnotatedOnly>

## J.15.1  Pragma Inline


#### Syntax

<MarginText>1/3_J.15.1</MarginText>
<MarginInfo items={["AI05-0229-1"]} />
<p class="Indented2">The form of a <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code> Inline, which is a program unit pragma (see <a href="/docs/arm/AA-10/AA-10.1#Subclause_10.1.5">10.1.5</a>), is as follows: <br /></p>

<MarginText>2/3_J.15.1</MarginText>
<p class="Indented2">  <strong>pragma</strong> Inline (<code><a href="/docs/arm/AA-4/AA-4.1#S0091">name</a></code>{"{"}, <code><a href="/docs/arm/AA-4/AA-4.1#S0091">name</a></code>{"}"}); <br /></p>


#### Legality Rules

<MarginText>3/3_J.15.1</MarginText>
<MarginInfo items={["AI05-0229-1"]} />
<p>The <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code> shall apply to one or more callable entities or generic subprograms. <br /></p>


#### Static Semantics

<MarginText>4/3_J.15.1</MarginText>
<MarginInfo items={["AI05-0229-1"]} />
<p><code><a href="/docs/arm/AA-2/AA-2.8#S0019">Pragma</a></code> Inline specifies that the Inline aspect (see <a href="/docs/arm/AA-6/AA-6.3#Subclause_6.3.2">6.3.2</a>) for each entity denoted by each <code><a href="/docs/arm/AA-4/AA-4.1#S0091">name</a></code> given in the <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code> has the value True.<br /></p>

<AnnotatedOnly>
<MarginText>4.a/3_J.15.1</MarginText>
<Admonition type="aarm" aarm="ramification">
<strong></strong>Note that inline expansion is desired no matter what <code><a href="/docs/arm/AA-4/AA-4.1#S0091">name</a></code> is used in the call. This allows one to request inlining for only one of several overloaded subprograms as follows:<br />
</Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>4.b/3_J.15.1</MarginText>
<CodeBlock language="ada">
package IO is{"\n"}   procedure Put(X : in Integer);{"\n"}   procedure Put(X : in String);{"\n"}   procedure Put(X : in Character);{"\n"}private{"\n"}   procedure Character{"_"}Put(X : in Character) renames Put;{"\n"}   pragma Inline(Character{"_"}Put);{"\n"}end IO;{"\n"}
<MarginText>4.c/3_J.15.1</MarginText>
with IO; use IO;{"\n"}procedure Main is{"\n"}   I : Integer;{"\n"}   C : Character;{"\n"}begin{"\n"}   ...{"\n"}   Put(C); -- Inline expansion is desired.{"\n"}   Put(I); -- Inline expansion is NOT desired.{"\n"}end Main;{"\n"}

</CodeBlock>
</AnnotatedOnly>

#### Implementation Permissions

<MarginText>5/3_J.15.1</MarginText>
<MarginInfo items={["AI05-0229-1"]} />
<p>An implementation may allow a <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code> Inline that has an argument which is a <code><a href="/docs/arm/AA-4/AA-4.1#S0092">direct_name</a></code> denoting a <code><a href="/docs/arm/AA-6/AA-6.3#S0216">subprogram_body</a></code> of the same <code><a href="/docs/arm/AA-3/AA-3.11#S0086">declarative_part</a></code>.<br /></p>

<AnnotatedOnly>
<MarginText>5.a/3_J.15.1</MarginText>
<Admonition type="aarm" aarm="reason">
<strong></strong>This is allowed for Ada 83 compatibility. This is only a permission as this usage was considered obsolescent even for Ada 95. <br />
</Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>5.b/3_J.15.1</MarginText>
<Admonition type="aarm" aarm="discussion">
<strong></strong>We only need to allow this in <code><a href="/docs/arm/AA-3/AA-3.11#S0086">declarative_part</a></code>s, because a <code><a href="/docs/arm/AA-3/AA-3.11#S0089">body</a></code> is only allowed in another <code><a href="/docs/arm/AA-3/AA-3.11#S0089">body</a></code>, and these all have <code><a href="/docs/arm/AA-3/AA-3.11#S0086">declarative_part</a></code>s. <br />
</Admonition>
</AnnotatedOnly>

<AnnotatedOnly>
<MarginText>6/5_J.15.1</MarginText>
<MarginInfo items={["AI05-0229-1", "AI12-0440-1"]} />
<Admonition type="aarm" aarm="note">
NOTE   The name in a <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code> Inline can denote more than one entity in the case of overloading. Such a <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code> applies to all of the denoted entities. <br />
</Admonition>
</AnnotatedOnly>

<AnnotatedOnly>

#### Incompatibilities With Ada 83

</AnnotatedOnly>
<AnnotatedOnly>
<MarginText>6.a/3_J.15.1</MarginText>
<MarginInfo items={["AI95-00309-01", "AI05-0229-1"]} />
<Admonition type="aarm" aarm="note">
A pragma Inline cannot refer to a <code><a href="/docs/arm/AA-6/AA-6.3#S0216">subprogram_body</a></code> outside of that body. The pragma can be given inside of the subprogram body. Ada 2005 adds an Implementation Permission to allow this usage for compatibility (and Ada 95 implementations also can use this permission), but implementations do not have to allow such <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code>s. <br />
</Admonition>
</AnnotatedOnly>

<AnnotatedOnly>

#### Extensions to Ada 83

</AnnotatedOnly>

<MarginText>6.b/3_J.15.1</MarginText>
<MarginInfo items={["AI05-0229-1"]} />
<Admonition type="aarm" aarm="note">
A <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code> Inline is allowed inside a <code><a href="/docs/arm/AA-6/AA-6.3#S0216">subprogram_body</a></code> if there is no corresponding <code><a href="/docs/arm/AA-6/AA-6.1#S0195">subprogram_declaration</a></code>. <strong>Amendment </strong> Implementations are allowed to let <code><a href="/docs/arm/AA-2/AA-2.8#S0019">Pragma</a></code> Inline apply to a <code><a href="/docs/arm/AA-6/AA-6.3#S0216">subprogram_body</a></code>. <br />

## J.15.4  Pragma Storage_Size


#### Syntax

<MarginText>1/3_J.15.4</MarginText>
<MarginInfo items={["AI05-0229-1"]} />
<p class="Indented2">The form of a <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code> Storage{"_"}Size is as follows: <br /></p>

<MarginText>2/3_J.15.4</MarginText>
<p class="Indented2">  <strong>pragma</strong> Storage{"_"}Size (<code><a href="/docs/arm/AA-4/AA-4.4#S0132">expression</a></code>);<br /></p>

<MarginText>3/3_J.15.4</MarginText>
<MarginInfo items={["AI05-0229-1"]} />
<p class="Indented2">A <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code> Storage{"_"}Size is allowed only immediately within a <code><a href="/docs/arm/AA-9/AA-9.1#S0246">task_definition</a></code>. <br /></p>

<MarginText>4/3_J.15.4</MarginText>
<MarginInfo items={["AI05-0229-1"]} />
<p>The <code><a href="/docs/arm/AA-4/AA-4.4#S0132">expression</a></code> of a <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code> Storage{"_"}Size is expected to be of any integer type. <br /></p>

## J.15.15  Distribution Pragmas

<MarginText>1/5_J.15.15</MarginText>
<MarginInfo items={["AI12-0417-1"]} />
<p>This subclause defines pragmas that specify properties of units for distributed systems.<br /></p>

<AnnotatedOnly>
<MarginText>1.a/5_J.15.15</MarginText>
<MarginInfo items={["AI12-0417-1"]} />
<Admonition type="aarm" aarm="discussion">
<strong></strong>These pragmas are presented in the order that the aspects are defined in <a href="/docs/arm/AA-E/">Annex E</a>. <br />
</Admonition>
</AnnotatedOnly>


#### Syntax

<MarginText>2/5_J.15.15</MarginText>
<MarginInfo items={["AI12-0417-1"]} />
<p class="Indented2">The following <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code>s are defined with the given forms: <br /></p>

<MarginText>3/5_J.15.15</MarginText>
<p class="Indented2">  <strong>pragma</strong> Shared{"_"}Passive[(<em>library{"_"}unit{"_"}</em><code><a href="/docs/arm/AA-4/AA-4.1#S0091">name</a></code>)];<br /></p>

<MarginText>4/5_J.15.15</MarginText>
<p class="Indented2">  <strong>pragma</strong> Remote{"_"}Types[(<em>library{"_"}unit{"_"}</em><code><a href="/docs/arm/AA-4/AA-4.1#S0091">name</a></code>)];<br /></p>

<MarginText>5/5_J.15.15</MarginText>
<p class="Indented2">  <strong>pragma</strong> Remote{"_"}Call{"_"}Interface[(<em>library{"_"}unit{"_"}</em><code><a href="/docs/arm/AA-4/AA-4.1#S0091">name</a></code>)];<br /></p>

<MarginText>6/5_J.15.15</MarginText>
<p class="Indented2">  <strong>pragma</strong> All{"_"}Calls{"_"}Remote[(<em>library{"_"}unit{"_"}</em><code><a href="/docs/arm/AA-4/AA-4.1#S0091">name</a></code>)];<br /></p>

<MarginText>7/5_J.15.15</MarginText>
<MarginInfo items={["AI12-0417-1"]} />
<p class="Indented2">Each of these pragmas is a library unit pragma. <br /></p>


#### Static Semantics

<MarginText>8/5_J.15.15</MarginText>
<MarginInfo items={["AI12-0417-1"]} />
<p>A <em>categorization pragma</em> is a pragma that specifies a corresponding categorization aspect.<br /></p>

<MarginText>9/5_J.15.15</MarginText>
<MarginInfo items={["AI12-0417-1"]} />
<p>The pragmas Shared{"_"}Passive, Remote{"_"}Types, and Remote{"_"}Call{"_"}Interface are categorization pragmas. In addition, the pragma Pure (see <a href="/docs/arm/AA-J/AA-J.15#Subclause_J.15.14">J.15.14</a>) is considered a categorization pragma. <br /></p>

<MarginText>10/5_J.15.15</MarginText>
<MarginInfo items={["AI12-0417-1"]} />
<p>A <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code> Shared{"_"}Passive specifies that a library unit is a shared passive library unit, namely that the Shared{"_"}Passive aspect (see <a href="/docs/arm/AA-E/AA-E.2#Subclause_E.2.1">E.2.1</a>) of the library unit is True.<br /></p>

<MarginText>11/5_J.15.15</MarginText>
<MarginInfo items={["AI12-0417-1"]} />
<p>A <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code> Remote{"_"}Types specifies that a library unit is a remote types library unit, namely that the Remote{"_"}Types aspect (see <a href="/docs/arm/AA-E/AA-E.2#Subclause_E.2.2">E.2.2</a>) of the library unit is True.<br /></p>

<MarginText>12_J.15.15</MarginText>
<p>A <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code> Remote{"_"}Call{"_"}Interface specifies that a library unit is a remote call interface, namely that the Remote{"_"}Call{"_"}Interface aspect (see E.2.3) of the library unit is True.<br /></p>

<MarginText>13_J.15.15</MarginText>
<p>A <code><a href="/docs/arm/AA-2/AA-2.8#S0019">pragma</a></code> All{"_"}Calls{"_"}Remote specifies that the All{"_"}Calls{"_"}Remote aspect (see E.2.3) of the library unit is True.<br /></p>
