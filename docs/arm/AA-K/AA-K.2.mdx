<p>{"{"}<em>{"AI05-0229-1"}</em>{"}"} {"{"}<em>{"AI05-0299-1"}</em>{"}"} This subclause summarizes the definitions given elsewhere of the language-defined attributes. Attributes are properties of entities that can be queried by an Ada program.<br /></p>

<p>P'AccessFor a <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> P that denotes a subprogram:<br /></p>

<p>P'Access yields an access value that designates the subprogram denoted by P. The type of P'Access is an access-to-subprogram type (<em>S</em>), as determined by the expected type. See 3.10.2.<br /></p>

<p>X'AccessFor a <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> X that denotes an aliased view of an object:<br /></p>

<p>X'Access yields an access value that designates the object denoted by X. The type of X'Access is an access-to-object type, as determined by the expected type. The expected type shall be a general access type. See 3.10.2.<br /></p>

<p>X'AddressFor a <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> X that denotes an object, program unit, or label:<br /></p>

<p>Denotes the address of the first of the storage elements allocated to X. For a program unit or label, this value refers to the machine code associated with the corresponding body or <code><a href="../AA-5/AA-5.1#S0167">statement</a></code>. The value of this attribute is of type System.Address. See 13.3.<br /></p>

<p>S'AdjacentFor every subtype S of a floating point type T:<br /></p>

<p>S'Adjacent denotes a function with the following specification: <br /></p>

<CodeBlock>
function S'Adjacent (X, Towards : T){"\n"}  return T{"\n"}

</CodeBlock>
<p>If Towards = X, the function yields X; otherwise, it yields the machine number of the type T adjacent to X in the direction of Towards, if that machine number exists. If the result would be outside the base range of S, Constraint_Error is raised. When T'Signed_Zeros is True, a zero result has the sign of X. When Towards is zero, its sign has no bearing on the result. See A.5.3.<br /></p>

<p>S'AftFor every fixed point subtype S:<br /></p>

<p>S'Aft yields the number of decimal digits needed after the decimal point to accommodate the <em>delta</em> of the subtype S, unless the <em>delta</em> of the subtype S is greater than 0.1, in which case the attribute yields the value one. (S'Aft is the smallest positive integer N for which (10**N)*S'Delta is greater than or equal to one.) The value of this attribute is of the type <em>universal_integer</em>. See 3.5.10.<br /></p>

<p>S'AlignmentFor every subtype S:<br /></p>

<p>The value of this attribute is of type <em>universal_integer</em>, and nonnegative.<br /></p>

<p>For an object X of subtype S, if S'Alignment is not zero, then X'Alignment is a nonzero integral multiple of S'Alignment unless specified otherwise by a representation item. See 13.3.<br /></p>

<p>X'AlignmentFor a <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> X that denotes an object:<br /></p>

<p>The value of this attribute is of type <em>universal_integer</em>, and nonnegative; zero means that the object is not necessarily aligned on a storage element boundary. If X'Alignment is not zero, then X is aligned on a storage unit boundary and X'Address is an integral multiple of X'Alignment (that is, the Address modulo the Alignment is zero).<br /></p>

<p><em>This paragraph was deleted.</em> See 13.3.<br /></p>

<p>S'BaseFor every scalar subtype S:<br /></p>

<p>S'Base denotes an unconstrained subtype of the type of S. This unconstrained subtype is called the <em>base subtype</em> of the type. See 3.5.<br /></p>

<p>S'Bit_OrderFor every specific record subtype S:<br /></p>

<p>Denotes the bit ordering for the type of S. The value of this attribute is of type System.Bit_Order. See 13.5.3.<br /></p>

<p>P'Body_VersionFor a <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> P that statically denotes a program unit:<br /></p>

<p>Yields a value of the predefined type String that identifies the version of the compilation unit that contains the body (but not any subunits) of the program unit. See E.3.<br /></p>

<p>T'CallableFor a <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> T that is of a task type (after any implicit dereference):<br /></p>

<p>Yields the value True when the task denoted by T is <em>callable</em>, and False otherwise; See 9.9.<br /></p>

<p>E'CallerFor a <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> E that denotes an <code><a href="../AA-9/AA-9.5#S0257">entry_declaration</a></code>:<br /></p>

<p>Yields a value of the type Task_Id that identifies the task whose call is now being serviced. Use of this attribute is allowed only inside an <code><a href="../AA-9/AA-9.5#S0258">accept_statement</a></code>, or <code><a href="../AA-9/AA-9.5#S0260">entry_body</a></code> after the <code><a href="../AA-9/AA-9.5#S0262">entry_barrier</a></code>, corresponding to the <code><a href="../AA-9/AA-9.5#S0257">entry_declaration</a></code> denoted by E. See C.7.1.<br /></p>

<p>S'CeilingFor every subtype S of a floating point type T:<br /></p>

<p>S'Ceiling denotes a function with the following specification: <br /></p>

<CodeBlock>
function S'Ceiling (X : T){"\n"}  return T{"\n"}

</CodeBlock>
<p>The function yields the value X, i.e., the smallest (most negative) integral value greater than or equal to X. When X is zero, the result has the sign of X; a zero result otherwise has a negative sign when S'Signed_Zeros is True. See A.5.3.<br /></p>

<p>S'ClassFor every subtype S of a tagged type <em>T</em> (specific or class-wide):<br /></p>

<p>S'Class denotes a subtype of the class-wide type (called <em>T</em>'Class in this document) for the class rooted at <em>T</em> (or if S already denotes a class-wide subtype, then S'Class is the same as S).<br /></p>

<p>S'Class is unconstrained. However, if S is constrained, then the values of S'Class are only those that when converted to the type <em>T</em> belong to S. See 3.9.<br /></p>

<p>S'ClassFor every subtype S of an untagged private type whose full view is tagged:<br /></p>

<p>Denotes the class-wide subtype corresponding to the full view of S. This attribute is allowed only from the beginning of the private part in which the full view is declared, until the declaration of the full view. After the full view, the Class attribute of the full view can be used. See 7.3.1.<br /></p>

<p>X'Component_SizeFor a <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> X that denotes an array subtype or array object (after any implicit dereference):<br /></p>

<p>Denotes the size in bits of components of the type of X. The value of this attribute is of type <em>universal_integer</em>. See 13.3.<br /></p>

<p>S'ComposeFor every subtype S of a floating point type T:<br /></p>

<p>S'Compose denotes a function with the following specification: <br /></p>

<CodeBlock>
function S'Compose (Fraction : T;{"\n"}                    Exponent : universal_integer){"\n"}  return T{"\n"}

</CodeBlock>
<p>Let v be the value Fraction  T'Machine_RadixExponentk, where k is the normalized exponent of Fraction. If v is a machine number of the type T, or if |v|  T'Model_Small, the function yields v; otherwise, it yields either one of the machine numbers of the type T adjacent to v. Constraint_Error is optionally raised if v is outside the base range of S. A zero result has the sign of Fraction when S'Signed_Zeros is True. See A.5.3.<br /></p>

<p>A'ConstrainedFor a <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> A that is of a discriminated type (after any implicit dereference):<br /></p>

<p>Yields the value True if A denotes a constant, a value, a tagged object, or a constrained variable, and False otherwise. The value of this attribute is of the predefined type Boolean. See 3.7.2.<br /></p>

<p>S'Copy_SignFor every subtype S of a floating point type T:<br /></p>

<p>S'Copy_Sign denotes a function with the following specification: <br /></p>

<CodeBlock>
function S'Copy_Sign (Value, Sign : T){"\n"}  return T{"\n"}

</CodeBlock>
<p>If the value of Value is nonzero, the function yields a result whose magnitude is that of Value and whose sign is that of Sign; otherwise, it yields the value zero. Constraint_Error is optionally raised if the result is outside the base range of S. A zero result has the sign of Sign when S'Signed_Zeros is True. See A.5.3.<br /></p>

<p>E'CountFor a <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> E that denotes an entry of a task or protected unit:<br /></p>

<p>Yields the number of calls presently queued on the entry E of the current instance of the unit. The value of this attribute is of the type <em>universal_integer</em>. See 9.9.<br /></p>

<p>S'DefiniteFor a <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> S that denotes a formal indefinite subtype:<br /></p>

<p>S'Definite yields True if the actual subtype corresponding to S is definite; otherwise, it yields False. The value of this attribute is of the predefined type Boolean. See 12.5.1.<br /></p>

<p>S'DeltaFor every fixed point subtype S:<br /></p>

<p>S'Delta denotes the <em>delta</em> of the fixed point subtype S. The value of this attribute is of the type <em>universal_real</em>. See 3.5.10.<br /></p>

<p>S'DenormFor every subtype S of a floating point type T:<br /></p>

<p>Yields the value True if every value expressible in the form<br />     mantissa  T'Machine_RadixT'Machine_Emin<br />where <em>mantissa</em> is a nonzero T'Machine_Mantissa-digit fraction in the number base T'Machine_Radix, the first digit of which is zero, is a machine number (see 3.5.7) of the type T; yields the value False otherwise. The value of this attribute is of the predefined type Boolean. See A.5.3.<br /></p>

<p>S'DigitsFor every floating point subtype S:<br /></p>

<p>S'Digits denotes the requested decimal precision for the subtype S. The value of this attribute is of the type <em>universal_integer</em>. See 3.5.8.<br /></p>

<p>S'DigitsFor every decimal fixed point subtype S:<br /></p>

<p>S'Digits denotes the <em>digits</em> of the decimal fixed point subtype S, which corresponds to the number of decimal digits that are representable in objects of the subtype. The value of this attribute is of the type <em>universal_integer</em>. See 3.5.10.<br /></p>

<p>S'Enum_RepFor every discrete subtype S:<br /></p>

<p>S'Enum_Rep denotes a function with the following specification: <br /></p>

<CodeBlock>
function S'Enum_Rep (Arg : S'Base) return universal_integer{"\n"}

</CodeBlock>
<p>This function returns the representation value of the value of Arg, as a value of type <em>universal_integer</em>. The <em>representation value</em> is the internal code specified in an enumeration representation clause, if any, for the type corresponding to the value of Arg, and otherwise is the position number of the value. See 13.4.<br /></p>

<p>S'Enum_ValFor every discrete subtype S:<br /></p>

<p>S'Enum_Val denotes a function with the following specification: <br /></p>

<CodeBlock>
function S'Enum_Val (Arg : universal_integer) return S'Base{"\n"}

</CodeBlock>
<p>This function returns a value of the type of S whose representation value equals the value of Arg. For the evaluation of a call on S'Enum_Val, if there is no value in the base range of its type with the given representation value, Constraint_Error is raised. See 13.4.<br /></p>

<p>S'ExponentFor every subtype S of a floating point type T:<br /></p>

<p>S'Exponent denotes a function with the following specification: <br /></p>

<CodeBlock>
function S'Exponent (X : T){"\n"}  return universal_integer{"\n"}

</CodeBlock>
<p>The function yields the normalized exponent of X. See A.5.3.<br /></p>

<p>S'External_TagFor every subtype S of a tagged type <em>T</em> (specific or class-wide):<br /></p>

<p>S'External_Tag denotes an external string representation for S'Tag; it is of the predefined type String. External_Tag may be specified for a specific tagged type via an <code><a href="../AA-13/AA-13.3#S0349">attribute_definition_clause</a></code>; the expression of such a clause shall be static. The default external tag representation is implementation defined. See 13.13.2. See 13.3.<br /></p>

<p>A'FirstFor a <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> A that is of an array type (after any implicit dereference), or denotes a constrained array subtype:<br /></p>

<p>A'First denotes the lower bound of the first index range; its type is the corresponding index type. See 3.6.2.<br /></p>

<p>S'FirstFor every scalar subtype S:<br /></p>

<p>S'First denotes the lower bound of the range of S. The value of this attribute is of the type of S. See 3.5.<br /></p>

<p>A'First(N)For a <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> A that is of an array type (after any implicit dereference), or denotes a constrained array subtype:<br /></p>

<p>A'First(N) denotes the lower bound of the N-th index range; its type is the corresponding index type. See 3.6.2.<br /></p>

<p>R.C'First_BitFor a component C of a composite, non-array object R:<br /></p>

<p>If the nondefault bit ordering applies to the composite type, and if a <code><a href="../AA-13/AA-13.5#S0353">component_clause</a></code> specifies the placement of C, denotes the value given for the <code><a href="../AA-13/AA-13.5#S0355">first_bit</a></code> of the <code><a href="../AA-13/AA-13.5#S0353">component_clause</a></code>; otherwise, denotes the offset, from the start of the first of the storage elements occupied by C, of the first bit occupied by C. This offset is measured in bits. The first bit of a storage element is numbered zero. The value of this attribute is of the type <em>universal_integer</em>. See 13.5.2.<br /></p>

<p>S'First_ValidFor every static discrete subtype S for which there exists at least one value belonging to S that satisfies the predicates of S:<br /></p>

<p>S'First_Valid denotes the smallest value that belongs to S and satisfies the predicates of S. The value of this attribute is of the type of S. See 3.5.5.<br /></p>

<p>S'FloorFor every subtype S of a floating point type T:<br /></p>

<p>S'Floor denotes a function with the following specification: <br /></p>

<CodeBlock>
function S'Floor (X : T){"\n"}  return T{"\n"}

</CodeBlock>
<p>The function yields the value X, i.e., the largest (most positive) integral value less than or equal to X. When X is zero, the result has the sign of X; a zero result otherwise has a positive sign. See A.5.3.<br /></p>

<p>S'ForeFor every fixed point subtype S:<br /></p>

<p>S'Fore yields the minimum number of characters needed before the decimal point for the decimal representation of any value of the subtype S, assuming that the representation does not include an exponent, but includes a one-character prefix that is either a minus sign or a space. (This minimum number does not include superfluous zeros or underlines, and is at least 2.) The value of this attribute is of the type <em>universal_integer</em>. See 3.5.10.<br /></p>

<p>S'FractionFor every subtype S of a floating point type T:<br /></p>

<p>S'Fraction denotes a function with the following specification: <br /></p>

<CodeBlock>
function S'Fraction (X : T){"\n"}  return T{"\n"}

</CodeBlock>
<p>The function yields the value X  T'Machine_Radixk, where k is the normalized exponent of X. A zero result, which can only occur when X is zero, has the sign of X. See A.5.3.<br /></p>

<p>X'Has_Same_StorageFor a <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> X that denotes an object:<br /></p>

<p>X'Has_Same_Storage denotes a function with the following specification:<br /></p>

<CodeBlock>
function X'Has_Same_Storage (Arg : any_type){"\n"}  return Boolean{"\n"}

</CodeBlock>
<p>The actual parameter shall be a name that denotes an object. The object denoted by the actual parameter can be of any type. This function evaluates the names of the objects involved. It returns True if the representation of the object denoted by the actual parameter occupies exactly the same bits as the representation of the object denoted by X and the objects occupy at least one bit; otherwise, it returns False. See 13.3.<br /></p>

<p>E'IdentityFor a <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> E that denotes an exception:<br /></p>

<p>E'Identity returns the unique identity of the exception. The type of this attribute is Exception_Id. See 11.4.1.<br /></p>

<p>T'IdentityFor a <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> T that is of a task type (after any implicit dereference):<br /></p>

<p>Yields a value of the type Task_Id that identifies the task denoted by T. See C.7.1.<br /></p>

<p>S'ImageFor every subtype S of a type T:<br /></p>

<p>S'Image denotes a function with the following specification: <br /></p>

<CodeBlock>
function S'Image(Arg : S'Base){"\n"}  return String{"\n"}

</CodeBlock>
<p>S'Image calls S'Put_Image passing Arg (which will typically store a sequence of character values in a text buffer) and then returns the result of retrieving the contents of that buffer with function Get. See 4.10.<br /></p>

<p>X'ImageFor a <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> X of a type T other than <em>universal_real</em> or <em>universal_fixed</em>:<br /></p>

<p>X'Image denotes the result of calling function S'Image with <em>Arg</em> being X, where S is the nominal subtype of X. See 4.10.<br /></p>

<p>E'IndexFor a <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> E that denotes an entry declaration of an entry family:<br /></p>

<p>Within a precondition or postcondition expression for entry family E, denotes the value of the entry index for the call of E. The nominal subtype of this attribute is the entry index subtype. See 6.1.1.<br /></p>

<p>S'Class'InputFor every subtype S'Class of a class-wide type <em>T</em>'Class:<br /></p>

<p>S'Class'Input denotes a function with the following specification: <br /></p>

<CodeBlock>
function S'Class'Input({"\n"}   Stream : not null access Ada.Streams.Root_Stream_Type'Class){"\n"}   return T'Class{"\n"}

</CodeBlock>
<p>First reads the external tag from <em>Stream</em> and determines the corresponding internal tag (by calling Tags.Descendant_Tag(String'Input(<em>Stream</em>), S'Tag) which can raise Tag_Error - see 3.9) and then dispatches to the subprogram denoted by the Input attribute of the specific type identified by the internal tag; returns that result. If the specific type identified by the internal tag is abstract, Constraint_Error is raised. See 13.13.2.<br /></p>

<p>S'InputFor every subtype S of a specific type <em>T</em>:<br /></p>

<p>S'Input denotes a function with the following specification: <br /></p>

<CodeBlock>
function S'Input({"\n"}   Stream : not null access Ada.Streams.Root_Stream_Type'Class){"\n"}   return T{"\n"}

</CodeBlock>
<p>S'Input reads and returns one value from <em>Stream</em>, using any bounds or discriminants written by a corresponding S'Output to determine how much to read. See 13.13.2.<br /></p>

<p>A'LastFor a <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> A that is of an array type (after any implicit dereference), or denotes a constrained array subtype:<br /></p>

<p>A'Last denotes the upper bound of the first index range; its type is the corresponding index type. See 3.6.2.<br /></p>

<p>S'LastFor every scalar subtype S:<br /></p>

<p>S'Last denotes the upper bound of the range of S. The value of this attribute is of the type of S. See 3.5.<br /></p>

<p>A'Last(N)For a <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> A that is of an array type (after any implicit dereference), or denotes a constrained array subtype:<br /></p>

<p>A'Last(N) denotes the upper bound of the N-th index range; its type is the corresponding index type. See 3.6.2.<br /></p>

<p>R.C'Last_BitFor a component C of a composite, non-array object R:<br /></p>

<p>If the nondefault bit ordering applies to the composite type, and if a <code><a href="../AA-13/AA-13.5#S0353">component_clause</a></code> specifies the placement of C, denotes the value given for the <code><a href="../AA-13/AA-13.5#S0356">last_bit</a></code> of the <code><a href="../AA-13/AA-13.5#S0353">component_clause</a></code>; otherwise, denotes the offset, from the start of the first of the storage elements occupied by C, of the last bit occupied by C. This offset is measured in bits. The value of this attribute is of the type <em>universal_integer</em>. See 13.5.2.<br /></p>

<p>S'Last_ValidFor every static discrete subtype S for which there exists at least one value belonging to S that satisfies the predicates of S:<br /></p>

<p>S'Last_Valid denotes the largest value that belongs to S and satisfies the predicates of S. The value of this attribute is of the type of S. See 3.5.5.<br /></p>

<p>S'Leading_PartFor every subtype S of a floating point type T:<br /></p>

<p>S'Leading_Part denotes a function with the following specification: <br /></p>

<CodeBlock>
function S'Leading_Part (X : T;{"\n"}                         Radix_Digits : universal_integer){"\n"}  return T{"\n"}

</CodeBlock>
<p>Let v be the value T'Machine_RadixkRadix_Digits, where k is the normalized exponent of X. The function yields the value <br /></p>

<p>X/v  v, when X is nonnegative and Radix_Digits is positive;<br /></p>

<p>X/v  v, when X is negative and Radix_Digits is positive. <br /></p>

<p>Constraint_Error is raised when Radix_Digits is zero or negative. A zero result, which can only occur when X is zero, has the sign of X. See A.5.3.<br /></p>

<p>A'LengthFor a <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> A that is of an array type (after any implicit dereference), or denotes a constrained array subtype:<br /></p>

<p>A'Length denotes the number of values of the first index range (zero for a null range); its type is <em>universal_integer</em>. See 3.6.2.<br /></p>

<p>A'Length(N)For a <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> A that is of an array type (after any implicit dereference), or denotes a constrained array subtype:<br /></p>

<p>A'Length(N) denotes the number of values of the N-th index range (zero for a null range); its type is <em>universal_integer</em>. See 3.6.2.<br /></p>

<p>S'MachineFor every subtype S of a floating point type T:<br /></p>

<p>S'Machine denotes a function with the following specification: <br /></p>

<CodeBlock>
function S'Machine (X : T){"\n"}  return T{"\n"}

</CodeBlock>
<p>If X is a machine number of the type T, the function yields X; otherwise, it yields the value obtained by rounding or truncating X to either one of the adjacent machine numbers of the type T. Constraint_Error is raised if rounding or truncating X to the precision of the machine numbers results in a value outside the base range of S. A zero result has the sign of X when S'Signed_Zeros is True. See A.5.3.<br /></p>

<p>S'Machine_EmaxFor every subtype S of a floating point type T:<br /></p>

<p>Yields the largest (most positive) value of <em>exponent</em> such that every value expressible in the canonical form (for the type T), having a <em>mantissa</em> of T'Machine_Mantissa digits, is a machine number (see 3.5.7) of the type T. This attribute yields a value of the type <em>universal_integer</em>. See A.5.3.<br /></p>

<p>S'Machine_EminFor every subtype S of a floating point type T:<br /></p>

<p>Yields the smallest (most negative) value of <em>exponent</em> such that every value expressible in the canonical form (for the type T), having a <em>mantissa</em> of T'Machine_Mantissa digits, is a machine number (see 3.5.7) of the type T. This attribute yields a value of the type <em>universal_integer</em>. See A.5.3.<br /></p>

<p>S'Machine_MantissaFor every subtype S of a floating point type T:<br /></p>

<p>Yields the largest value of p such that every value expressible in the canonical form (for the type T), having a p-digit <em>mantissa</em> and an <em>exponent</em> between T'Machine_Emin and T'Machine_Emax, is a machine number (see 3.5.7) of the type T. This attribute yields a value of the type <em>universal_integer</em>. See A.5.3.<br /></p>

<p>S'Machine_OverflowsFor every subtype S of a floating point type T:<br /></p>

<p>Yields the value True if overflow and divide-by-zero are detected and reported by raising Constraint_Error for every predefined operation that yields a result of the type T; yields the value False otherwise. The value of this attribute is of the predefined type Boolean. See A.5.3.<br /></p>

<p>S'Machine_OverflowsFor every subtype S of a fixed point type T:<br /></p>

<p>Yields the value True if overflow and divide-by-zero are detected and reported by raising Constraint_Error for every predefined operation that yields a result of the type T; yields the value False otherwise. The value of this attribute is of the predefined type Boolean. See A.5.4.<br /></p>

<p>S'Machine_RadixFor every subtype S of a floating point type T:<br /></p>

<p>Yields the radix of the hardware representation of the type T. The value of this attribute is of the type <em>universal_integer</em>. See A.5.3.<br /></p>

<p>S'Machine_RadixFor every subtype S of a fixed point type T:<br /></p>

<p>Yields the radix of the hardware representation of the type T. The value of this attribute is of the type <em>universal_integer</em>. See A.5.4.<br /></p>

<p>S'Machine_RoundingFor every subtype S of a floating point type T:<br /></p>

<p>S'Machine_Rounding denotes a function with the following specification: <br /></p>

<CodeBlock>

<p>S'Wide_ImageFor every subtype S of a type T:<br /></p>

<p>S'Wide_Image denotes a function with the following specification: <br /></p>

<CodeBlock>
function S'Wide_Image(Arg : S'Base){"\n"}  return Wide_String{"\n"}

</CodeBlock>
<p>S'Wide_Image calls S'Put_Image passing Arg (which will typically store a sequence of character values in a text buffer) and then returns the result of retrieving the contents of that buffer with function Wide_Get. See 4.10.<br /></p>

<p>X'Wide_ImageFor a <code><a href="../AA-4/AA-4.1#S0093">prefix</a></code> X of a type T other than <em>universal_real</em> or <em>universal_fixed</em>:<br /></p>

<p>X'Wide_Image denotes the result of calling function S'Wide_Image with <em>Arg</em> being X, where S is the nominal subtype of X. See 4.10.<br /></p>
