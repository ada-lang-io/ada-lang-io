<p>This subclause summarizes the definitions given elsewhere of the language-defined attributes. Attributes are properties of entities that can be queried by an Ada program.<br /></p>

<MarginText>2</MarginText>
<dt><br/>P'Access</dt>
<dl>
<dd>For a <code><a href="/docs/arm/AA-4/AA-4.1#S0093">prefix</a></code> P that denotes a subprogram:<br /></dd>
<MarginText>3</MarginText>
<dd>P'Access yields an access value that designates the subprogram denoted by P. The type of P'Access is an access-to-subprogram type (<em>S</em>), as determined by the expected type. See <a href="/docs/arm/AA-3/AA-3.10#Subclause_3.10.2">3.10.2</a>.<br /></dd>
<MarginText>4</MarginText>
<dt><br/>X'Access</dt>
<dd>For a <code><a href="/docs/arm/AA-4/AA-4.1#S0093">prefix</a></code> X that denotes an aliased view of an object:<br /></dd>
<MarginText>5</MarginText>
<dd>X'Access yields an access value that designates the object denoted by X. The type of X'Access is an access-to-object type, as determined by the expected type. The expected type shall be a general access type. See <a href="/docs/arm/AA-3/AA-3.10#Subclause_3.10.2">3.10.2</a>.<br /></dd>
<MarginText>6/1</MarginText>
<dt><br/>X'Address</dt>
<dd>For a <code><a href="/docs/arm/AA-4/AA-4.1#S0093">prefix</a></code> X that denotes an object, program unit, or label:<br /></dd>
<MarginText>7</MarginText>
<dd>Denotes the address of the first of the storage elements allocated to X. For a program unit or label, this value refers to the machine code associated with the corresponding body or <code><a href="/docs/arm/AA-5/AA-5.1#S0167">statement</a></code>. The value of this attribute is of type System.Address. See <a href="/docs/arm/AA-13/AA-13.3">13.3</a>.<br /></dd>
<MarginText>8</MarginText>
<dt><br/>S'Adjacent</dt>
<dd>For every subtype S of a floating point type <em>T</em>:<br /></dd>
<MarginText>9</MarginText>
<dd>S'Adjacent denotes a function with the following specification: <br /></dd>
</dl>
<MarginText>10</MarginText>
<CodeBlock language="ada">
function S'Adjacent (X, Towards : T){"\n"}  return T{"\n"}

</CodeBlock>
<MarginText>11</MarginText>
<dl>
<dd>If <em>Towards</em> = <em>X</em>, the function yields <em>X</em>; otherwise, it yields the machine number of the type <em>T</em> adjacent to <em>X</em> in the direction of <em>Towards</em>, if that machine number exists. If the result would be outside the base range of S, Constraint{"_"}Error is raised. When <em>T</em>'Signed{"_"}Zeros is True, a zero result has the sign of <em>X</em>. When <em>Towards</em> is zero, its sign has no bearing on the result. See <a href="/docs/arm/AA-A/AA-A.5#Subclause_A.5.3">A.5.3</a>.<br /></dd>
<MarginText>12</MarginText>
<dt><br/>S'Aft</dt>
<dd>For every fixed point subtype S:<br /></dd>
<MarginText>13</MarginText>
<dd>S'Aft yields the number of decimal digits needed after the decimal point to accommodate the <em>delta</em> of the subtype S, unless the <em>delta</em> of the subtype S is greater than 0.1, in which case the attribute yields the value one. (S'Aft is the smallest positive integer N for which (10{"*"}{"*"}N){"*"}S'Delta is greater than or equal to one.) The value of this attribute is of the type <em>universal{"_"}integer</em>. See <a href="/docs/arm/AA-3/AA-3.5#Subclause_3.5.10">3.5.10</a>.<br /></dd>
<MarginText>13.1/2</MarginText>
<dt><br/>S'Alignment</dt>
<dd>For every subtype S:<br /></dd>
<MarginText>13.2/2</MarginText>
<dd>The value of this attribute is of type <em>universal{"_"}integer</em>, and nonnegative.<br /></dd>
<MarginText>13.3/2</MarginText>
<dd>For an object X of subtype S, if S'Alignment is not zero, then X'Alignment is a nonzero integral multiple of S'Alignment unless specified otherwise by a representation item. See <a href="/docs/arm/AA-13/AA-13.3">13.3</a>.<br /></dd>
<MarginText>14/1</MarginText>
<dt><br/>X'Alignment</dt>
<dd>For a <code><a href="/docs/arm/AA-4/AA-4.1#S0093">prefix</a></code> X that denotes an object:<br /></dd>
<MarginText>15/2</MarginText>
<dd>The value of this attribute is of type <em>universal{"_"}integer</em>, and nonnegative; zero means that the object is not necessarily aligned on a storage element boundary. If X'Alignment is not zero, then X is aligned on a storage unit boundary and X'Address is an integral multiple of X'Alignment (that is, the Address modulo the Alignment is zero).<br /></dd>

<dt><br/></dt>
<dd><em>This paragraph was deleted.</em> See <a href="/docs/arm/AA-13/AA-13.3">13.3</a>.<br /></dd>
<MarginText>17</MarginText>
<dt><br/>S'Base</dt>
<dd>For every scalar subtype S:<br /></dd>
<MarginText>18</MarginText>
<dd>S'Base denotes an unconstrained subtype of the type of S. This unconstrained subtype is called the <em>base subtype</em> of the type. See <a href="/docs/arm/AA-3/AA-3.5">3.5</a>.<br /></dd>
<MarginText>19</MarginText>
<dt><br/>S'Bit{"_"}Order</dt>
<dd>For every specific record subtype S:<br /></dd>
<MarginText>20</MarginText>
<dd>Denotes the bit ordering for the type of S. The value of this attribute is of type System.Bit{"_"}Order. See <a href="/docs/arm/AA-13/AA-13.5#Subclause_13.5.3">13.5.3</a>.<br /></dd>
<MarginText>21/1</MarginText>
<dt><br/>P'Body{"_"}Version</dt>
<dd>For a <code><a href="/docs/arm/AA-4/AA-4.1#S0093">prefix</a></code> P that statically denotes a program unit:<br /></dd>
<MarginText>22</MarginText>
<dd>Yields a value of the predefined type String that identifies the version of the compilation unit that contains the body (but not any subunits) of the program unit. See <a href="/docs/arm/AA-E/AA-E.3">E.3</a>.<br /></dd>
<MarginText>23</MarginText>
<dt><br/>T'Callable</dt>
<dd>For a <code><a href="/docs/arm/AA-4/AA-4.1#S0093">prefix</a></code> T that is of a task type (after any implicit dereference):<br /></dd>
<MarginText>24</MarginText>
<dd>Yields the value True when the task denoted by T is <em>callable</em>, and False otherwise; See <a href="/docs/arm/AA-9/AA-9.9">9.9</a>.<br /></dd>
<MarginText>25</MarginText>
<dt><br/>E'Caller</dt>
<dd>For a <code><a href="/docs/arm/AA-4/AA-4.1#S0093">prefix</a></code> E that denotes an <code><a href="/docs/arm/AA-9/AA-9.5#S0257">entry_declaration</a></code>:<br /></dd>
<MarginText>26/3</MarginText>
<dd>Yields a value of the type Task{"_"}Id that identifies the task whose call is now being serviced. Use of this attribute is allowed only inside an <code><a href="/docs/arm/AA-9/AA-9.5#S0258">accept_statement</a></code>, or <code><a href="/docs/arm/AA-9/AA-9.5#S0260">entry_body</a></code> after the <code><a href="/docs/arm/AA-9/AA-9.5#S0262">entry_barrier</a></code>, corresponding to the <code><a href="/docs/arm/AA-9/AA-9.5#S0257">entry_declaration</a></code> denoted by E. See <a href="/docs/arm/AA-C/AA-C.7#Subclause_C.7.1">C.7.1</a>.<br /></dd>
<MarginText>27</MarginText>
<dt><br/>S'Ceiling</dt>
<dd>For every subtype S of a floating point type <em>T</em>:<br /></dd>
<MarginText>28</MarginText>
<dd>S'Ceiling denotes a function with the following specification: <br /></dd>
</dl>
<MarginText>29</MarginText>
<CodeBlock language="ada">
function S'Ceiling (X : T){"\n"}  return T{"\n"}

</CodeBlock>
<MarginText>30/5</MarginText>
<dl>
<dd>The function yields the value &lceil;<em>X</em>&rceil;, that is, the smallest (most negative) integral value greater than or equal to <em>X</em>. When <em>X</em> is zero, the result has the sign of <em>X</em>; a zero result otherwise has a negative sign when S'Signed{"_"}Zeros is True. See <a href="/docs/arm/AA-A/AA-A.5#Subclause_A.5.3">A.5.3</a>.<br /></dd>
<MarginText>31</MarginText>
<dt><br/>S'Class</dt>
<dd>For every subtype S of a tagged type <em>T</em> (specific or class-wide):<br /></dd>
<MarginText>32</MarginText>
<dd>S'Class denotes a subtype of the class-wide type (called <em>T</em>'Class in this document) for the class rooted at <em>T</em> (or if S already denotes a class-wide subtype, then S'Class is the same as S).<br /></dd>
<MarginText>33</MarginText>
<dd>S'Class is unconstrained. However, if S is constrained, then the values of S'Class are only those that when converted to the type <em>T</em> belong to S. See <a href="/docs/arm/AA-3/AA-3.9">3.9</a>.<br /></dd>
<MarginText>34</MarginText>
<dt><br/>S'Class</dt>
<dd>For every subtype S of an untagged private type whose full view is tagged:<br /></dd>
<MarginText>35</MarginText>
<dd>Denotes the class-wide subtype corresponding to the full view of S. This attribute is allowed only from the beginning of the private part in which the full view is declared, until the declaration of the full view. After the full view, the Class attribute of the full view can be used. See <a href="/docs/arm/AA-7/AA-7.3#Subclause_7.3.1">7.3.1</a>.<br /></dd>
<MarginText>36/1</MarginText>
<dt><br/>X'Component{"_"}Size</dt>
<dd>For a <code><a href="/docs/arm/AA-4/AA-4.1#S0093">prefix</a></code> X that denotes an array subtype or array object (after any implicit dereference):<br /></dd>
<MarginText>37</MarginText>
<dd>Denotes the size in bits of components of the type of X. The value of this attribute is of type <em>universal{"_"}integer</em>. See <a href="/docs/arm/AA-13/AA-13.3">13.3</a>.<br /></dd>
<MarginText>38</MarginText>
<dt><br/>S'Compose</dt>
<dd>For every subtype S of a floating point type <em>T</em>:<br /></dd>
<MarginText>39</MarginText>
<dd>S'Compose denotes a function with the following specification: <br /></dd>
</dl>
<MarginText>40</MarginText>
<CodeBlock language="ada">
function S'Compose (Fraction : T;{"\n"}                    Exponent : universal{"_"}integer){"\n"}  return T{"\n"}

</CodeBlock>
<MarginText>41</MarginText>
<dl>
<dd>Let <em>v</em> be the value <em>Fraction</em> · <em>T</em>'Machine{"_"}Radix<em>Exponent</em>&ndash;<em>k</em>, where <em>k</em> is the normalized exponent of <em>Fraction</em>. If <em>v</em> is a machine number of the type <em>T</em>, or if |<em>v</em>| &ge; <em>T</em>'Model{"_"}Small, the function yields <em>v</em>; otherwise, it yields either one of the machine numbers of the type <em>T</em> adjacent to <em>v</em>. Constraint{"_"}Error is optionally raised if <em>v</em> is outside the base range of S. A zero result has the sign of <em>Fraction</em> when S'Signed{"_"}Zeros is True. See <a href="/docs/arm/AA-A/AA-A.5#Subclause_A.5.3">A.5.3</a>.<br /></dd>
<MarginText>42</MarginText>
<dt><br/>A'Constrained</dt>
<dd>For a <code><a href="/docs/arm/AA-4/AA-4.1#S0093">prefix</a></code> A that is of a discriminated type (after any implicit dereference):<br /></dd>
<MarginText>43/5</MarginText>
<dd>Yields the value True if A denotes a constant, a value, a tagged object, or a constrained variable, and False otherwise. The value of this attribute is of the predefined type Boolean. See <a href="/docs/arm/AA-3/AA-3.7#Subclause_3.7.2">3.7.2</a>.<br /></dd>
<MarginText>44</MarginText>
<dt><br/>S'Copy{"_"}Sign</dt>
<dd>For every subtype S of a floating point type <em>T</em>:<br /></dd>
<MarginText>45</MarginText>
<dd>S'Copy{"_"}Sign denotes a function with the following specification: <br /></dd>
</dl>
<MarginText>46</MarginText>
<CodeBlock language="ada">
function S'Copy{"_"}Sign (Value, Sign : T){"\n"}  return T{"\n"}

</CodeBlock>
<MarginText>47</MarginText>
<dl>
<dd>If the value of <em>Value</em> is nonzero, the function yields a result whose magnitude is that of <em>Value</em> and whose sign is that of <em>Sign</em>; otherwise, it yields the value zero. Constraint{"_"}Error is optionally raised if the result is outside the base range of S. A zero result has the sign of <em>Sign</em> when S'Signed{"_"}Zeros is True. See <a href="/docs/arm/AA-A/AA-A.5#Subclause_A.5.3">A.5.3</a>.<br /></dd>
<MarginText>48</MarginText>
<dt><br/>E'Count</dt>
<dd>For a <code><a href="/docs/arm/AA-4/AA-4.1#S0093">prefix</a></code> E that denotes an entry of a task or protected unit:<br /></dd>
<MarginText>49</MarginText>
<dd>Yields the number of calls presently queued on the entry E of the current instance of the unit. The value of this attribute is of the type <em>universal{"_"}integer</em>. See <a href="/docs/arm/AA-9/AA-9.9">9.9</a>.<br /></dd>
<MarginText>50/1</MarginText>
<dt><br/>S'Definite</dt>
<dd>For a <code><a href="/docs/arm/AA-4/AA-4.1#S0093">prefix</a></code> S that denotes a formal indefinite subtype:<br /></dd>
<MarginText>51/3</MarginText>
<dd>S'Definite yields True if the actual subtype corresponding to S is definite; otherwise, it yields False. The value of this attribute is of the predefined type Boolean. See <a href="/docs/arm/AA-12/AA-12.5#Subclause_12.5.1">12.5.1</a>.<br /></dd>
<MarginText>52</MarginText>
<dt><br/>S'Delta</dt>
<dd>For every fixed point subtype S:<br /></dd>
<MarginText>53</MarginText>
<dd>S'Delta denotes the <em>delta</em> of the fixed point subtype S. The value of this attribute is of the type <em>universal{"_"}real</em>. See <a href="/docs/arm/AA-3/AA-3.5#Subclause_3.5.10">3.5.10</a>.<br /></dd>
<MarginText>54</MarginText>
<dt><br/>S'Denorm</dt>
<dd>For every subtype S of a floating point type <em>T</em>:<br /></dd>
<MarginText>55</MarginText>
<dd>Yields the value True if every value expressible in the form<br />    ± <em>mantissa</em> · <em>T</em>'Machine{"_"}Radix<em>T</em>'Machine{"_"}Emin<br />where <em>mantissa</em> is a nonzero <em>T</em>'Machine{"_"}Mantissa-digit fraction in the number base <em>T</em>'Machine{"_"}Radix, the first digit of which is zero, is a machine number (see <a href="/docs/arm/AA-3/AA-3.5#Subclause_3.5.7">3.5.7</a>) of the type <em>T</em>; yields the value False otherwise. The value of this attribute is of the predefined type Boolean. See <a href="/docs/arm/AA-A/AA-A.5#Subclause_A.5.3">A.5.3</a>.<br /></dd>
<MarginText>56</MarginText>
<dt><br/>S'Digits</dt>
<dd>For every floating point subtype S:<br /></dd>
<MarginText>57</MarginText>
<dd>S'Digits denotes the requested decimal precision for the subtype S. The value of this attribute is of the type <em>universal{"_"}integer</em>. See <a href="/docs/arm/AA-3/AA-3.5#Subclause_3.5.8">3.5.8</a>.<br /></dd>
<MarginText>58</MarginText>

The value of this attribute is of the type <em>universal{"_"}integer</em>. See <a href="/docs/arm/AA-3/AA-3.5#Subclause_3.5.10">3.5.10</a>.<br /></dd>
<MarginText>59.1/5</MarginText>
<dt><br/>S'Enum{"_"}Rep</dt>

<MarginText>74</MarginText>

</dl>
<MarginText>76</MarginText>
<CodeBlock language="ada">
function S'Floor (X : T){"\n"}  return T{"\n"}

</CodeBlock>
<MarginText>77/5</MarginText>

When <em>X</em> is zero, the result has the sign of <em>X</em>; a zero result otherwise has a positive sign. See <a href="/docs/arm/AA-A/AA-A.5#Subclause_A.5.3">A.5.3</a>.<br /></dd>
<MarginText>78</MarginText>

<dd>For every fixed point subtype S:<br /></dd>
<MarginText>79</MarginText>
<dd>S'Fore yields the minimum number of characters needed before the decimal point for the decimal representation of any value of the subtype S, assuming that the representation does not include an exponent, but includes a one-character prefix that is either a minus sign or a space. (This minimum number does not include superfluous zeros or underlines, and is at least 2.) The value of this attribute is of the type <em>universal{"_"}integer</em>.
